<objective>
Protocol-branch the `-rtsp_transport tcp` flag in both `ffprobe.service.ts` and `ffmpeg-command.builder.ts`. Today both unconditionally emit the flag for every stream URL — which is silently ignored for RTMP/SRT but pollutes ffmpeg logs and will break on stricter future ffmpeg versions. D-13 locks this as a correctness fix: emit the flag ONLY for `rtsp://` URLs.

Purpose: Make RTMP/RTMPS/SRT ingest clean end-to-end. This is the last piece (after P01 DTO allowlist) that lets an operator register an `rtmp://` camera and have the FFmpeg pipeline work without warnings.

Output: Both builder files gain a 3-line conditional; existing Wave 0 test stubs (`ffmpeg-command-builder.test.ts` + `ffprobe.test.ts`) convert to real passing assertions.
</objective>

/** Returns the input-specific flags string for ffprobe. D-13 — only RTSP needs -rtsp_transport. */
