## Interaction Patterns

### Row-Click Entry (from `/app/recordings` DataTable)

- Full row click (anywhere except checkbox cell and actions menu) navigates to `/app/recordings/[recording.id]` via `router.push`.
- Checkbox click: toggles selection (unchanged).
- Row actions (Download, Delete) dropdown: unchanged; `e.stopPropagation()` on its trigger so it doesn't cascade to row navigation.
- Row cursor: `cursor-pointer` on non-interactive cells.
- Keyboard: each row becomes focusable (`tabIndex={0}`), `Enter` key triggers navigation, `aria-label={"Open recording from " + camera.name + " at " + timeRange}`.

### Date Navigation

- Prev day button: `setDate(subDays(date, 1))`. Prefetches timeline + list via SWR-like hook cache.
- Next day button: `setDate(addDays(date, 1))`. Disabled when `date` ≥ today (startOfDay comparison).
- Date label button → opens shadcn `Popover` with `Calendar` inside:
  - `mode="single"`, `selected={date}`, `onSelect={(d) => { if (d) { setDate(d); setPopoverOpen(false); } }}`
  - `modifiers={{ hasRecording: daysWithRecordingsInCurrentMonth }}`, `modifiersClassNames={{ hasRecording: "relative after:absolute after:bottom-1 after:left-1/2 after:-translate-x-1/2 after:h-1 after:w-1 after:rounded-full after:bg-chart-1" }}` (no inline styles — use Tailwind classes).
  - Month change re-runs `useRecordingCalendar(cameraId, year, month)`.
- Date change behavior (per D-05):
  - Refetch timeline and list for new date (hooks auto-refetch via dep change).
  - If new date has ≥1 recording: `router.push('/app/recordings/' + firstRecording.id)` — URL always reflects playing recording.
  - If new date has 0 recordings: keep URL unchanged, show empty states in player + list, TimelineBar shows all-empty heatmap.

### Timeline Interaction (reuse)

- Click on an hour: `handleSeek(hour)` → find recording containing that hour → `router.push('/app/recordings/' + target.id)`. If no recording at that hour, do nothing (no error toast — heatmap already conveys absence).
- Drag select across hours: `onRangeSelect(start, end)` → same finding logic → navigate to containing recording if found.
- Keyboard: `ArrowLeft`/`ArrowRight` shift selection ±1h (existing behavior), `Enter` seeks to start. Existing `aria-slider` role preserved.

### List Row Click

- Click any row → `router.push('/app/recordings/' + rec.id)`. Current-playing row: `Play` icon prefix in `text-primary` and `bg-accent/40` row background (subtle highlight, no color overreach).
- No separate row action menu in this phase's list (user can download/delete from the `/app/recordings` DataTable).

### Autoplay

Per D-Claude-Discretion: `autoPlay={false}`. User clicks the native HLS play control. Matches existing `recordings-tab.tsx:233`.

### Back Navigation

- Back button: `router.back()` preferred (preserves filter state automatically). Fallback if history is empty: `router.push('/app/recordings')`.
- Browser back button: same native behavior, works because we use `router.push` to enter (not `replace`).

## Accessibility

- Row-click navigation is supplemented by keyboard: each table row is `tabIndex={0}`, `role="button"` or native `button` wrapping content is avoided to keep semantic table; `onKeyDown` handles `Enter` and `Space`.
- `aria-label` on rows: `Open recording from {camera.name} at {startTime}`.
- Date nav buttons have explicit `aria-label` (Prev/Next/Select date).
- Timeline already exposes `role="slider"`, `aria-valuemin/max/now`, `aria-label="24-hour recording timeline"` (unchanged).
- Focus ring: all interactive elements inherit `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` (shadcn default). No custom focus styles.
- Color contrast: accent green `hsl(142 71% 45%)` on white ≥ 4.5:1 for UI elements (AA). Heatmap filled hour vs. empty hour (muted) distinguishable in both light and dark mode.
- Player container keeps `bg-[hsl(0,0%,9%)]` in both themes to preserve letterbox consistency (not a WCAG issue — no text content).
