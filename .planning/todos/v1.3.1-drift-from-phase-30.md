## Code / Config items

### #19 — SRS WebRTC + SRT UDP listeners closed
- **Symptom:** External nmap UDP 8000 (WebRTC) + UDP 10080 (SRT) return `port-unreach` ICMP despite compose port mapping at `deploy/docker-compose.yml:121-122`.
- **Root cause confirmed (2026-04-30):** SRS process is not binding UDP at all. `deploy/srs.conf` has the vhost-level `rtc { enabled on; rtmp_to_rtc on; }` but **NO top-level `rtc_server` block** and **NO `srt_server` block** — without those, SRS skips UDP socket creation entirely. Cloud firewall is innocent.
- **Action (when WebRTC ingest is needed):**
  ```
  # deploy/srs.conf — top level, sibling to http_server/http_api
  rtc_server {
      enabled on;
      listen 8000;
      candidate $CANDIDATE;   # set CANDIDATE env in compose to VM's public IP
  }
  srt_server {
      enabled on;
      listen 10080;
      maxbw 1000000000;
  }
  ```
  Add `CANDIDATE: ${SERVER_PUBLIC_IP}` to the srs service environment block in compose. The candidate must be the public IP browsers will reach (STUN-discoverable).
- **Owner:** Phase 26 (DEPLOY-11) follow-up
- **Required-by:** any future WebRTC ingest enablement (RTMP-only deploy unaffected — DEFER until needed)

## Process / cross-phase items

### v1.4 — Replace static-only verifiers with live-runtime smoke gates
- **Lesson:** All eighteen #1-#18 wiring bugs were Phase 24-29 verifier blind spots. `compose config --quiet`, `bash -n`, `shellcheck`, and even a successful image build do not catch:
  - env-var name mismatches between compose and source (8 of 18 bugs)
  - `host.docker.internal` callback URLs that fail on Linux Docker Engine
  - `prisma generate` postinstall skipped via `--ignore-scripts` → engines missing in runtime image
  - Caddy `acme_ca {$VAR:default}` parsing of empty-string-set
  - NestJS Throttler `100/min` global pool too tight for 7+ cameras polling
  - `--wait` flag misuse on init containers that exit 0
- **Action:** Phase 24-29's verifier scripts (`verify-deploy.sh`, `verify-playback.sh`, etc.) must include live-runtime stages — not just static contract checks. Specifically, every env-var read in api source should have a paired compose round-trip assertion (`exec api printenv ${VAR}` → expected value).
- **Owner:** v1.4 milestone scoping
