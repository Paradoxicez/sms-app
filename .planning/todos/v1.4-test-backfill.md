## What v1.4 Should Do

1. **Inventory:** `grep -rn "it\.todo" apps/api/tests/` produces a current list (count drift may be small — fine).
2. **Cluster by file:** Group todos by test file. Largest cluster is the priority backfill target.
3. **Backfill in waves:** Each wave = one test file's todos converted to real `it(...)` blocks. PRs scoped one file at a time keep review tractable.
4. **Coverage threshold:** Once backfill is underway, add `pnpm test --coverage` with a soft threshold (e.g., 60% lines on api package). Hard-enforce later.

## What v1.4 Should NOT Do

- Convert `describe.skipIf` integration tests to non-skip variants. Those are gated on Redis being available; CI does not currently run with Redis (per Phase 23 D-22 — only Postgres). That gate stays as Phase 23 designed.
- Delete any `it.todo` without replacing it with a real test. The placeholder was a deliberate signal of intent.
