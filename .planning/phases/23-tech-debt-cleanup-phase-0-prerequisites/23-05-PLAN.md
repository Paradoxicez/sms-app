## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public PR -> main branch | Branch protection ensures only green-test commits land |
| GH Actions runner -> Postgres service | Service container is ephemeral, no external exposure |
| GH Actions runner -> public PR fork | PRs from forks run with restricted GITHUB_TOKEN by default |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-15 | Tampering | main branch history | mitigate | Branch protection (Task 5) requires green `test` check; admins can override only with `enforce_admins: false` (current setting). Future stricter setting (`enforce_admins: true`) would close this; defer to Phase 28 alongside tag-protection rules. |
| T-23-16 | Information Disclosure | CI logs | mitigate | Postgres credentials in service env are NOT secrets (ephemeral container). Workflow does NOT echo DATABASE_URL anywhere (no `echo` of env vars). pnpm + prisma commands respect default log levels.
