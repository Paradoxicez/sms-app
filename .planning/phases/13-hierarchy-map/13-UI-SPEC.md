# Phase 13 — UI Design Contract

> Visual and interaction contract for hierarchy tree viewer and map enhancements: resizable split panel with tree + DataTable on projects page, floating tree overlay on map page, drag-drop marker placement, and camera preview popup updates. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova style) |
| Preset | base-nova (neutral base, CSS variables, Tailwind 4.2) |
| Component library | base-ui (render prop pattern, NOT Radix asChild) |
| Icon library | lucide-react |
| Font | Inter (ui-sans-serif fallback) |

Source: Inherited from Phase 8/10/11 UI-SPEC. No changes.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (p-1) | Icon gaps in tree nodes, gap between chevron and node label, gap between status dot and camera name |
| sm | 8px (p-2) | Tree node vertical padding, tree search input padding, breadcrumb item gaps, floating panel internal padding |
| md | 16px (p-4) | Gap between tree panel and DataTable, floating panel margin from map edges, gap between breadcrumb and table, placement mode instruction banner padding |
| lg | 24px (p-6) | Empty state container padding, dialog internal padding, confirm/cancel popup padding |
| xl | 32px (p-8) | Page-level content padding (existing `main` uses `p-4 md:p-8`) |

Exceptions:
- Tree node indentation per level: 20px (not a standard token, but required for visual hierarchy clarity)
- Resizer drag handle width: 8px (matches `sm` token)

## Color

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `hsl(0 0% 100%)` / `var(--background)` | Page background, tree panel background, DataTable background, map popup background |
| Secondary (30%) | `hsl(220 14% 96%)` / `var(--muted)` | Tree node hover state (`bg-muted/50`), selected tree node (`bg-muted`), resizer handle background, floating panel background (`bg-background/95 backdrop-blur`) |
| Accent (10%) | `hsl(142 71% 45%)` / `var(--primary)` | Selected tree node left border highlight, "Create Project" / "Create Site" CTA buttons, active breadcrumb text, focus ring, "Confirm" button in placement mode, tree search focus ring |
| Destructive | `hsl(0 84% 60%)` / `var(--destructive)` | Delete menu item text, delete confirmation button |

Accent reserved for:
- "Create Project" and "Create Site" primary CTA buttons
- Selected tree node left border indicator (3px solid `var(--primary)`)
- "Confirm" button in map placement mode
- Active breadcrumb segment text color
- Focus rings on interactive elements
- Placement mode crosshair cursor accent ring

NOT used for: tree chevrons at rest, tree panel border, resizer handle, floating panel border, breadcrumb separators, tree node icons at rest.

## Tree Viewer Specification (D-01, D-02, D-03, D-04)

### Node Anatomy

```
[Chevron] [Icon] [Label]                    [(count)]
```

| Node Type | Icon | Label | Count Badge | Expand |
|-----------|------|-------|-------------|--------|
| Project | `Folder` (lucide) | Project name, `text-sm font-medium truncate` | "(N sites)" `text-xs text-muted-foreground` | Chevron if has sites |
| Site | `MapPin` (lucide) | Site name, `text-sm font-medium truncate` | "(N cameras)" `text-xs text-muted-foreground` | Chevron if has cameras |
| Camera | Status dot (8px circle) | Camera name, `text-sm truncate` | No count | Not expandable |

| Property | Value |
|----------|-------|
| Node height | 36px (py-2 px-2) |
| Indent per level | 20px (`pl-5` per depth) |
| Chevron | `ChevronRight` (lucide, 16px), rotates 90deg when expanded, `transition-transform duration-150` |
| Selected state | `bg-muted` + `border-l-3 border-primary` left accent |
| Hover state | `bg-muted/50` |
| Click | Single click selects node, updates right panel DataTable |
| Double-click on expandable | Expand/collapse toggle |

### Tree Search (D-03)

| Property | Value |
|----------|-------|
| Position | Top of tree panel, sticky |
| Component | `Input` with `Search` icon (lucide) prefix |
| Placeholder | "Search hierarchy..." |
| Behavior | Filters tree to show matching nodes + parent chain; debounce 200ms |
| Clear | X button appears when input has value, clears filter on click |
| No results | "No matching items" text centered, `text-sm text-muted-foreground` |

### Tree Data Shape

```typescript
interface TreeNode {
  id: string;
  type: 'project' | 'site' | 'camera';
  name: string;
  childCount: number;       // sites for project, cameras for site, 0 for camera
  status?: string;          // only for camera nodes
  children?: TreeNode[];
}
```

## Drag-Drop Marker Placement (D-18, D-19, D-20, D-21)

### Entry Points

| Entry | Trigger | Context |
|-------|---------|---------|
| Map popup | "Set Location" button in CameraPopup | Camera already on map (has coordinates) |
| Tree context | "Set Location" in tree node right-click menu or "..." dropdown | Camera without coordinates (visible in tree but not on map) |

### Placement Mode Flow

```
1. User clicks "Set Location"
2. Instruction banner appears at top of map:
   "Click on the map to set camera location. Press Escape to cancel."
3. Map cursor changes to crosshair
4. User clicks a point on the map
5. Pulsing green marker appears at clicked position
6. Confirm/cancel card appears near marker:
   +-------------------------------+
   | Set location for "Camera A"?  |
   | [Cancel]         [Confirm]    |
   +-------------------------------+
7a. Confirm: PATCH /api/cameras/{id} with lat/lng, toast "Location updated", exit placement mode
7b. Cancel: Remove preview marker, exit placement mode
```

| Property | Value |
|----------|-------|
| Instruction banner | `fixed top-0 left-0 right-0 z-[2000]`, `bg-primary text-primary-foreground py-2 px-4 text-sm text-center` |
| Cursor | `cursor-crosshair` applied to map container via class toggle |
| Preview marker | Pulsing green dot: `animate-pulse`, same style as `connecting` status dot but larger (12px) |
| Confirm card | `bg-card border rounded-lg shadow-lg p-4`, positioned via Leaflet popup anchored to marker |
| Cancel button | `Button variant="outline" size="sm"`, text "Cancel" |
| Confirm button | `Button variant="default" size="sm"`, text "Confirm" |
| Escape key | Cancels placement mode, removes preview marker |
| Other map clicks | Prevented during placement mode (only one marker placement per session) |

### Cameras Without Location (D-21)

| Property | Value |
|----------|-------|
| Tree display | Camera nodes without lat/lng show `MapPinOff` icon (lucide) instead of status dot |
| Tree tooltip | "No location set" tooltip on `MapPinOff` icon |
| Map visibility | Not shown on map (no marker) |
| Discovery | Tree panel surfaces these cameras so user can right-click > "Set Location" |

## Accessibility Contract

Inherited from Phase 8/10/11 plus:

| Requirement | Implementation |
|-------------|----------------|
| Tree keyboard navigation | Arrow keys navigate nodes: Up/Down moves focus, Right expands, Left collapses, Enter selects |
| Tree ARIA | `role="tree"` on container, `role="treeitem"` on each node, `aria-expanded` on expandable nodes, `aria-selected` on selected node |
| Tree search | `aria-label="Search hierarchy"` on input |
| Resizer keyboard | Resizer is focusable (`tabindex="0"`), Left/Right arrow keys adjust width by 20px increments, `role="separator"` with `aria-orientation="vertical"` |
| Floating panel | `aria-label="Hierarchy filter panel"`, toggle button has `aria-expanded` |
| Placement mode | Instruction banner uses `role="alert"` for screen reader announcement |
| Breadcrumb | `nav` with `aria-label="Hierarchy navigation"`, `aria-current="page"` on active segment |
| Map popup buttons | `aria-label` includes camera name: "View stream for {name}", "Set location for {name}" |
