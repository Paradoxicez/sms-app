## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-04 | Tampering | PATCH camera location | mitigate | Backend Zod schema already validates lat/lng as numbers. Add range validation in placement-mode.tsx: clamp lat to [-90, 90] and lng to [-180, 180] before sending PATCH. The backend UpdateCameraSchema should also validate range (existing or add). |
| T-13-05 | Information Disclosure | Tree data on map page | accept | Same RLS + org_id scoping as Plan 01. useHierarchyData only fetches data for the authenticated org. |
| T-13-06 | Denial of Service | Rapid tree filter clicks | accept | Each filter operation is client-side filtering of already-fetched cameras array. No additional API calls per filter change. FitBounds is debounced by React effect dependencies. |
| T-13-07 | Spoofing | ViewStreamSheet camera data | accept | Camera data comes from the local cameras array (fetched from authenticated API). No user-controlled injection point. |
</threat_model>
