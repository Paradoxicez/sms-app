# Phase 1 — UI Design Contract

> Visual and interaction contract for Foundation & Multi-Tenant phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | Green theme (initialize with `npx shadcn@latest init`) |
| Component library | Radix UI (via shadcn) |
| Icon library | Lucide React |
| Font | Inter (system fallback: ui-sans-serif, system-ui, sans-serif) |

**Initialization:** Run `npx shadcn@latest init` during Phase 1 setup. Configure green primary color in `globals.css` CSS variables. Set `cssVariables: true` in components.json.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, form group gaps |
| xl | 32px | Layout gaps between cards, sidebar padding |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Page-level spacing, auth form centering margins |

Exceptions: Touch targets for mobile menu toggle and sidebar collapse button must be minimum 44x44px.

## Copywriting Contract

### Primary CTAs

| Context | Label | Icon |
|---------|-------|------|
| Super admin - Organizations page | "Create Organization" | Plus |
| Super admin - Packages page | "Create Package" | Plus |
| Org admin - Users page | "Invite User" | UserPlus |
| Org admin - Users page (secondary) | "Create User" | UserPlus |
| Auth - Sign in page | "Sign In" | none |
| Package editor - save | "Save Package" | none |
| Settings page - save | "Save Settings" | none |

### Empty States

| Context | Heading | Body |
|---------|---------|------|
| Organizations list | No organizations yet | Create your first organization to get started. |
| Users list | No users in this organization | Invite team members to start collaborating. |
| Packages list | No packages defined | Create a package to set limits for organizations. |

Empty state layout: Centered vertically in content area. Icon (from Lucide, muted color, 48px) above heading. Heading in 20px/600. Body in 14px/400 muted text. Primary CTA button below body with 16px gap.

### Error States

| Context | Copy |
|---------|------|
| Auth - invalid credentials | Invalid email or password. Please try again. |
| Permission denied | You don't have permission to perform this action. Contact your organization admin. |
| Network error | Could not connect to the server. Check your connection and try again. |
| Form validation | Inline under each field: "{Field} is required." / "{Field} must be at least {n} characters." |

Error text: 12px, destructive color, displayed below the relevant input with 4px gap.

### Destructive Confirmations

| Action | Dialog Title | Dialog Body | Confirm Button | Extra |
|--------|-------------|-------------|----------------|-------|
| Deactivate Organization | Deactivate Organization | This will suspend all users and streams in this organization. Type the organization name to confirm. | "Deactivate" (destructive variant) | Text input confirmation required |
| Remove User | Remove User | This will revoke access for this user. They will need a new invitation to rejoin. | "Remove" (destructive variant) | none |
| Delete Package | Delete Package | Organizations using this package will need to be reassigned. Continue? | "Delete" (destructive variant) | none |

All destructive dialogs use AlertDialog. Confirm button uses destructive variant (red background). Cancel button uses outline variant. Cancel is always on the left, destructive confirm on the right.

## Interaction Patterns

### Sign-in Flow
1. User visits `/auth/sign-in`
2. Centered card with email + password inputs
3. On submit: button shows loading spinner (replace text with Loader2 icon, spinning)
4. On error: shake animation on form, error message appears below form
5. On success: redirect to `/admin` (super admin) or `/dashboard` (org user)

### Table Patterns
- Default sort: created date descending
- Row hover: muted background highlight
- Actions column: DropdownMenu with "..." trigger (MoreHorizontal icon)
- Pagination: bottom-right, "Previous" / "Next" buttons with page count text
- Loading: Skeleton rows (3 rows of skeleton blocks matching column widths)

### Sidebar
- Fixed 240px width on desktop
- Collapsible on mobile (sheet/drawer overlay)
- Active item: accent background (10% opacity) + left 3px accent border + accent text
- Inactive item: muted text, hover shows muted background

### Toast Notifications
- Position: bottom-right
- Duration: 5 seconds (auto-dismiss)
- Success: default style with check icon
- Error: destructive style with alert icon
- Max visible: 3 stacked
