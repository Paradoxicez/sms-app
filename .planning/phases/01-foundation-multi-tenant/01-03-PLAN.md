  <behavior>
    Package CRUD:
    - POST /api/admin/packages with { name, maxCameras, maxViewers, maxBandwidthMbps, maxStorageGb, features } returns 201 with created package
    - GET /api/admin/packages returns array of all packages
    - GET /api/admin/packages/:id returns single package
    - PATCH /api/admin/packages/:id updates package fields
    - DELETE /api/admin/packages/:id soft-deletes (sets isActive=false)
    - Validation: name required (1-100 chars), maxCameras >= 1, maxViewers >= 1, maxBandwidthMbps >= 1, maxStorageGb >= 1
    - features defaults to {} if not provided
    - Non-super-admin gets 403

    Feature toggles:
    - features field accepts any JSON object like { recordings: true, webhooks: true, map: false }
    - GET /api/admin/packages/:id returns features as parsed JSON
    - PATCH can update individual features without replacing entire object

    Organization management:
    - POST /api/admin/organizations with { name, slug, packageId? } returns 201
    - GET /api/admin/organizations returns array with package info included
    - PATCH /api/admin/organizations/:id updates org fields including packageId
    - PATCH /api/admin/organizations/:id/deactivate sets isActive=false
    - Slug must be unique, lowercase, alphanumeric + hyphens, 3-50 chars
    - Cannot deactivate System org
  </behavior>

    User invitation (per D-10, D-12):
    - POST /api/organizations/:orgId/invitations with { email, role } sends invitation via Better Auth
    - Invitation stored with status "pending", expiresAt 7 days from now
    - Only org admin can invite
    - Role must be one of: admin, operator, developer, viewer

    Direct user creation (per D-10):
    - POST /api/organizations/:orgId/users with { email, name, password, role } creates user and adds to org
    - Only org admin can create users directly
    - Password min 8 chars, max 128 chars

    User listing:
    - GET /api/organizations/:orgId/users returns members with user details and roles
    - Only org admin can list users

    User role update:
    - PATCH /api/organizations/:orgId/users/:userId with { role } updates member role
    - Only org admin can change roles

    User removal:
    - DELETE /api/organizations/:orgId/users/:userId removes member from org
    - Cannot remove last admin
    - Only org admin can remove users
  </behavior>

    2. Create UsersService (`apps/api/src/users/users.service.ts`):
    - Inject PrismaService
    - inviteUser(orgId, dto): Use Better Auth organization API or create invitation record directly
      - prisma.invitation.create({ data: { id: crypto.randomUUID(), organizationId: orgId, email: dto.email, role: dto.role, status: "pending", expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000), inviterId: currentUserId } })
    - createUser(orgId, dto):
      - Create user via prisma.user.create({ data: { id: crypto.randomUUID(), name: dto.name, email: dto.email, emailVerified: true } })
      - Create account with hashed password (use Better Auth's internal API or bcrypt)
      - Add as member: prisma.member.create({ data: { id: crypto.randomUUID(), organizationId: orgId, userId: user.id, role: dto.role } })
    - listMembers(orgId): prisma.member.findMany({ where: { organizationId: orgId }, include: { user: { select: { id: true, name: true, email: true, role: true, createdAt: true } } } })
    - updateRole(orgId, userId, role): prisma.member.updateMany({ where: { organizationId: orgId, userId }, data: { role } })
    - removeMember(orgId, userId):
      - Count admins: prisma.member.count({ where: { organizationId: orgId, role: "admin" } })
      - If count <= 1 and target user is admin, throw error "Cannot remove the last admin"
      - Otherwise: prisma.member.deleteMany({ where: { organizationId: orgId, userId } })
