<!-- API endpoints from Plan 03 that the frontend calls -->
Auth endpoints (Better Auth):
- POST /api/auth/sign-in/email { email, password } -> { user, session }
- GET /api/auth/session -> { user, session } (cookie-based)

Admin endpoints:
- GET /api/admin/packages -> Package[]
- POST /api/admin/packages { name, maxCameras, maxViewers, maxBandwidthMbps, maxStorageGb, features }
- GET /api/admin/organizations -> Organization[] (with package)
- POST /api/admin/organizations { name, slug, packageId? }
- PATCH /api/admin/organizations/:id/deactivate

User endpoints:
- GET /api/organizations/:orgId/users -> Member[] (with user)
- POST /api/organizations/:orgId/invitations { email, role }
- POST /api/organizations/:orgId/users { email, name, password, role }
</interfaces>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to API | Auth client sends credentials over HTTP (dev) / HTTPS (prod) |
| Session cookie | httpOnly cookie prevents XSS access |
| Admin route protection | Client-side redirect + server-side guard double protection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12 | Spoofing | sign-in page | mitigate | Better Auth handles credential validation with timing-safe comparison |
| T-01-13 | Information Disclosure | auth-client.ts | mitigate | baseURL from env var, no secrets in client bundle |
| T-01-14 | Elevation of Privilege | admin/layout.tsx | mitigate | Client-side role check redirects non-admins; server-side SuperAdminGuard enforces on API |
| T-01-15 | Tampering | create-org-dialog.tsx | mitigate | Client-side Zod validation + server-side validation double layer |
</threat_model>
