# Phase 26 Plan 02: Production Env Template + Secrets Generator Summary

**One-liner:** Operator-facing secrets surface for v1.3 GA — `deploy/.env.production.example` documents every required variable in 4 D-25 sections, and `deploy/scripts/init-secrets.sh` is an idempotent bash generator that fills `change-me-*` placeholders with `openssl rand -base64 32` values and chmods the resulting `deploy/.env` to 600 (Pitfall 8 mitigation).

## What Shipped

### Variable surface (deploy/.env.production.example)

The template establishes the canonical variable contract for Phase 26 Plan 03 (compose), Phase 27 (Caddy), and Phase 29 (bootstrap.sh). Every variable is grouped per CONTEXT D-25:

**Section 1 — Required (no default):**
- `DOMAIN` (operator fills) — Caddy reads in Phase 27
- `DB_PASSWORD` (placeholder) — generated by init-secrets.sh
- `NEXTAUTH_SECRET` (placeholder) — web Better Auth client secret
- `BETTER_AUTH_SECRET` (placeholder) — api Better Auth server secret (must differ from NEXTAUTH_SECRET)
- `MINIO_ROOT_PASSWORD` (placeholder)
- `JWT_PLAYBACK_SECRET` (placeholder) — playback session signing
- `ADMIN_EMAIL` (operator fills)
- `ADMIN_PASSWORD` (placeholder) — first super-admin (Phase 29 `bin/sms create-admin`)

**Section 2 — Image refs:**
- `GHCR_ORG` (operator fills)
- `IMAGE_TAG=latest` (default; operator pins semver in prod)

**Section 3 — Defaults (override-only):**
- `POSTGRES_USER=sms`
- `POSTGRES_DB=sms_platform`
- `MINIO_ROOT_USER=minioadmin`
- `REDIS_PASSWORD=` (empty = no auth; safe on internal:true network)

**Section 4 — Computed (commented out):**
- `# DATABASE_URL=postgresql://${POSTGRES_USER}:${DB_PASSWORD}@postgres:5432/${POSTGRES_DB}?schema=public`
- `# REDIS_URL=redis://redis:6379`
- `# MINIO_ENDPOINT=minio`, `MINIO_PORT=9000`, `MINIO_USE_SSL=false`

Plan 03 (compose) constructs DATABASE_URL / REDIS_URL inline from the components — the commented variants are documented so an operator with an external Postgres can override.

### Secrets generator (deploy/scripts/init-secrets.sh)

86-line bash script. Every line carries weight:

| Section | Purpose |
|---------|---------|
| Shebang `#!/usr/bin/env bash` | Bash-explicit (D-14 portability) |
| `set -euo pipefail` | Fail-fast (matches Phase 24 `scripts/dev-smoke.sh`) |
| `SCRIPT_DIR=...` resolution | Works whether invoked from repo root or `deploy/scripts/` |
| `SECRET_VARS=(...)` array | DB_PASSWORD, NEXTAUTH_SECRET, BETTER_AUTH_SECRET, MINIO_ROOT_PASSWORD, JWT_PLAYBACK_SECRET, ADMIN_PASSWORD |
| `sed_inplace()` shim | BSD `sed -i ''` (macOS) vs GNU `sed -i` (Linux) |
| Idempotency guard | `[[ -z VALUE ]] || [[ VALUE == change-me-* ]] || [[ VALUE == "<generated>" ]]` |
| `openssl rand -base64 32 \| tr -d '=' \| tr -d '\n'` | 256-bit entropy, 43 base64 chars, no padding |
| `chmod 600 "${ENV_FILE}"` | T-26-05 mitigation (Pitfall 8 GA blocker) |
| Summary echo | `Generated: N / Already set: M / Placeholders hit: K` |

## Idempotency Contract (downstream consumers must respect)

`init-secrets.sh` mutates only values matching ALL of:
- `^${VAR}=` line exists in `deploy/.env`
- Current value is empty OR starts with `change-me-` OR is the literal `<generated>`

Real values (anything else) are skipped with `= Skipped ${VAR} (already set)`. **Operators running the script twice never lose credentials.** This is the safety net Phase 29's `deploy/scripts/bootstrap.sh` will rely on when wrapping init-secrets.sh in a one-shot installer.

## Placeholder Convention (Plan 03 compose must respect)

The template ships every secret with the `change-me-${kebab-name}` prefix (e.g. `change-me-db-password`). Plan 03's compose YAML MUST NOT:
- Hardcode any of these placeholders as defaults (they are operator-replaceable, not fallbacks)
- Pre-validate that values are non-placeholder at compose-up time (Phase 30 smoke test catches that scenario)
- Add new secrets without also adding them to `SECRET_VARS` in init-secrets.sh AND a `change-me-*` placeholder in the template

## Threat Mitigations Implemented

| Threat | Disposition | Implementation |
|--------|-------------|----------------|
| T-26-05 (info disclosure: world-readable .env) | mitigate | `chmod 600 "${ENV_FILE}"` runs unconditionally before the summary echo. End-to-end smoke verifies post-condition. |
| T-26-06 (info disclosure: .env in git) | accept (Phase 24 covers) | Verified `git check-ignore -v deploy/.env` matches `.gitignore:5:.env` pattern; no new gitignore entry needed |
| T-26-07 (tampering: re-run rotates secrets) | mitigate | Three-condition idempotency guard (empty / change-me-* / <generated>) skips real values |
| T-26-08 (repudiation: which run generated which) | accept | Summary log `Generated: N / Already set: M / Placeholders hit: K` is minimum audit (DEPLOY-30 backup rotation handles full audit in v1.3.x) |
| T-26-09 (placeholders ship to prod) | partial mitigate | `change-me-` prefix is highly visible in any logs; full pre-flight check deferred to Phase 30 smoke / Phase 29 bootstrap.sh |
