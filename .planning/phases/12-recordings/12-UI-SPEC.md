# Phase 12 — UI Design Contract

> Visual and interaction contract for recordings page: DataTable with cross-camera filters, faceted filter bar, DateRangePicker, bulk delete, and download via presigned URLs. Generated by gsd-ui-researcher.

---

## Color

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `hsl(0 0% 100%)` / `var(--background)` | Page background, table background |
| Secondary (30%) | `hsl(220 14% 96%)` / `var(--muted)` | Table row hover (`bg-muted/50`), selected row highlight (`bg-muted`), toolbar area |
| Accent (10%) | `hsl(142 71% 45%)` / `var(--primary)` | Checkbox checked fill, active sort arrow, focus ring, active pagination page, "Delete Selected" count badge in toolbar |
| Destructive | `hsl(0 84% 60%)` / `var(--destructive)` | Delete menu item text, "Delete Selected" toolbar button, bulk delete confirmation button |

Accent reserved for:
- Checkbox checked fill color
- Active column sort direction arrow
- Focus ring on interactive elements
- Active pagination page number
- Camera name link hover color (`text-primary`)

NOT used for: table borders, filter chips at rest, row action "..." button, status badge backgrounds.

## Recordings Table Specification

### Columns (D-01)

| Column | Width | Content | Sortable |
|--------|-------|---------|----------|
| Checkbox | 40px | `Checkbox` component for row selection | No |
| Camera Name | flex-1 (min 160px) | Link text (`text-primary hover:underline`), navigates to camera page | Yes |
| Project | 120px | Plain text | Yes |
| Site | 120px | Plain text | Yes |
| Date | 100px | Formatted date (e.g., "Apr 17, 2026") | Yes |
| Time Range | 140px | Start - End (e.g., "14:30 - 15:45") or "14:30 - ..." if ongoing | No |
| Duration | 80px | Formatted (e.g., "1h 15m") via `formatDuration` | Yes |
| Size | 80px | Formatted (e.g., "245 MB") via `formatSize` | Yes |
| Status | 100px | `RecordingStatusBadge` component | Yes |
| Actions | 48px | "..." menu | No |

### Default Sort (D-02)

`startedAt` descending (newest recordings first).

### Pagination (D-03)

Server-side pagination. Backend returns `{ data, total, page, pageSize }`.

| Property | Value |
|----------|-------|
| Default page size | 10 |
| Page size options | 10, 25, 50 |
| Pagination controls | DataTablePagination with page numbers + prev/next |
| URL state | `page` and `pageSize` stored in URL query params |

### Toolbar Layout (D-04, D-05, D-06)

```
[Search (240px)]  [Camera Filter]  [Project Filter]  [Site Filter]  [DateRangePicker]  [Status Filter]  ----stretches----  [Delete Selected (N)]
```

- Search: `placeholder="Search recordings..."` -- queries camera name and project name
- Camera filter: `DataTableFacetedFilter` with camera names from API
- Project filter: `DataTableFacetedFilter` with project names from API
- Site filter: `DataTableFacetedFilter` with site names from API
- DateRangePicker: Phase 8 `DateRangePicker` component
- Status filter: `DataTableFacetedFilter` with options: Complete, Recording, Processing, Error
- "Delete Selected (N)": appears only when 1+ rows selected (D-09)

### Filter State (D-07)

All filter values stored in URL query params for shareable links and back-button preservation. Pattern matches Phase 8 DataTable toolbar.

| Param | Type | Example |
|-------|------|---------|
| `search` | string | `?search=lobby` |
| `camera` | comma-separated IDs | `?camera=id1,id2` |
| `project` | comma-separated IDs | `?project=id1` |
| `site` | comma-separated IDs | `?site=id1` |
| `from` | ISO date | `?from=2026-04-01` |
| `to` | ISO date | `?to=2026-04-17` |
| `status` | comma-separated values | `?status=complete,error` |
| `page` | number | `?page=2` |
| `pageSize` | number | `?pageSize=25` |

## Quick Actions Menu (D-08)

| Action | Icon | Label | Type |
|--------|------|-------|------|
| Download | `Download` | "Download" | Non-destructive |
| Separator | --- | --- | --- |
| Delete | `Trash2` | "Delete" | Destructive |

- Download: calls `GET /api/recordings/:id/download`, receives `{ url }`, triggers `window.open(url)` for browser download (D-10)
- Delete: opens AlertDialog confirmation before DELETE request
- Destructive items: `text-destructive` class on menu item
- Separator before destructive actions

## Bulk Delete Specification (D-09)

### Toolbar Button

| Property | Value |
|----------|-------|
| Visibility | Only when 1+ checkboxes selected |
| Component | `Button variant="destructive" size="sm"` |
| Icon | `Trash2` (16px, left of label) |
| Label | "Delete Selected ({N})" where N = selected count |
| Position | Right end of toolbar |

### Confirmation Dialog

| Element | Value |
|---------|-------|
| Component | `AlertDialog` |
| Title | "Delete {N} Recording{s}" |
| Description | "This will permanently delete {N} selected recording{s} and their files. This action cannot be undone." |
| Cancel | `variant="outline"`, text "Cancel" |
| Confirm | `variant="destructive"`, text "Delete {N} Recording{s}" |
| Loading state | Confirm button shows `Loader2 animate-spin` + "Deleting..." while request in flight, disabled |

### Partial Failure Handling

- If some deletions succeed and some fail: show toast "Deleted {success} of {total} recordings. {failed} failed." with `variant="destructive"`
- Refresh table data after bulk operation completes
- Clear checkbox selection after operation completes (regardless of partial failure)

## Interaction Contracts

### Destructive Actions

| Action | Confirmation Copy |
|--------|-------------------|
| Delete single recording | "This will permanently delete this recording and its files. This action cannot be undone." |
| Bulk delete | "This will permanently delete {N} selected recording{s} and their files. This action cannot be undone." |

### Non-Destructive Actions

| Action | Behavior |
|--------|----------|
| Download | Fetches presigned URL from API, opens in new tab for browser download. Toast: "Download started" |
| Camera name click | Navigates to `/app/cameras/{cameraId}?tab=recordings` |
| Filter change | Updates URL query params, triggers server-side data refetch |
| Page change | Updates `page` query param, triggers server-side data refetch |

### Loading States

| Context | Loading State |
|---------|---------------|
| Initial page load | 5 `Skeleton` rows matching column widths and heights |
| Filter change / page change | Table body replaced with 5 `Skeleton` rows (header + toolbar remain visible) |
| Download in progress | Row action menu item shows `Loader2 animate-spin` replacing `Download` icon |
| Bulk delete in progress | Confirm button shows `Loader2 animate-spin` + "Deleting...", disabled state |

### Empty States

| Context | Heading | Body | Action |
|---------|---------|------|--------|
| No recordings exist (no filters active) | "No recordings yet" | "Recordings will appear here when cameras start recording. Go to Cameras to start a recording." | "Go to Cameras" button (navigates to `/app/cameras`) |
| Filters return no results | "No recordings found" | "Try adjusting your search or filter criteria." | "Clear filters" button (resets all URL params) |
| Error loading | "Failed to load recordings" | "Something went wrong while loading recordings. Check your connection and try again." | "Retry" button (refetches data) |

## Layout Specifications

### Page Layout

```
[Page Heading: "Recordings"]
                          16px gap
[DataTable]
  [Toolbar: Search | Camera Filter | Project Filter | Site Filter | DateRangePicker | Status Filter | --- | Delete Selected (N)]
                          16px gap
  [Table rows]
                          16px gap
  [Pagination bar]
```

- Page heading: `text-xl font-semibold` left-aligned
- No primary CTA button in heading row (recordings are created from camera actions, not from this page)
- Heading row: `flex items-center justify-between mb-4`
- Table-only layout (D-11) -- no inline player, no calendar, no timeline

## Accessibility Contract

Inherited from Phase 8/10/11 plus:

| Requirement | Implementation |
|-------------|----------------|
| Row selection | Checkbox uses `aria-label="Select recording from {camera name} at {time}"` |
| Select all | Header checkbox uses `aria-label="Select all recordings"` |
| Bulk delete button | `aria-label="Delete {N} selected recordings"` |
| Camera name link | Standard anchor semantics, no additional ARIA needed |
| Download action | Menu item `aria-label="Download recording from {camera name} at {time}"` |
| Status badge | Badge text serves as accessible label (no additional ARIA needed) |
| Filter reset | "Clear filters" button has `aria-label="Clear all active filters"` |
| Pagination | DataTablePagination handles ARIA internally (Phase 8) |
