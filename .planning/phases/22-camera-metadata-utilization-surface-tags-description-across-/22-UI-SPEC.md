## Color

Phase 22 uses the existing green-theme tokens declared at `globals.css:50-82` (light)
and `:85-116` (dark). No new color variables, no per-tag color (deferred per
CONTEXT.md `<deferred>`).

| Role | Token | HSL value (light) | Usage in Phase 22 |
|------|-------|---|---|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` (white) | Sheet body, popover surface, map popup card background |
| Secondary (30%) | `--secondary` / `--muted` | `hsl(142 30% 97%)` / `hsl(220 14% 96%)` | Bulk popover surface, chip combobox input fill, Notes block subtle `bg-muted/40` if used |
| Accent (10%) | `--primary` | `hsl(142 71% 45%)` (green) | Reserved-for: see explicit list below |
| Destructive | `--destructive` | `hsl(0 84% 60%)` | Reserved-for: see explicit list below |

**Tag badge color (D-15 вҖ” single uniform token, no per-tag):**
- Class: `bg-neutral-100 text-neutral-700` (light), `dark:bg-neutral-800 dark:text-neutral-300` (dark).
- Rationale: matches the `Auto` mode of `STREAM_PROFILE_MODE_BADGE` at `stream-profile-mode.ts:21` вҖ” the neutral family is the existing "no semantic meaning" token. Tags are descriptive, not status-bearing, so neutral is correct.
- Applied via `<Badge variant="outline" className="bg-neutral-100 text-neutral-700 dark:bg-neutral-800 dark:text-neutral-300">`.
- The `+N` overflow chip uses the SAME tokens вҖ” visually equivalent to a tag, so the column reads as "more of the same".

**Accent (`--primary` green) is reserved for ONLY these elements in Phase 22:**
1. Filter "active" indicator on the Tags filter button (the existing `Badge variant="secondary"` count-bubble in `data-table-faceted-filter.tsx:64-69` вҖ” already in `--secondary`, NOT primary; we make NO change here).
2. Primary "Apply" / "Save" buttons inside the bulk Add/Remove tag popover (`<Button>` default variant) вҖ” only ONE primary button per popover (D-13 no-confirm bulk вҶ’ the popover Apply IS the action, no second confirm step).
3. "Show more" disclosure link inside map popup description block вҖ” `text-primary hover:underline`, matching the existing "details" link at `camera-popup.tsx:258`.

**Accent is NOT used for:**
- Tag badges (neutral, see above).
- "Add tag" / "Remove tag" toolbar buttons (`variant="outline"` вҖ” matches Phase 20 toolbar pattern).
- Description tooltip (no border, no fill вҖ” uses shadcn default `bg-popover text-popover-foreground`).
- Filter chip Г— buttons (`text-muted-foreground hover:text-foreground`).

**Destructive (`--destructive` red) is reserved for ONLY these elements in Phase 22:**
- None. Phase 22 has zero destructive actions:
  - Bulk Remove tag is NON-destructive (D-11/D-13 вҖ” easily reversible by Add tag).
  - No camera/tag deletion in this phase.
- The `--destructive` token MUST NOT appear in any Phase 22 component.

**60/30/10 audit:**
- Background (60%): Sheet body, page background, popover surfaces вҖ” all `--background` / `bg-background`.
- Secondary (30%): Bulk popover content, chip combobox input fill, Notes block separator border (`border-border`), tag badge fill (`bg-neutral-100`).
- Accent (10%): The 3 reserved-for elements above. Each is a single instance per surface вҖ” no row-level accent flooding.

Behavioral contract:
- **Input box** вҖ” single `<input>` element, chips rendered as `<Badge variant="secondary">` siblings inside the same `flex flex-wrap` container (border, rounded-md, p-1, h-min). Chips appear BEFORE the input cursor.
- **Chip Г— button** вҖ” `<button>` with `<X size-3>` inside the badge; `aria-label="Remove tag {name}"`.
- **Enter/comma** вҖ” commits the current input value as a chip; resets input. Trims whitespace, rejects empty/duplicate (case-insensitive per D-04).
- **Backspace on empty input** вҖ” removes the last chip (D-08).
- **Autocomplete dropdown** вҖ” opens on focus, populated from `GET /cameras/tags/distinct` (cached on form open per D-09). Filters by case-insensitive substring of input value.
- **Selecting a suggestion** вҖ” same as Enter; commits chip, resets input.
- **`+ Add "{query}"` row** вҖ” only renders when query is non-empty AND no exact case-insensitive match exists in suggestions; clicking commits as a freetext chip.
- **No-API fallback** вҖ” if distinct-tags fetch fails, suggestions list shows nothing; freetext-add still works (typing + Enter). One toast.error fires (per Copywriting вҶ’ "Distinct tags fetch failed").

Bulk Add popover variant: same component, but `multi={false}` вҖ” only ONE chip allowed
(per D-11 single-tag-per-bulk-action). Bulk Remove popover variant: same component
in "selection-only" mode вҖ” no freetext add, suggestions are ONLY tags currently on
вүҘ1 selected camera (computed client-side per D-11).

## Negative Assertions (what this phase MUST NOT do)

These are guard-rails for the executor and gsd-ui-auditor. Each one was an
explicit decision in CONTEXT.md.

1. **No new color tokens.** All tag/description visual styling uses existing CSS variables and Tailwind utilities. No `--tag-color-*`, no per-tag color hash function.
2. **No `--destructive` token usage** anywhere in Phase 22 files (Remove tag is non-destructive per D-13).
3. **No confirmation `<AlertDialog>`** for bulk Add or Remove (D-13).
4. **No URL persistence** for tags filter or map filter (deferred per CONTEXT.md `<deferred>`).

6. **No Bulk Replace mode.** Only Add and Remove ship (D-11).

8. **No per-tag color, no tag-management UI, no Tag entity.** All deferred.
9. **No `description` in webhook payloads.** Only `tags` is added (D-22). UI-side: no developer-portal docs page edit references `description` in webhook event schema.
10. **Tag normalization NEVER lowercases the display string.** The chip combobox shows the original casing the user typed (D-03/D-04). Lowercase is comparison-only.
11. **No new font weights or sizes** beyond the 3 declared in Typography table.
12. **No emoji** in any UI string (per project CLAUDE.md communication preferences).
