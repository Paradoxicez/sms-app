```bash
#!/usr/bin/env bash
# ============================================================================
# deploy/scripts/backup.sh — Phase 29 (DEPLOY-20)
#
# Offline atomic backup. Produces one tar.gz containing:
#   - postgres.dump        (pg_dump -Fc)
#   - minio/avatars/       (mc mirror)
#   - minio/recordings/
#   - minio/snapshots/
#   - caddy_data.tar.gz    (volume tar; ACME state + TLS cert)
#
# api + web are stopped for the duration; postgres + minio + caddy keep
# running. Downtime: ~30-90s depending on archive size.
#
# Usage:
#   bash deploy/scripts/backup.sh
#   BACKUP_DIR=/mnt/backups bash deploy/scripts/backup.sh
#
# Output: ${BACKUP_DIR:-./backups}/sms-backup-<UTC-ts>.tar.gz
#
# Exclusions (D-19): .env (secrets — operator stores separately),
# redis_data (sessions/jobs — replay on api boot), hls_data (live segments).
# ============================================================================

set -euo pipefail
IFS=$'\n\t'

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
DEPLOY_DIR="$(cd "${SCRIPT_DIR}/.." && pwd)"
ENV_FILE="${DEPLOY_DIR}/.env"
COMPOSE_FILE="${DEPLOY_DIR}/docker-compose.yml"
COMPOSE_PROJECT="sms-platform"   # mirrors `name:` in docker-compose.yml

# --- TTY-aware color (D-29) ---
if [[ -t 1 ]] && [[ "$(tput colors 2>/dev/null || echo 0)" -ge 8 ]]; then
  GREEN=$'\033[32m'; RED=$'\033[31m'; YELLOW=$'\033[33m'; BOLD=$'\033[1m'; RESET=$'\033[0m'
else
  GREEN=''; RED=''; YELLOW=''; BOLD=''; RESET=''
fi
log()  { printf '%s[backup]%s %s\n' "${BOLD}" "${RESET}" "$*"; }
ok()   { printf '%s[backup] ✓%s %s\n' "${GREEN}" "${RESET}" "$*"; }
warn() { printf '%s[backup] ⚠%s %s\n' "${YELLOW}" "${RESET}" "$*" >&2; }
die()  { printf '%s[backup] ✗%s %s\n' "${RED}" "${RESET}" "$*" >&2; exit 1; }

[[ -f "${ENV_FILE}" ]]    || die "${ENV_FILE} missing. Run bootstrap.sh first."
[[ -f "${COMPOSE_FILE}" ]] || die "${COMPOSE_FILE} missing."

# shellcheck disable=SC1090
set -a; source "${ENV_FILE}"; set +a

DC="docker compose -f ${COMPOSE_FILE} --env-file ${ENV_FILE}"

# --- D-17: filename + BACKUP_DIR override ---
# I4 (Phase 29 revision): mkdir BEFORE realpath-style normalisation. The
# previous form ran `cd "$(dirname "${ARCHIVE_DIR}")"` first, which fails
# under `set -e` if the parent directory does not yet exist (and we crash
# before the `mkdir -p` ever runs). Mkdir-first guarantees the path exists,
# then `cd` + `pwd` normalises it.
TS=$(date -u +%Y-%m-%dT%H%MZ)
ARCHIVE_DIR="${BACKUP_DIR:-${DEPLOY_DIR}/../backups}"
mkdir -p "${ARCHIVE_DIR}"
ARCHIVE_DIR="$(cd "${ARCHIVE_DIR}" && pwd)"
ARCHIVE="${ARCHIVE_DIR}/sms-backup-${TS}.tar.gz"

# --- Working dir + cleanup trap (always restart api+web on exit) ---
TMP=$(mktemp -d)
RESTARTED=0
cleanup() {
  local rc=$?
  if [[ "${RESTARTED}" -eq 0 ]]; then
    log "Restarting api + web..."
    ${DC} start api web 2>/dev/null || warn "api/web restart failed — inspect: ${DC} ps"
    RESTARTED=1
  fi
  rm -rf "${TMP}"
  exit "${rc}"
}
trap cleanup EXIT

log "Backup target: ${ARCHIVE}"
log "Stopping api + web for atomic snapshot..."
${DC} stop api web
ok "api + web stopped"

# --- Step 1: D-20 pg_dump -Fc (custom format) ---
log "Step 1/3: pg_dump (custom format)..."
${DC} exec -T postgres pg_dump \
  -U "${POSTGRES_USER:-sms}" \
  -d "${POSTGRES_DB:-sms_platform}" \
  -Fc \
  > "${TMP}/postgres.dump"
PG_BYTES=$(wc -c < "${TMP}/postgres.dump" | tr -d ' ')
[[ "${PG_BYTES}" -gt 0 ]] || die "pg_dump produced empty file."
ok "postgres.dump (${PG_BYTES} bytes)"

# --- Step 2: D-19 mc mirror buckets (avatars + recordings + snapshots) ---
log "Step 2/3: mc mirror MinIO buckets..."
mkdir -p "${TMP}/minio"
for BUCKET in avatars recordings snapshots; do
  log "  mirroring ${BUCKET}..."
  # mc is preconfigured with `local` alias inside the minio image entrypoint.
  ${DC} exec -T minio sh -c "mkdir -p /tmp/backup/${BUCKET} && mc mirror --quiet local/${BUCKET} /tmp/backup/${BUCKET}" \
    || warn "  ${BUCKET} mirror returned non-zero (may be empty bucket — continuing)"
done
# Copy the in-container backup dir to the host TMP. I5 (Phase 29 revision):
# use `docker cp` directly (resolved via `${DC} ps -q minio` to get the
# container id). `${DC} cp` has had directory-tree bugs across compose
# v2.10..v2.20; the raw docker CLI is stable.
docker cp "$(${DC} ps -q minio):/tmp/backup/." "${TMP}/minio/"
${DC} exec -T minio rm -rf /tmp/backup
ok "MinIO buckets mirrored"

# --- Step 3: D-19 caddy_data volume tar ---
log "Step 3/3: caddy_data volume tar..."
docker run --rm \
  -v "${COMPOSE_PROJECT}_caddy_data:/data:ro" \
  -v "${TMP}:/backup" \
  alpine \
  tar -C /data -czf /backup/caddy_data.tar.gz .
[[ -s "${TMP}/caddy_data.tar.gz" ]] || die "caddy_data tar produced empty file."
ok "caddy_data.tar.gz ($(wc -c < "${TMP}/caddy_data.tar.gz" | tr -d ' ') bytes)"

# --- Bundle into single archive ---
log "Bundling archive..."
tar -C "${TMP}" -czf "${ARCHIVE}" postgres.dump minio caddy_data.tar.gz
chmod 600 "${ARCHIVE}"   # archive contains hashed creds + cert keys
ARCHIVE_BYTES=$(wc -c < "${ARCHIVE}" | tr -d ' ')
ok "Archive: ${ARCHIVE} (${ARCHIVE_BYTES} bytes)"

# --- Restart api + web (also handled by trap, but explicit on success path) ---
log "Restarting api + web..."
${DC} start api web
RESTARTED=1
ok "Stack live again"

echo
log "Restore command:"
log "  bash ${SCRIPT_DIR}/restore.sh ${ARCHIVE}"
log "Offsite copy hint (operator's responsibility):"
log "  rclone copy ${ARCHIVE} <remote>:sms-backups/"
```

**Critical correctness notes (do NOT deviate):**

1. **EXIT trap MUST restart api + web** — without the trap, a `die` between `stop api web` and the success path leaves the platform offline. The `RESTARTED` flag prevents double-restart on the success path. The trap also cleans up TMP regardless of success/failure.

2. **`COMPOSE_PROJECT="sms-platform"`** is the literal value of the `name:` directive on docker-compose.yml line 29. Docker prefixes named volumes with this string from the host POV — i.e. `caddy_data` in the compose file becomes `sms-platform_caddy_data` when referenced via `docker run -v`. Without this exact prefix the alpine container mounts a non-existent volume, gets an empty `/data`, and the tar produces an empty (but non-zero-byte) archive that silently fails restore. The hardcode is intentional and matches Phase 26 D-08 / docker-compose.yml line 29.

3. **`pg_dump -Fc`** — D-20. Custom format = smaller archive + parallel restore via `pg_restore -j N`. Cannot be `pg_dump -Fp` (plain SQL — bigger, no parallel restore) or `pg_dumpall` (would include cluster-wide objects we don't need).

4. **`compose exec -T`** — `-T` disables TTY allocation. Required when stdout is captured to a host file (`> "${TMP}/postgres.dump"`); without `-T` compose attaches a pseudo-TTY and pg_dump's binary output gets corrupted by terminal escape sequences.

5. **MinIO mc backup is two-stage** — exec into minio container to run mc mirror writing to /tmp/backup INSIDE the container, then `docker cp` the in-container path back to the host TMP. We can't pipe mc mirror's output directly because mc mirror writes a directory tree, not a stream. `docker cp` (resolved via `${DC} ps -q minio` for the container id) handles directory trees deterministically; we deliberately bypass `${DC} cp` because that wrapper has had directory-tree bugs across compose v2.10..v2.20.

6. **Empty-bucket tolerance** — `mc mirror local/${BUCKET}` returns non-zero on a bucket that exists but is empty (depending on mc version). We `|| warn` instead of `die` because empty buckets are normal on a fresh deploy. The bundle still includes the bucket subdirectory (even if empty) so restore.sh sees the same structure.

7. **`chmod 600` on the final archive** — the bundle contains hashed credentials, ACME private keys, and pg_dump output (which has scrypt-hashed passwords from the Account table). 600 prevents other host users from reading.

8. **chmod +x in source tree:**
```bash
chmod +x deploy/scripts/backup.sh
git update-index --chmod=+x deploy/scripts/backup.sh
```

9. **BACKUP_DIR resolution** — D-17 explicitly supports operator override (`BACKUP_DIR=/mnt/backups bash backup.sh`). Default = `${DEPLOY_DIR}/../backups` (one level up from deploy/, i.e. repo-root-adjacent `./backups/`). The double `cd` + `dirname`/`basename` dance normalises paths so relative inputs work from any CWD.
