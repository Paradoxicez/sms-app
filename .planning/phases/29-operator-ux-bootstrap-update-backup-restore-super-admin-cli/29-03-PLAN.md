```bash
#!/usr/bin/env bash
# ============================================================================
# deploy/scripts/update.sh — Phase 29 (DEPLOY-19)
#
# Atomic image-tag upgrade. Pre-flight migrate test runs against the new
# image with .env unchanged; only on green light is .env rewritten and the
# stack recycled. If migrate fails: .env untouched, services keep running
# the old tag, exit 1.
#
# Usage:
#   bash deploy/scripts/update.sh v1.3.1
#   bash deploy/scripts/update.sh v1.3.2-rc1
#   bash deploy/scripts/update.sh latest    # not recommended for prod
#
# Manual rollback (after a successful update went bad in production):
#   cp deploy/.env.backup-<utc-timestamp> deploy/.env
#   bash deploy/scripts/update.sh <old-tag>
# ============================================================================

set -euo pipefail
IFS=$'\n\t'

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
DEPLOY_DIR="$(cd "${SCRIPT_DIR}/.." && pwd)"
ENV_FILE="${DEPLOY_DIR}/.env"
COMPOSE_FILE="${DEPLOY_DIR}/docker-compose.yml"

# --- TTY-aware color (D-29) ---
if [[ -t 1 ]] && [[ "$(tput colors 2>/dev/null || echo 0)" -ge 8 ]]; then
  GREEN=$'\033[32m'; RED=$'\033[31m'; YELLOW=$'\033[33m'; BOLD=$'\033[1m'; RESET=$'\033[0m'
else
  GREEN=''; RED=''; YELLOW=''; BOLD=''; RESET=''
fi
log()  { printf '%s[update]%s %s\n' "${BOLD}" "${RESET}" "$*"; }
ok()   { printf '%s[update] ✓%s %s\n' "${GREEN}" "${RESET}" "$*"; }
warn() { printf '%s[update] ⚠%s %s\n' "${YELLOW}" "${RESET}" "$*" >&2; }
die()  { printf '%s[update] ✗%s %s\n' "${RED}" "${RESET}" "$*" >&2; exit 1; }

# --- D-13: positional arg + semver/latest regex ---
TAG="${1:-}"
if [[ -z "${TAG}" ]]; then
  printf 'Usage: %s <tag>\n' "$0" >&2
  printf '  tag: vX.Y.Z (e.g. v1.3.1) | vX.Y.Z-prerelease (e.g. v1.3.2-rc1) | latest\n' >&2
  exit 2
fi
if ! [[ "${TAG}" =~ ^v[0-9]+\.[0-9]+\.[0-9]+(-[a-z0-9]+(\.[a-z0-9]+)*)?$|^latest$ ]]; then
  die "Invalid tag format: ${TAG}. Expected vX.Y.Z, vX.Y.Z-prerelease, or latest."
fi

# --- Pre-flight ---
[[ -f "${ENV_FILE}" ]] || die "${ENV_FILE} missing. Cannot update — run bootstrap.sh first."
[[ -f "${COMPOSE_FILE}" ]] || die "${COMPOSE_FILE} missing. Repo layout broken?"

# Source DOMAIN for the post-recycle health probe.
# shellcheck disable=SC1090
set -a; source "${ENV_FILE}"; set +a
[[ -n "${DOMAIN:-}" ]] || die "DOMAIN not set in ${ENV_FILE}. Cannot probe health endpoint."

DC="docker compose -f ${COMPOSE_FILE} --env-file ${ENV_FILE}"

# --- Step 1: pull new images (idempotent — docker layer cache) ---
log "Pulling images for tag ${TAG}..."
IMAGE_TAG="${TAG}" ${DC} pull
ok "Images pulled"

# --- Step 2: D-15 pre-flight migrate test (env override, .env unchanged) ---
log "Pre-flight migrate test on image ${TAG}..."
if ! IMAGE_TAG="${TAG}" ${DC} run --rm sms-migrate; then
  die "Migrate failed on image ${TAG}. Stack unchanged. Inspect: IMAGE_TAG=${TAG} ${DC} run --rm sms-migrate"
fi
ok "Pre-flight migrate green"

# --- Step 3: D-14 backup .env then sed in-place ---
TS=$(date -u +%Y%m%dT%H%M%SZ)
BACKUP="${ENV_FILE}.backup-${TS}"
cp "${ENV_FILE}" "${BACKUP}"
chmod 600 "${BACKUP}"   # match init-secrets.sh perms
sed -i.tmp "s|^IMAGE_TAG=.*|IMAGE_TAG=${TAG}|" "${ENV_FILE}"
rm -f "${ENV_FILE}.tmp"
ok "IMAGE_TAG → ${TAG} (backup: ${BACKUP})"

# --- Step 4: recycle via compose up -d (depends_on chain handles order) ---
log "Recycling stack on ${TAG}..."
${DC} up -d
ok "compose up -d issued"

# --- Step 5: D-16 health verify (poll Caddy /api/health) ---
log "Waiting for /api/health on https://${DOMAIN}..."
for i in $(seq 1 24); do
  if curl -fsS -o /dev/null --max-time 5 "https://${DOMAIN}/api/health" 2>/dev/null; then
    ok "Update complete: ${TAG} live at https://${DOMAIN}"
    log "Rollback (if needed): cp ${BACKUP} ${ENV_FILE} && bash ${SCRIPT_DIR}/update.sh <old-tag>"
    exit 0
  fi
  sleep 5
done

warn "Services unhealthy after 120s. Rollback hint:"
warn "  cp ${BACKUP} ${ENV_FILE} && bash ${SCRIPT_DIR}/update.sh <old-tag>"
warn "Inspect: ${DC} logs --tail 100 api"
exit 1
```

**Critical correctness notes (do NOT deviate):**

1. **Semver regex MUST allow prereleases** (D-13: `v1.3.2-rc1`, `v1.3.0-beta.1`). The exact regex is `^v[0-9]+\.[0-9]+\.[0-9]+(-[a-z0-9]+(\.[a-z0-9]+)*)?$|^latest$`. Lowercase only because Phase 28 D-04 emits lowercase tags via `docker/metadata-action@v5`. Reject `vX.Y` (2-part) and `sha-<7>` per D-13 — encourage explicit semver.

2. **Pre-flight migrate uses `IMAGE_TAG=${TAG} ... run --rm sms-migrate`** — the env var prefix overrides the .env value for THIS shell invocation only. .env on disk is untouched. If migrate exits non-zero, sed never runs and `compose up -d` (which compose-pull + recycle) is never issued. The stack stays on the old tag. THIS is the atomic guarantee that justifies the entire script's existence.

3. **`sed -i.tmp ... && rm -f *.tmp`** — D-14 portable pattern. Works on BSD sed (macOS dev) and GNU sed (Ubuntu prod) without OS detection. The `.tmp` suffix creates a one-shot backup file that's removed immediately on success; on failure the partial sed leaves the original at `<file>.tmp` for forensics.

4. **chmod 600 on the backup** — the backup file inherits .env's secrets (DB_PASSWORD, ADMIN_PASSWORD, ...). Match init-secrets.sh perms (which set 600 on the live .env). Without this, the backup may end up at default 644 (depending on umask) and leak.

5. **`compose up -d` (no service arg)** is enough to trigger the entire recycle chain — Phase 26 `depends_on` declarations make compose recycle services in the correct order. Do NOT add `--force-recreate` (would re-pull volumes and lose data) or `--no-deps` (would skip the chain). Default `up -d` notices image diff, recycles affected services, leaves volumes intact.

6. **Health probe = `https://${DOMAIN}/api/health`** (NOT `http://localhost:3003/api/health`). D-16 ground-truth: probe through Caddy because Caddy is the user-facing surface. If Caddy can answer 200 then the entire reverse-proxy → api chain is healthy, including TLS, healthcheck routing, and api boot.

7. **Exit codes:** 0 success, 1 = migrate failure or post-recycle health failure, 2 = misuse (missing/invalid tag arg). D-29 conventions.

8. **chmod +x in source tree:**
```bash
chmod +x deploy/scripts/update.sh
git update-index --chmod=+x deploy/scripts/update.sh
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator argv → update.sh | Operator types the tag (`v1.3.1`) — no secret material in argv. |
