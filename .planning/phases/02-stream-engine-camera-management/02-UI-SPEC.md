## Spacing Scale

Inherited from Phase 1 (8-point scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status dot margins |
| sm | 8px | Compact element spacing, table cell padding, badge gaps |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, form group gaps, dialog section spacing |
| xl | 32px | Layout gaps between cards, sidebar padding |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets for Start/Stop stream buttons: minimum 44x44px
- Video player container: 16:9 aspect ratio, no fixed pixel spacing
- Status indicator dots: 8px diameter circle with 4px margin-right

Inherited from Phase 1:

| Role | Size | Weight | Line Height | Phase 2 Usage |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 (regular) | 1.5 | Table cells, descriptions, camera details, codec info, form labels |
| Label | 12px | 400 (regular) | 1.4 | Table headers, badges, timestamps, stream profile metadata, status labels |

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(0 0% 100%) | Page backgrounds, main content area |

### Accent reserved for (Phase 2 additions to Phase 1 list):
- "Add Camera" / "Create Project" / "Create Site" primary CTA buttons
- "Start Stream" button
- Active sidebar navigation item
- Camera status badge: "online"
- Focus ring on form inputs
- HLS player play button overlay
- Stream profile "active" indicator
- "Test Connection" button (primary variant)

### Camera Status Colors (5-state machine per D-06):

| Status | Color | CSS Variable | Indicator |
|--------|-------|-------------|-----------|
| online | hsl(142 71% 45%) | --primary (green) | Solid filled dot + "Online" badge |
| offline | hsl(215 16% 47%) | --muted-foreground (gray) | Hollow ring dot + "Offline" badge |
| degraded | hsl(38 92% 50%) | --chart-4 (amber) | Solid filled dot + "Degraded" badge |
| connecting | hsl(142 71% 45%) | --primary (green) | Pulsing dot animation + "Connecting" badge |
| reconnecting | hsl(38 92% 50%) | --chart-4 (amber) | Pulsing dot animation + "Reconnecting" badge |

Status dot: 8px circle, inline before camera name in table rows and camera detail header. Pulsing animation: CSS `animate-pulse` from Tailwind for connecting/reconnecting states.

|---------|------|

| Invalid stream profile | Invalid profile settings: {specific issue}. Adjust the values and try again. |
| Config save failed | Could not save Stream Engine settings. Please try again. |

Error text: 12px, destructive color, displayed below the relevant element with 4px gap. For bulk import, inline red highlight on failing rows with tooltip showing the specific error.

## Interaction Patterns

### Camera Hierarchy Navigation
1. Sidebar nav item "Projects" opens project list
2. Click project row expands to show sites (or navigates to project detail)
3. Click site row shows cameras within that site
4. Breadcrumb always visible: Projects > {Project} > {Site} > Cameras
5. "Cameras" nav item shows flat list of all cameras (with project/site columns)

### Camera Registration Flow
1. Click "Add Camera" on camera list page
2. Dialog opens with form: Name (required), Stream URL (required), Project (select), Site (select), Location (optional lat/lng), Tags (optional comma-separated), Description (optional textarea)
3. "Test Connection" button runs ffprobe in background -- shows Progress bar during test, then displays codec, resolution, FPS results in a success card, or error message on failure
4. If codec is H.265, show amber warning badge: "H.265 detected -- will be transcoded to H.264 for browser playback"
5. "Save Camera" creates camera with status `offline`

### Bulk Import Flow (per D-04, D-05)
1. Click "Import Cameras" button
2. Dialog opens with file upload zone (drag-and-drop + click, accepts .csv and .json)
3. After upload: dialog expands to medium size showing editable table
4. Table columns: Row #, Name, Stream URL, Project, Site, Location, Tags, Status (validation icon)
5. Rows with errors highlighted in destructive/10% background, error tooltip on status column
6. User can click any cell to edit inline
7. Footer shows: "{valid_count} valid, {error_count} errors" + "Confirm Import" button (disabled if errors exist)
8. On confirm: Progress bar shows import progress, cameras created with status `offline`
9. After import: toast notification "Imported {count} cameras successfully" + BullMQ runs ffprobe per camera in background

### Start/Stop Stream
1. Camera detail page: prominent "Start Stream" button (accent/primary variant, Play icon)
2. Click Start: button changes to loading state (Loader2 spinning), text changes to "Starting..."
3. Stream starts: button changes to "Stop Stream" (destructive outline variant, Square icon)
4. HLS preview player appears in Preview tab within 10 seconds (per success criteria)
5. Stream stats appear below player: resolution, FPS, bitrate, codec, viewer count, uptime
6. Click Stop: AlertDialog confirmation, then button returns to "Start Stream"

### Camera Status Real-Time Updates (per D-07)
1. Socket.IO connection established on camera list and detail pages
2. Status dot updates in real-time without page refresh
3. Connecting/reconnecting states show pulsing animation
4. Status change triggers subtle row highlight flash (200ms accent/10% background fade)
5. Degraded status shows tooltip: "Stream quality is below expected threshold"

### Stream Profile Management
1. Profile list shows cards in 2-3 column grid
2. Each card: profile name, codec mode (passthrough/transcode), resolution, bitrate, badge for default
3. "Create Profile" opens dialog with form:
   - Name (text input)
   - Mode: Passthrough / Transcode (RadioGroup)
   - If Transcode: Resolution (Select), FPS (Select), Bitrate (Slider with numeric display), Audio mode (Select: copy/transcode/mute)
4. Validation warnings (per D-11): amber inline messages for incompatible settings
5. Profile assignment: on camera detail, Stream Profile tab, Select dropdown to assign profile

### Stream Engine Settings (per D-15, D-16)
1. Page title: "Stream Engine Settings" (never "SRS Settings" per D-15)
2. System tab (Super admin only):
   - HLS fragment size: number input (seconds), default 2
   - HLS window size: number input (seconds), default 10
   - RTMP port: number input, default 1935
   - SRT port: number input, default 10080
   - Connection timeout: number input (seconds), default 30
   - HLS encryption: Switch toggle
   - "Save Settings" triggers srs.conf regeneration + reload
3. Organization Defaults tab:
   - Default stream profile: Select from existing profiles
   - Max reconnect attempts: number input, default 10
   - Auto-start on boot: Switch toggle
   - Default recording mode: Select (off/continuous/event)
4. Save shows loading state, then success toast "Settings saved"

### HLS Preview Player
1. 16:9 container with dark background (hsl(0 0% 9%))
2. When stream inactive: centered text "Stream not active" with muted foreground
3. When stream starting: centered Loader2 spinning animation
4. When stream active: hls.js player renders video, native controls visible
5. Below player: stats bar with resolution, codec, FPS, bitrate, viewers, uptime in Label size (12px)

### Test Connection UI
1. "Test Connection" button in camera form (secondary variant)
2. On click: button disabled, Progress component appears below with indeterminate animation
3. On success: green-bordered card appears showing:
   - Codec: H.264 / H.265 (with badge)
   - Resolution: 1920x1080
   - FPS: 30
   - Audio: AAC
4. On failure: destructive-bordered card with error message
5. Results persist until form is closed or URL is changed
