**4. Create `apps/api/src/api-keys/api-keys.service.ts`:**
- Inject `TENANCY_CLIENT` (Prisma) and `Redis` (ioredis) via constructor
- `generateKey()`: Use `crypto.randomBytes(32).toString('hex')`, prefix with `sk_live_`, hash with `crypto.createHash('sha256').update(rawKey).digest('hex')`, extract last 4 chars
- `create(orgId, dto)`: Validate scopeId exists (query Project or Site by id+orgId). Generate key. Store hashed key. Return `{ id, name, rawKey, prefix, lastFour, scope, scopeId, createdAt }`. The rawKey is returned ONLY on creation.
- `findAll(orgId)`: Return all keys for org (id, name, prefix, lastFour, scope, scopeId, revokedAt, lastUsedAt, createdAt). Never return keyHash.
- `findByHash(keyHash)`: Lookup by hash for guard authentication. Return full record including orgId, scope, scopeId, revokedAt.
- `revoke(id, orgId)`: Set revokedAt = new Date(). Return updated key.
- `updateLastUsed(id)`: Update lastUsedAt timestamp (called from guard).
- `recordUsage(keyId, bytes)`: Redis INCR `apikey:usage:{keyId}:{YYYY-MM-DD}:requests` and INCRBY `apikey:usage:{keyId}:{YYYY-MM-DD}:bandwidth` by bytes.
- `aggregateDaily()`: Scan Redis keys matching `apikey:usage:*:requests`, parse keyId and date, upsert ApiKeyUsage records in PostgreSQL, delete processed Redis keys. This is called by a BullMQ repeatable job.
- `getUsageStats(keyId, days)`: Query ApiKeyUsage for last N days, return array of `{ date, requests, bandwidth }`.
- For Redis, use `ioredis` -- import the Redis class and inject via a custom provider or create directly from env vars (REDIS_HOST, REDIS_PORT same as BullMQ config).

**5. Create `apps/api/src/api-keys/api-key.guard.ts`:**
Per D-01, authenticate via `X-API-Key` header:
```typescript
@Injectable()
export class ApiKeyGuard implements CanActivate {
  constructor(
    private readonly cls: ClsService,
    private readonly apiKeysService: ApiKeysService,
  ) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest<Request>();
    const apiKey = request.headers['x-api-key'] as string;
    if (!apiKey) throw new UnauthorizedException('API key required');

    const keyHash = createHash('sha256').update(apiKey).digest('hex');
    const keyRecord = await this.apiKeysService.findByHash(keyHash);
    if (!keyRecord || keyRecord.revokedAt) {
      throw new UnauthorizedException('Invalid or revoked API key');
    }

    this.cls.set('ORG_ID', keyRecord.orgId);
    (request as any).apiKey = keyRecord;

    // Fire-and-forget: update lastUsedAt
    this.apiKeysService.updateLastUsed(keyRecord.id).catch(() => {});

    return true;
  }
}
```

**1. Create `apps/api/src/api-keys/api-key-usage.middleware.ts`:**

A NestJS middleware that runs AFTER ApiKeyGuard sets `request.apiKey`. It counts each request and tracks response size:

```typescript
import { Injectable, NestMiddleware } from '@nestjs/common';
import { Request, Response, NextFunction } from 'express';
import { ApiKeysService } from './api-keys.service';

@Injectable()
export class ApiKeyUsageMiddleware implements NestMiddleware {
  constructor(private readonly apiKeysService: ApiKeysService) {}

  use(req: Request, res: Response, next: NextFunction) {
    const apiKey = (req as any).apiKey;
    if (!apiKey) {
      return next();
    }

    // Track response size for bandwidth
    const originalEnd = res.end.bind(res);
    let totalBytes = 0;

    const originalWrite = res.write.bind(res);
    res.write = (...args: any[]) => {
      if (args[0]) {
        totalBytes += Buffer.byteLength(args[0]);
      }
      return originalWrite(...args);
    };

    res.end = (...args: any[]) => {
      if (args[0]) {
        totalBytes += Buffer.byteLength(args[0]);
      }
      // Fire-and-forget usage recording
      this.apiKeysService.recordUsage(apiKey.id, totalBytes).catch(() => {});
      return originalEnd(...args);
    };

    next();
  }
}
```

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-01 | Spoofing | ApiKeyGuard | mitigate | SHA-256 hash lookup indexed for constant-time DB query; never log raw keys |
