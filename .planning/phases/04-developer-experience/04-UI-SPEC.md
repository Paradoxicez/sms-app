## Spacing Scale

Inherited from Phase 1/2/3 (8-point scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status indicator margins |
| sm | 8px | Compact element spacing, table cell padding, badge gaps |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, form group gaps, dialog section spacing |
| xl | 32px | Layout gaps between cards, guide section spacing |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Code snippet blocks (curl examples, embed snippets): 12px internal padding (monospace content)
- Touch targets for copy-to-clipboard buttons: minimum 44x44px
- Swagger UI embed iframe: full-width container, 0px internal padding (Swagger controls its own layout)
- Quick Start step cards: 20px internal padding (compact but readable for numbered steps)

## Typography

Inherited from Phase 1/2/3:

| Role | Size | Weight | Line Height | Phase 4 Usage |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 (regular) | 1.5 | API key descriptions, webhook URLs, form descriptions, documentation prose, delivery log entries |
| Label | 12px | 400 (regular) | 1.4 | Table headers, usage stats labels, key prefix display, delivery timestamps, guide section labels |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles ("API Keys", "Webhooks", "Quick Start"), card headers, documentation guide titles |
| Display | 28px | 600 (semibold) | 1.2 | Developer Portal overview heading ("Developer Portal"), usage stat large numbers |

Monospace exception: API keys (`sk_live_...****`), curl command blocks, embed code snippets, webhook payloads (JSON), and webhook URLs use `font-mono text-xs` (12px monospace) for technical readability.

## Interaction Patterns

### API Key Creation Flow
1. Click "Create API Key" on API Keys page
2. Dialog opens with: Name input, Scope selector (RadioGroup: Project / Site), Entity selector (Select dropdown filtered by scope)
3. Click "Create Key" -- backend generates key, returns raw key once
4. Dialog transitions to ApiKeyRevealCard view:
   - Large monospace display of full key: `sk_live_abc123...xyz`
   - Prominent "Copy Key" button (accent color)
   - Warning text in amber: "Copy this key now. It will not be shown again."
   - "Done" button to close dialog
5. After closing, key appears in table showing prefix + last 4 chars only (e.g., `sk_live_...a1b2`)
6. Toast: "API key created"

### API Key Revocation Flow
1. Click dropdown menu (MoreHorizontal icon) on key row -> "Revoke"
2. AlertDialog confirmation with key name and last 4 chars
3. On confirm: key marked as revoked, row shows "Revoked" badge in muted gray
4. Toast: "API key revoked"

### Webhook Subscription Creation Flow
1. Click "Create Webhook" on Webhooks page
2. Dialog opens with: Name input, URL input (with HTTPS validation), Event type checkboxes (camera.online, camera.offline, camera.degraded, camera.reconnecting)
3. Click "Create Webhook" -- backend generates HMAC secret
4. Dialog transitions to WebhookSecretReveal view:
   - Display of HMAC signing secret
   - "Copy Secret" button
   - Warning: "Copy this secret now. It will not be shown again."
   - "Done" button to close
5. New subscription appears in webhooks table
6. Toast: "Webhook created"

### Webhook Delivery Inspection Flow
1. Click webhook subscription row to navigate to detail page
2. Detail page shows subscription info card at top
3. Below: delivery log table with columns: Event (EventTypeBadge), Status (success/failed/pending badge), Response Code, Timestamp, Attempts
4. Click a delivery row to expand inline: shows full JSON payload and response body
5. Failed deliveries show retry schedule info: "Next retry in {time}" or "All retries exhausted"

### Quick Start Guide Interaction
1. Developer Portal overview page loads with 3-step QuickStartGuide
2. Each step is a card with: step number (accent circle), title, description, and code example
3. Code examples are pre-filled with:
   - Step 1: curl to create API key (uses session auth, not API key -- the user is creating their first key)
   - Step 2: curl to create playback session (placeholder `YOUR_API_KEY` + first camera ID from org)
   - Step 3: iframe embed snippet with placeholder session URL
4. When user has an active API key, Step 2 and Step 3 examples auto-populate with real key value
5. Each code block has "Copy" button (top-right)

### Documentation Page Interaction
1. Docs index shows 5 guide cards in a 2-column grid
2. Each card: Lucide icon (24px, accent color) + title (Heading 20px) + 1-line description (Body 14px, muted)
3. Click card to navigate to guide page
4. Guide pages are static content with Heading/Body typography, code examples in CodeBlock components
5. No interactive elements in guide content (read-only reference)

### Swagger UI Access
1. "Open API Reference" button in Developer Portal header area (or sidebar nav item "API Reference")
2. Opens `/api/docs` in new tab (target="_blank")
3. Swagger UI is served by NestJS, completely separate from Next.js app
4. No login required -- public access per D-06
