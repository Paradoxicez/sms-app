## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` / `hsl(0 0% 9%)` dark | Background, page surfaces |
| Secondary (30%) | `hsl(142 30% 97%)` / `hsl(142 20% 15%)` dark | Cards, sidebar, table header background |
| Accent (10%) | `hsl(142 71% 45%)` | Primary CTA buttons, active sidebar items, focus rings, status "Active" badge accent |
| Destructive | `hsl(0 84% 60%)` | Delete API key, remove member, remove node, deactivate org |

Accent reserved for: "Create" / "Add" primary buttons, active nav highlight, focus ring, chart primary color. Never for table row backgrounds or status badges (badges use outline variant with semantic colors).

### New Pattern: API Key Reveal Dialog

After creating an API key, show a dialog with:
- Monospace full key text in a bordered box with `bg-muted` background
- Copy button (Lucide `Copy` icon) aligned right inside the key box
- Warning text below the key box using `text-sm text-muted-foreground` with Lucide `AlertTriangle` icon
- Single "Done" button at dialog footer (no cancel needed, key is already created)

## Interaction Patterns

### DataTable Standard Behavior (all 4 tables)

- **Search:** Debounced text filter (300ms) across primary text column (name for most tables)
- **Sort:** Click column header to toggle asc/desc/none. Sort indicator arrow in header.
- **Filter:** Faceted filter badges in toolbar. Multi-select. Show active filter count.
- **Pagination:** 10 rows per page default. Show "X of Y rows" count. Previous/Next buttons.
- **Loading:** Skeleton rows (5 rows of skeleton blocks matching column layout)
- **Row actions:** Three-dot menu (MoreHorizontal icon) in last column. Opens dropdown menu.

### API Key Copy Fix (FIX-02)

- Remove "Copy key" action from table row actions (it can only copy masked version)
- Copy is only available in the post-creation dialog where the full key is shown
- After copy, show checkmark icon briefly (1.5s) then revert to copy icon

### API Key Delete Fix (FIX-03)

- Change from soft-delete (revoke) to hard delete
- Show AlertDialog confirmation before delete
- On success: remove row from table, show success toast
- On error: show error toast, keep row in table

### Destructive Actions Summary

| Action | Page | Confirmation Type | Button Variant |
|--------|------|-------------------|----------------|
| Delete API Key | API Keys | AlertDialog | destructive |
| Remove Member | Team | AlertDialog | destructive |
| Deactivate Org | Organizations | AlertDialog | destructive |
| Remove Node | Cluster Nodes | AlertDialog | destructive |

All destructive actions use the existing AlertDialog pattern: title, description, Cancel + Confirm buttons.
