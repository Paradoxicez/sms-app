**FIX-01 (D-03): System org user creation RLS fix**

In `apps/api/src/users/users.service.ts`, modify the `createUser` method. Replace the direct `this.prisma.member.create()` call (line 62-68) with a transaction that sets the RLS context first:

```typescript
// Replace lines 62-68:
const member = await this.prisma.member.create({
  data: {
    id: randomUUID(),
    organizationId: orgId,
    userId,
    role: dto.role,
  },
});

// With this transaction-based approach:
const [, member] = await this.prisma.$transaction([
  this.prisma.$executeRaw`SELECT set_config('app.current_org_id', ${orgId}, TRUE)`,
  this.prisma.member.create({
    data: {
      id: randomUUID(),
      organizationId: orgId,
      userId,
      role: dto.role,
    },
  }),
]);
```

This sets the RLS context within the transaction so the Member INSERT passes the FORCE RLS policy. The `TRUE` parameter makes it transaction-local.

**FIX-03 (D-02): API key hard delete**

```typescript
/**
 * Hard-delete an API key and its usage records (cascade).
 */
async delete(id: string, orgId: string) {
  const key = await this.tenancy.apiKey.findFirst({
    where: { id, orgId },
  });
  if (!key) {
    throw new NotFoundException(`API key ${id} not found`);
  }

  await this.tenancy.apiKey.delete({ where: { id } });
  return { deleted: true };
}
```
