```yaml
# .github/workflows/build-images.yml — Phase 28 (DEPLOY-03, DEPLOY-04, DEPLOY-05).
#
# Builds production api + web images on a parallel matrix, smoke-tests each
# locally-loaded build (Plan 01 .github/scripts/smoke-*.sh), pushes to GHCR
# with the 4-tag semver scheme via metadata-action@v5, and attaches sigstore
# build provenance attestation via attest-build-provenance@v2.
#
# Trigger matrix (D-01):
#   push: branches: [main]    -> :main + :latest + :sha-<7>          (push: yes, attest: yes)
#   push: tags: ['v*.*.*']    -> stable: :vX.Y.Z + :vX.Y + :latest    (push: yes, attest: yes)
#                              prerelease: :vX.Y.Z-test + :sha-<7>    (push: yes, attest: yes)
#   pull_request:             -> :pr-<n> (build only — D-02)         (push: NO,  attest: NO)
#   workflow_dispatch:        -> manual rebuild (no inputs — D-22 deferred)  (push: yes, attest: yes)
#
# Pre-release tag policy (D-06): metadata-action's type=semver pattern
# automatically skips prerelease semver tags (-alpha/-beta/-rc/-test), so
# `latest` and `vX.Y` aliases never get assigned to a prerelease accidentally.
#
# Auth: GITHUB_TOKEN only — no PAT (Pitfall 11 mitigated by D-19 public images).
# Test-gate co-existence: .github/workflows/test.yml (Phase 23 DEBT-02) runs
# in parallel; this workflow does NOT duplicate vitest (D-20).

name: Build & Publish Images

on:
  push:
    branches:
      - main
    tags:
      - 'v*.*.*'
  pull_request:
  workflow_dispatch:

permissions:
  contents: read           # checkout
  packages: write          # GHCR push (Pitfall 11)
  id-token: write          # OIDC for sigstore attestation
  attestations: write      # actions/attest-build-provenance@v2

# D-03: cancel in-progress on PR + main; NEVER cancel tag builds (every release
# must complete + attest, even if a follow-up tag arrives mid-build).
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: ${{ github.event_name != 'push' || !startsWith(github.ref, 'refs/tags/') }}

env:
  REGISTRY: ghcr.io
  IMAGE_NAMESPACE: ${{ github.repository_owner }}/sms

jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    strategy:
      fail-fast: false      # api and web are independent; one failing should not abort the other
      matrix:
        app: [api, web]
    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to GHCR
        if: github.event_name != 'pull_request'
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Compute image tags + OCI labels (metadata-action D-05, D-07)
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAMESPACE }}-${{ matrix.app }}
          tags: |
            type=ref,event=branch
            type=ref,event=pr
            type=sha,format=short,prefix=sha-
            type=semver,pattern={{version}}
            type=semver,pattern={{major}}.{{minor}}
            type=raw,value=latest,enable={{is_default_branch}}

      - name: Build image and load to local Docker daemon (D-10 step 1)
        id: build_load
        uses: docker/build-push-action@v6
        with:
          context: .
          file: apps/${{ matrix.app }}/Dockerfile
          platforms: linux/amd64
          load: true
          tags: smoke-${{ matrix.app }}:latest
          cache-from: type=gha,scope=${{ matrix.app }}
          cache-to: type=gha,mode=max,scope=${{ matrix.app }}

      - name: Smoke test loaded image (D-10 step 2 — gates push)
        run: bash .github/scripts/smoke-${{ matrix.app }}.sh smoke-${{ matrix.app }}:latest

      - name: Build & push image with full tag set (D-10 step 3)
        id: push
        if: github.event_name != 'pull_request'
        uses: docker/build-push-action@v6
        with:
          context: .
          file: apps/${{ matrix.app }}/Dockerfile
          platforms: linux/amd64
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha,scope=${{ matrix.app }}
          cache-to: type=gha,mode=max,scope=${{ matrix.app }}

      - name: Attest build provenance (DEPLOY-05, D-09)
        if: github.event_name != 'pull_request'
        uses: actions/attest-build-provenance@v2
        with:
          subject-name: ${{ env.REGISTRY }}/${{ env.IMAGE_NAMESPACE }}-${{ matrix.app }}
          subject-digest: ${{ steps.push.outputs.digest }}
          push-to-registry: true
```

Decision references in this file:
- D-01 trigger matrix: `on: { push: { branches: [main], tags: ['v*.*.*'] }, pull_request: {}, workflow_dispatch: {} }`
- D-02 conditional push: `if: github.event_name != 'pull_request'` on login + push + attest steps
- D-03 concurrency: `cancel-in-progress: ${{ github.event_name != 'push' || !startsWith(github.ref, 'refs/tags/') }}`
- D-04 image namespace: `IMAGE_NAMESPACE: ${{ github.repository_owner }}/sms` → final image name `ghcr.io/<owner>/sms-${matrix.app}`
- D-05 metadata-action tag list: 6 type entries (ref+branch, ref+pr, sha+short+sha- prefix, semver version, semver major.minor, raw latest gated by is_default_branch)
- D-06 prerelease policy: handled implicitly by metadata-action — type=semver skips prerelease semver tags; type=raw `enable={{is_default_branch}}` ensures `:latest` never lands on a non-main commit
- D-07 OCI labels: `labels: ${{ steps.meta.outputs.labels }}` passes metadata-action's auto-generated `org.opencontainers.image.*` set to build-push-action
- D-08 permissions: `contents: read`, `packages: write`, `id-token: write`, `attestations: write` — minimal viable scope
- D-09 attestation: `actions/attest-build-provenance@v2` with `subject-digest: ${{ steps.push.outputs.digest }}` + `push-to-registry: true` so attestation is co-located with image
- D-10 two-step build pattern: build_load (load: true to local docker) → smoke test → build push (cache hit makes step 3 ~30s)
- D-13 smoke script location: `bash .github/scripts/smoke-${{ matrix.app }}.sh smoke-${{ matrix.app }}:latest`
- D-14 cache: `cache-from: type=gha,scope=${{ matrix.app }}` + `cache-to: type=gha,mode=max,scope=${{ matrix.app }}` — separate cache scope per matrix arm
- D-20 test-gate co-existence: this file does NOT run vitest; test.yml owns that
- D-21 branch protection: this file does not modify ruleset; that is operator action post-Phase-28

Why `fail-fast: false` on the matrix: api and web are independent images. If api smoke fails on a PR but web build succeeds, we want the web success signal preserved (operator can still merge a web-only fix while api regression is debugged separately). GitHub default `fail-fast: true` would cancel web mid-build.

Why `timeout-minutes: 30`: Phase 25 Dockerfile builds on a cold cache take ~5-8 min (deps + builder + prod-deps + runtime stages with FFmpeg apt install). Smoke ~1 min. Push ~1-2 min. Attestation ~30s. Total ~10-12 min on cold cache, ~2-3 min on warm. 30 min ceiling = 2.5× cold-cache budget; matches CONTEXT.md "Claude's discretion" allowance for timeout-minutes.

Why `if: github.event_name != 'pull_request'` on the LOGIN step (not just push step): logging in to GHCR on a PR build wastes a step + briefly stores credentials in the runner cache. Skipping the login step on PRs is a tiny defense-in-depth + faster step count for PR feedback.

Why two separate `build-push-action@v6` invocations (build_load then push) instead of one `build-push-action` with `outputs: type=docker,dest=...` + `docker load`: D-10 picks the two-step pattern explicitly because the GH Cache backend reuses ALL stage layers between the two invocations; the second build is purely a re-pack-and-push. The alternative `outputs: type=docker` exports a tarball + requires extra `docker load` shell step + does not interact with `cache-to` cleanly. Two-step build_load → push trades ~30s push-step time for a much simpler step structure.

Why `push-to-registry: true` on attest-build-provenance@v2: stores the attestation as an OCI artifact alongside the image in GHCR (rather than only in the GitHub Sigstore transparency log). This makes `gh attestation verify oci://ghcr.io/...` work without authenticating against the GitHub-side log — operators can verify with only their GHCR pull credentials (which are anonymous for public images).
