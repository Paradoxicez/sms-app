# Phase 11 — UI Design Contract

> Visual and interaction contract for camera management: DataTable with faceted filters, card view with HLS hover preview, quick actions menu, and slide-in View Stream sheet. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova style) |
| Preset | base-nova (neutral base, CSS variables, Tailwind 4.2) |
| Component library | base-ui (render prop pattern, NOT Radix asChild) |
| Icon library | lucide-react |
| Font | Inter (ui-sans-serif fallback) |

Source: Inherited from Phase 8/10 UI-SPEC. No changes.

## Color

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `hsl(0 0% 100%)` / `var(--background)` | Page background, table background, card background |
| Secondary (30%) | `hsl(220 14% 96%)` / `var(--muted)` | Table row hover (`bg-muted/50`), selected row (`bg-muted`), toolbar area, card hover background |
| Accent (10%) | `hsl(142 71% 45%)` / `var(--primary)` | "Add Camera" CTA button, checkbox checked state, active sort arrow, focus ring, active pagination page, table/grid toggle active state |
| Destructive | `hsl(0 84% 60%)` / `var(--destructive)` | Delete menu item text, destructive confirmation dialog button |

Accent reserved for:
- "Add Camera" primary CTA button in toolbar
- Checkbox checked fill color
- Active column sort direction arrow
- Focus ring on interactive elements
- Active pagination page number
- Active view toggle button (table/grid)

NOT used for: table borders, filter chips at rest, row action "..." button, card borders, status badge backgrounds, view toggle inactive state.

### Grid Layout (D-04)

| Breakpoint | Columns | Gap |
|------------|---------|-----|

### Card Anatomy (D-05)

```
+------------------------------------------+
|                                          |
|   [Camera icon placeholder / thumbnail]  |   aspect-video
|   [Status badge overlay, top-right]      |
|                                          |
+------------------------------------------+
| Camera Name              [Status dot]    |   p-3
| Site name                [...] menu      |   text-xs text-muted-foreground
+------------------------------------------+
```

| Element | Specification |
|---------|---------------|
| Card container | `rounded-lg border bg-card hover:bg-muted/50 cursor-pointer transition-colors` |
| Thumbnail area | `aspect-video bg-muted rounded-t-lg` with centered `Video` icon (lucide, 48px, `text-muted-foreground/40`) |
| Status badge | `CameraStatusBadge` positioned `absolute top-2 right-2` |
| Camera name | `text-sm font-medium truncate` |
| Site label | `text-xs text-muted-foreground truncate` |
| Status dot | `CameraStatusDot` inline next to name |
| Actions menu | "..." `MoreHorizontal` icon button, `absolute bottom-3 right-3` |
| Card dimensions | Natural height from content, no fixed height |

### Card Hover Preview (D-06)

| Property | Value |
|----------|-------|
| Trigger | `mouseenter` on card thumbnail area |
| Debounce | 300ms delay before starting HLS player (prevents flicker on quick mouse passes) |
| Player | `HlsPlayer` with `autoPlay=true`, muted, no controls visible |
| Buffer limits | `maxBufferLength: 4`, `backBufferLength: 0` (minimal memory) |
| Destroy | On `mouseleave`, immediately destroy hls.js instance and remove video element |
| Max concurrent | 6 simultaneous hover players via IntersectionObserver + ref counter |
| Transition | Fade in over 200ms (`transition-opacity duration-200`) |
| Fallback | If stream offline, keep showing placeholder icon; do not show error state |

### Card Click (D-07)

Click anywhere on card (except "..." menu) opens View Stream sheet for that camera.

### Policies Tab (D-18)

Renders `ResolvedPolicyCard` for the selected camera. If no policies resolved, show:
- "No policies applied" heading
- "This camera inherits default platform policies." body

### Activity Tab (D-19)

Renders `AuditLogDataTable` with `cameraId` filter prop. Reuses existing component with server-side pagination.

### Camera Switching (D-20)

When a different camera row or card is clicked while the sheet is open, update the sheet content without closing/reopening. Implementation: controlled `selectedCameraId` state that the sheet reads from. Sheet content re-renders when ID changes.

## Accessibility Contract

Inherited from Phase 8/10 plus:

| Requirement | Implementation |
|-------------|----------------|
| Card view keyboard navigation | Cards are focusable (`tabindex="0"`), Enter/Space opens View Stream sheet |
| View toggle | `aria-label="Switch to table view"` / `"Switch to card view"`, `aria-pressed` on active button |
| Sheet focus trap | Sheet traps focus when open, Escape closes, focus returns to triggering element |
| HLS hover preview | Hover preview is decorative; no keyboard trigger needed (card click opens sheet with full player) |
| Status dot | `title` attribute with status label for screen readers (already in `CameraStatusDot`) |
| Record toggle | Menu item `aria-label` includes current state: "Start recording for {camera name}" |
| Tab panel | Tabs use proper `role="tablist"` / `role="tab"` / `role="tabpanel"` (shadcn Tabs handles this) |
