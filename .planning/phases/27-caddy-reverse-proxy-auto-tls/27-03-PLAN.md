<objective>
Fix the **mixed-content blocker (T-27-MIXED)** in `apps/api/src/recordings/minio.service.ts` per CONTEXT D-26: when DOMAIN-based URLs are emitted into HTTPS-served pages, the API currently produces `http://${DOMAIN}:443/avatars/<id>.webp` because `getAvatarUrl()` and `getSnapshotUrl()` derive the URL scheme from `MINIO_USE_SSL`, which is correctly `false` for the api↔minio internal SDK connection but is the WRONG signal for the browser-facing URL scheme. Browsers block these URLs as mixed content → avatars + camera snapshots disappear in production.

**Decision (locked in this plan):** Implement **Option A** from CONTEXT D-26 — introduce a single new env var `MINIO_PUBLIC_URL` (e.g. `https://${DOMAIN}`) that, when set, becomes the EXCLUSIVE prefix for public URLs. When unset, fall back to the legacy `${scheme}://${endpoint}:${port}/<bucket>/<id>` composition for dev-loop compatibility. This is minimal-blast-radius (no env semantics changed for the api→minio SDK path) and gives a single source of truth for the public host.

Purpose: Closes T-27-MIXED before Phase 30 GA gate. Aligns api-emitted URLs with the Caddy `/avatars/*` + `/snapshots/*` reverse_proxy paths that plan 27-01 introduces.

Output: ~20 LOC in `minio.service.ts` (helper + two call-site swaps) + 2-3 added/updated test cases in `minio-avatars.test.ts`. Plan 27-04 wires `MINIO_PUBLIC_URL=https://${DOMAIN}` into the compose api service env block.
</objective>

**Step 1 — Add a private helper to MinioService** (insert just BEFORE `getAvatarUrl` at line ~111). The helper returns the public URL prefix and bucket sub-path:

```typescript
/**
 * Build a public URL for a MinIO object. Phase 27 (D-26): when
 * MINIO_PUBLIC_URL is set (production via Caddy), use it as the exact
 * prefix — this eliminates the mixed-content blocker where the legacy
 * `${MINIO_USE_SSL ? 'https' : 'http'}://${endpoint}:${port}/...` path
 * emitted `http://` even when the page was served over HTTPS, because
 * MINIO_USE_SSL describes the api↔minio SDK connection (internal HTTP)
 * not the browser-facing URL. Falls back to the legacy composition for
 * dev/non-Caddy environments.
 */
private buildPublicUrl(bucket: string, objectName: string, version: number): string {
  const publicUrl = this.configService.get<string>('MINIO_PUBLIC_URL');
  if (publicUrl) {
    const base = publicUrl.replace(/\/+$/, ''); // strip trailing slashes
    return `${base}/${bucket}/${objectName}?v=${version}`;
  }
  // Legacy dev fallback: derive endpoint+port+scheme like before.
  const endpoint =
    this.configService.get<string>('MINIO_PUBLIC_ENDPOINT') ??
    this.configService.get<string>('MINIO_ENDPOINT', 'localhost');
  const port =
    this.configService.get<string>('MINIO_PUBLIC_PORT') ??
    this.configService.get<string>('MINIO_PORT', '9000');
  const scheme =
    this.configService.get<string>('MINIO_USE_SSL') === 'true' ? 'https' : 'http';
  return `${scheme}://${endpoint}:${port}/${bucket}/${objectName}?v=${version}`;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| api → minio (SDK, internal Docker bridge) | HTTP-only on `:9000`; `MINIO_USE_SSL=false` is correct for this leg. Unchanged by this plan. |
| api response body → browser | Public URLs in JSON responses; if scheme is `http` on a TLS page, browser blocks (mixed content). THIS IS WHAT WE'RE FIXING. |
| Operator-supplied env var (MINIO_PUBLIC_URL) | Trusted (set in deploy/.env, chmod 600). Plan 27-04 wires `MINIO_PUBLIC_URL=https://${DOMAIN}` into the compose api service. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-27-MIXED | Information Disclosure / Availability | api JSON responses with avatar/snapshot URLs | mitigate | New `MINIO_PUBLIC_URL` env-var path emits the exact prefix supplied by the operator — production sets `https://${DOMAIN}` (plan 27-04), so all URLs start with `https://`. The `expect(url).toMatch(/^https:\/\//)` test (Test 6) prevents regression. |
| T-27-PUBLIC-URL-INJECTION | Tampering | URL string composition | accept | The new env var is operator-controlled (chmod 600 on deploy/.env). If an attacker has write access to `.env`, they already have system-level compromise; this code path is not the weakest link. |
| T-27-LEGACY-FALLBACK-DRIFT | Information Disclosure | dev-compose URLs when MINIO_PUBLIC_URL unset | accept | Dev compose intentionally serves over `http://localhost`; legacy fallback emits `http://` on dev. Documented in helper comment; no production exposure. |
</threat_model>
