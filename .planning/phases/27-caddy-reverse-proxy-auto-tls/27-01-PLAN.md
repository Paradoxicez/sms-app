```caddy
# deploy/Caddyfile — Phase 27 (DEPLOY-06, DEPLOY-07, DEPLOY-08)
# Source patterns: https://caddyserver.com/docs/caddyfile/directives/{handle,reverse_proxy}
#                  https://caddyserver.com/docs/caddyfile/options
#                  https://caddyserver.com/docs/automatic-https
# See: deploy/DOMAIN-SETUP.md for ACME troubleshooting + staging toggle.

{
	# ACME — defaults to Let's Encrypt prod CA. Override via ACME_CA env var
	# (e.g. https://acme-staging-v02.api.letsencrypt.org/directory) for debug.
	acme_ca {$ACME_CA:https://acme-v02.api.letsencrypt.org/directory}
	email {$ACME_EMAIL}

	# Lock down attack surface: no admin API + no HTTP/3.
	admin off
	protocols h1 h2
}

{$DOMAIN} {
	# api — REST + Socket.IO handshake. Defensive matcher covers bare /api too
	# (Caddy `/api/*` does NOT match `/api` without trailing slash — Pitfall 2).
	@api path /api /api/*
	handle @api {
		reverse_proxy api:3003
	}

	# Socket.IO upgrade — single matcher covers all 4 namespaces
	# (/notifications, /camera-status, /cluster-status, /srs-logs); namespace
	# is in query/auth, not the URL path. WebSocket upgrade auto-handled.
	handle /socket.io/* {
		reverse_proxy api:3003
	}

	# MinIO public-read buckets — same-origin avoids CORS + mixed-content.
	handle /avatars/* {
		reverse_proxy minio:9000
	}
	handle /snapshots/* {
		reverse_proxy minio:9000
	}

	# Default: web (Next.js).
	handle {
		reverse_proxy web:3000
	}
}
```

Key constraints (DO NOT deviate):
- The global options block (`{ ... }` with no site name) MUST come first.
- `acme_ca {$ACME_CA:...}` MUST use the `{$VAR:default}` syntax — empty `ACME_CA` falls through to prod Let's Encrypt URL `https://acme-v02.api.letsencrypt.org/directory` (D-09, verified per Caddyfile concepts docs).
- `email {$ACME_EMAIL}` MUST NOT have a default — operator must supply (D-10). Empty value = anonymous ACME account (Caddy still works, but operator misses renewal warnings).
- `admin off` (D-11) and `protocols h1 h2` (D-12) MUST be inside the global options block.
- Site name MUST be `{$DOMAIN}` (env-var substitution at Caddyfile load time per Pitfall 7 — operator must set DOMAIN before first `up`).
- `handle` directives (mutually exclusive, per Architecture Pattern 1) — NEVER use `route` (sequential, would break exclusivity).
- The 5 `handle` blocks MUST appear in this readability order: api → socket.io → avatars → snapshots → catch-all. (Caddy auto-sorts by matcher specificity at runtime; order is for human readers.)
- Comments per the snippet — keep them; they explain non-obvious choices to future operators.

Mitigates threats:
- T-27-CADDYFILE-RW: file is the source-of-truth that plan 27-02 mounts read-only via `:ro` flag.
- T-27-ADMIN-API: `admin off` removes the :2019 admin endpoint entirely.
- T-27-WS-MITM: TLS termination + auto-HTTPS forces wss:// for all WebSocket clients.
- T-27-ACME-DOS: `acme_ca` env-var toggle lets operators debug against staging without burning prod rate limit.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Caddy :80/:443 | Untrusted public traffic terminates TLS at Caddy; only path matchers in this Caddyfile decide what reaches upstreams. |
| Caddy → upstreams (api/web/minio) | Internal Docker bridge network; service DNS resolves on `edge` + `internal`. |
| Operator → Caddyfile (read-only mount in plan 27-02) | Container runtime cannot rewrite the config; plan 27-02 enforces `:ro`. |
