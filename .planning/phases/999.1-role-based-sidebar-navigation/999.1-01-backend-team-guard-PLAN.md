Enable tenant-portal Team page (Plan 04) by relaxing UsersController to accept Org Admins
for their own org, and exposing a lightweight endpoint the web client uses to determine
the caller's Member.role in the active org.

Admin creates users within own org). We introduce OrgAdminGuard (union: super admin OR
Member.role='admin' in the `:orgId` route param) and add `GET .../members/me` so the web
client can role-gate its UI without client-side filtering of the full member list.
Output: Turn the red Wave 0 tests (`members-me.test.ts`, `org-admin-guard.test.ts`) green.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → Nest controller | Untrusted session token + orgId path param; guard is the first line of defense |
| Nest service → Prisma | orgId + userId used in `where` clause; Prisma parametrises — no SQL injection concern. RLS on Member table enforced by existing policies. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-999.1-03 | E (Elevation of privilege) | OrgAdminGuard | mitigate | Guard queries Member table with BOTH userId AND organizationId AND role='admin' — cannot be satisfied by org admin of a DIFFERENT org. Test `rejects org admin from creating user in a DIFFERENT org` in `org-admin-guard.test.ts` covers this. |
| T-999.1-04 | I (Information disclosure) | GET members/me | mitigate | Returns only caller's own Member row (userId from session, not from request body). 404 (not 403) on non-member to avoid leaking org existence — acceptable for SaaS where orgIds are not secret (they're UUIDs in URLs). |
| T-999.1-05 | S (Spoofing) | session extraction | mitigate | Reuses Better Auth `getSession` — same validation as SuperAdminGuard. No bespoke token parsing. |
| T-999.1-06 | T (Tampering) | orgId route param | accept | Existing Nest Param decorator returns string; downstream Prisma uses parametrised query. No regex validation needed. |
</threat_model>
