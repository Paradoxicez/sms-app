# Phase 999.1 Plan 02: Portal Shell Summary

Two portals, two sidebars, one root redirector — the routing + navigation backbone of the SaaS split. Super admins land on `/admin` with a 6-item platform sidebar; tenant users land on `/app/dashboard` with a sidebar filtered by their `Member.role` and the org's feature flags.

## What Was Built

**Nav primitives (Task 1)**

- `NavShell` — shared presentational component extracted verbatim from `sidebar-nav.tsx`. Preserves desktop `<aside className="hidden md:flex md:w-[240px]...">`, mobile 56px top bar + Sheet, green active-state border (`border-l-[3px] border-primary bg-primary/10 text-primary`), user dropdown with destructive Sign Out, NotificationBell in both portals.
- `NavRow` — single `<Link>` row. Uses `pathname === href || pathname.startsWith(href + "/")` by default, opt-in `exactMatch` for Dashboard which would otherwise match every `/app/*` route.
- `NavGroup` — caption + optional leading separator. `firstGroup` suppresses the leading `<Separator />` so "Overview" doesn't get a stray divider above it.
- `PlatformNav` — exactly 6 items in order: Organizations, Packages, Cluster Nodes, SRS Engine, Platform Audit, Users. Header badge = "Platform".
- `TenantNav` — 4 groups (Overview, Cameras, Organization, Developer). Filter composes (role matrix per D-11) ∩ (feature gate per D-13/D-14). Header badge = truncated active org name (16 chars, `title` attr holds full name).
- `useCurrentRole` — resolves `User.role` from the session; for tenant users with an active org, fetches `/api/organizations/:orgId/members/me` to obtain `Member.role` and the org name from `organization.list()`.

**Portal wiring (Task 2)**

- `/admin/layout.tsx` — swapped `SidebarNav` → `PlatformNav`; non-admin redirect now `/app/dashboard` (was `/dashboard`). Active-org bootstrap preserved for admins.
- `/app/layout.tsx` — new. Guards: no session → `/sign-in`; role=admin → `/admin`; role=user with no active org → list orgs, auto-set first or show toast + sign-out on zero orgs. Three explicit guard branches before TenantNav renders (loading, pending memberRole, missing activeOrgId) — no flash of undefined nav state.
- `/app/page.tsx` — client redirect to `/app/dashboard` (individual page migrations arrive in Plan 03).
- `/page.tsx` — root redirector. Reads session, branches: unauth → `/sign-in`, admin → `/admin`, user → `/app`.
- `/(auth)/sign-in/page.tsx` — non-admin path changed from `router.push("/")` to `router.push("/app/dashboard")` per D-21.
- `(dashboard)/` — deleted (both `layout.tsx` and `page.tsx`). Only two files existed; nothing else to preserve.
