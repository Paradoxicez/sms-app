## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Tenant pages linking to `/admin/*` | Org Admin clicks tenant Project/Camera row → admin layout → role guard → bounce to `/app/dashboard`. Breadcrumbs in detail pages did the same. | blocking | Before merging changes to `apps/web/src/components/pages/tenant-*.tsx` or `apps/web/src/app/admin/*/[id]/page.tsx`, grep the diff for `href="/admin` and `router.push('/admin` — switch to `/app/*` unless the route is truly super-admin-only. |
| Layout flipping `loading=false` before role redirect completes | AdminLayout/AppLayout previously set loading off in `finally` regardless of branch. Children rendered during the `router.push` window → hook tree mismatch → "Rendered more hooks" crash. | blocking | In any layout with role/auth redirects, only set `loading=false` on the happy-path branch. Return early on redirect without toggling loading so the skeleton covers the transition. |
| Assuming SRS loaded our config without verification | Image default CMD is `./objs/srs -c conf/docker.conf`. Our bind-mount `conf/srs.conf` was ignored silently → no `http_hooks`, wrong HLS timings, hours wasted chasing "callbacks not firing". | advisory | After any SRS config change: `docker compose exec srs ./objs/srs -t -c conf/srs.conf` + check `cat /proc/1/cmdline` before debugging behavior. |
