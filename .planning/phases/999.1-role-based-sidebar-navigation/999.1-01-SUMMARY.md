## Key Decisions

1. **Separate MembersController over per-method guard override** — Class-level `@UseGuards(OrgAdminGuard)` on UsersController cannot be cleanly relaxed for `members/me` (Org Admin is a *stricter* guard than AuthGuard, but the paths already diverge — `/users/*` vs `/members/me`). A second controller with `@UseGuards(AuthGuard)` gives each path its correct trust boundary with zero per-method override gymnastics.

2. **Super admin bypass first in guard** — OrgAdminGuard checks `session.user.role === 'admin'` *before* querying Prisma so existing super-admin tests and flows remain unchanged and we save a DB roundtrip in the hot path.

3. **404 (not 403) for non-member on members/me** — Per threat T-999.1-04, we accept disclosure of "this org exists" because orgIds are already UUIDs in URLs and the endpoint is reachable only with a valid session.

4. **Self-contained vi.mock of auth.config** — Tests inject sessions via an `x-test-session` header JSON-blob. Avoids standing up Better Auth in test, keeps tests within the existing `tests/users/` pattern where controllers/services are constructed directly with PrismaService.
