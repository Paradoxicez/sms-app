# Phase 999.1 Plan 03: Tenant Pages Migration Summary

All 12 tenant-scope pages now reachable at `/app/*` via shared components; `/admin/dashboard` + `/admin/audit-log` replaced with platform placeholder stubs; the 10 orphan `/admin/<tenant-route>` pages rewritten as client-side redirects. One authoritative implementation per page; zero double-edits; zero import cycles between portals.

## What Was Built

**Shared tenant components (Task 1a).** `components/pages/tenant-<flattened-route>-page.tsx` for every live orphan: dashboard, audit-log, cameras, projects, stream-profiles, policies, recordings, map, developer, developer-api-keys, developer-webhooks, developer-docs. Each file is the original admin page body copied verbatim, with:
- `"use client"` directive preserved (or added for developer/page.tsx and developer/docs/page.tsx which didn't have it — harmless; those bodies are still server-safe but consistent with the rest).
- Default export renamed to `Tenant<Pascal>Page` (e.g. `TenantDashboardPage`, `TenantDeveloperApiKeysPage`).
- Original relative `./components/*` imports rewritten to absolute `@/app/admin/<route>/components/*` so the co-located dialogs/badges resolve from their still-existing admin paths. No component file was moved.

**`/app/*` re-exports (Task 1b).** Twelve `/app/*/page.tsx` files authored exactly once. Seven simple re-exports (`export { default } from "@/components/pages/tenant-<route>-page";`): dashboard, cameras, projects, stream-profiles, policies, developer, developer/docs. Five feature-gated wrappers (recordings→`recordings`, map→`map`, audit-log→`auditLog`, developer/api-keys→`apiKeys`, developer/webhooks→`webhooks`) using the plan's verbatim template: `useCurrentRole` → `useFeatures(activeOrgId)` → `Skeleton | FeatureGateEmptyState | <SourcePage/>`.

**FeatureGateEmptyState + /app/sites (Task 1c).** `components/feature-gate-empty-state.tsx` renders the UI-SPEC copy verbatim, with a tiny singular/plural switch (`endsWith('s') ? 'are' : 'is'`). `/app/sites/page.tsx` is a standalone placeholder (building icon + "ships in a follow-up phase") so the new TenantNav "Sites" link doesn't 404.

**Platform stubs (Task 2).** `/admin/dashboard/page.tsx` and `/admin/audit-log/page.tsx` bodies replaced with UI-SPEC-compliant placeholders (`LayoutDashboard` / `FileText` icon + future-release copy). `/admin/page.tsx` was already redirecting to `/admin/organizations` before this plan — no change needed.

**Redirect stubs (Task 3).** Ten `/admin/<route>/page.tsx` files (cameras, projects, stream-profiles, policies, recordings, map, developer, developer/api-keys, developer/webhooks, developer/docs) rewritten as identical client-side redirects using `useRouter().replace()` plus a `Skeleton` loading placeholder. No `/app/*` file was touched; no shared component was touched; no import cycle possible because `grep -rn "@/app/admin/" apps/web/src/app/app/` returns zero matches.

## Deviations from Plan

### Rule 3 — tsc not available until `pnpm install`

- **Found during:** Task 1a verify step.
- **Issue:** The worktree had no `node_modules` installed, so `pnpm exec tsc` errored with "Command 'tsc' not found".
- **Fix:** Ran `pnpm install --frozen-lockfile` once. No other install invoked for the rest of the plan.
- **Files:** none (tooling only).

### Rule 1 — relative `./components/*` imports broke after extraction

- **Found during:** Task 1a verify (tsc run).
- **Issue:** Copying `/admin/cameras/page.tsx` verbatim into `/components/pages/tenant-cameras-page.tsx` left `import ... from './components/camera-status-badge'` pointing into `/components/pages/components/...` which does not exist. Same issue for policies and stream-profiles.
- **Fix:** Rewrote the three affected relative imports to absolute `@/app/admin/<route>/components/*` paths. The co-located admin components are not moved — only the page body is duplicated, so those paths remain valid and will continue to resolve after Task 3 rewrites only the `/admin/<route>/page.tsx` file (NOT the components subdirectory).
- **Files modified:** `components/pages/tenant-cameras-page.tsx`, `tenant-policies-page.tsx`, `tenant-stream-profiles-page.tsx`.
- **Commit:** folded into Task 1a (c41f990).

### Scope-bounded omissions (unchanged from plan)

- Pre-existing TypeScript errors in `apps/web/src/app/admin/{organizations,packages}/components/*-dialog.tsx` (zod v4 / `@hookform/resolvers` overload mismatch) and test-side `JSX`/`toBeInTheDocument` missing-namespace errors are unrelated to this plan. Per SCOPE BOUNDARY they are not touched here. Nothing this plan added introduces new type errors.
