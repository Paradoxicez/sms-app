## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 — Blocking issue] Missing vitest infrastructure in web app**
- **Found during:** Task 1 (plan assumed `apps/web/vitest.config.ts` existed)
- **Issue:** `pnpm --filter @sms-platform/web test` fails — no vitest, no jsdom, no RTL installed, no config file. Verify commands in every subsequent task would fail for the wrong reason.
- **Fix:** Installed vitest 3 + @vitest/ui 3 + @vitejs/plugin-react 4 + jsdom 25 + @testing-library/{react,jest-dom,user-event}. Created `apps/web/vitest.config.ts` (jsdom env, `@/` alias, setup file). Added `"test": "vitest"` script.
- **Files modified:** apps/web/package.json, apps/web/vitest.config.ts, apps/web/src/test-utils/setup.ts, pnpm-lock.yaml
- **Commit:** 7a9e4b6

**2. [Rule 3 — Blocking issue] Vitest 4 + Vite 5 transitive resolution error**
- **Found during:** Task 2 initial test run
- **Issue:** pnpm's workspace Vite is pinned at 5.4.21 (by Next.js indirect dep). Vitest 4 requires Vite 6+ and crashes with `Package subpath './module-runner' is not defined`.
- **Fix:** Downgraded web app to vitest 3 (matches VALIDATION.md's stated framework "Vitest 3") + @vitejs/plugin-react 4.
- **Files modified:** apps/web/package.json, pnpm-lock.yaml
- **Commit:** c84e5eb

**3. [Rule 3 — Blocking issue] jsdom 29 ESM/CJS require regression**
- **Found during:** Task 2 second test run
- **Issue:** `ERR_REQUIRE_ESM` in `html-encoding-sniffer` due to transitive CJS/ESM mismatch under pnpm + @noble/hashes 2.
- **Fix:** Pinned jsdom to 25 (stable under CJS consumers).
- **Commit:** c84e5eb

Followed the existing pattern to stay consistent. Both new API tests fail for the intended reason (missing method / missing guard module), which is the Nyquist anchor requirement.
