## Typography

Reuses project's existing Tailwind type scale. Phase uses 4 roles:

| Role | Size | Weight | Line Height | Token |
|------|------|--------|-------------|-------|
| Body | 14px | 500 (medium) | 1.5 | `text-sm font-medium` ‚Äî nav labels, table cells, form labels |
| Label | 12px | 500 (medium) | 1.25 | `text-xs font-medium` ‚Äî section headers ("Monitoring"), nav group captions, badge copy |
| Heading | 20px | 600 (semibold) | 1.3 | `text-xl font-semibold` ‚Äî dialog titles (`CardTitle` pattern), page H1 |
| Display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` ‚Äî brand mark "SMS" in sidebar header + sign-in |

Weights used: **500 medium** and **600 semibold** for interface; **700 bold** reserved for brand mark only. Max distinct sizes in one view: 4.

## Color

60/30/10 allocation confirmed from existing theme tokens:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `hsl(0 0% 100%)` | Page canvas, main content area, dialog surface |
| Secondary (30%) | `--secondary` / `--sidebar` `hsl(142 30% 97%)` | Sidebar surface (both portals), card backgrounds, subtle container fills |
| Accent (10%) | `--primary` `hsl(142 71% 45%)` | Reserved-for list below |
| Destructive | `--destructive` `hsl(0 84% 60%)` | Destructive actions only (see list) |
| Muted text | `--muted-foreground` `hsl(215 16% 47%)` | Inactive nav label, user-row email, empty-state body copy |
| Border | `--border` `hsl(220 13% 91%)` | Dividers, input outlines, table rows |

**Accent reserved exclusively for:**
1. Primary CTA buttons (`<Button>` default ‚Äî "Create User", "Save", "Sign In")
2. Active nav row ‚Äî left border (3px) + background tint (`bg-primary/10`) + text (`text-primary`)
3. Brand mark "SMS" in sidebar header and sign-in card
4. Focus rings (`--ring`)
5. Portal badge tint background (`bg-primary/10` with `text-primary`) ‚Äî see portal differentiation

**Accent is NOT used for:** secondary/ghost buttons, table row hover, icon defaults, link text in body content, success indicators, informational badges.

**Destructive reserved exclusively for:**
1. Sign Out menu item text
2. Delete/Remove user confirmation button
3. Form validation error text (`text-destructive`)
4. "You do not have access" redirect banner

### Portal differentiation (critical for this phase)

Both portals share the identical green palette ‚Äî differentiation is done by **a single badge pill** in the sidebar header, NOT by color palette swap:

| Portal | Brand row content | Badge copy | Badge style |
|--------|-------------------|------------|-------------|
| `/admin` (platform) | "SMS" (text-primary) + badge | `Platform` | `bg-primary/10 text-primary rounded px-2 py-0.5 text-xs font-medium` |
| `/app` (tenant) | "SMS" (text-primary) + badge | `{Org Name}` (truncate at 16 chars, tooltip on hover for full name) | same style |

Rationale: user always sees which portal they are in at a glance via the header badge. No palette divergence ‚Äî this is a single design system with two surfaces.

## Layout Contract

### Platform portal `/admin`
- Reuse existing `SidebarNav` layout skeleton ‚Äî rename internally to `PlatformNav`
- Desktop: fixed 240px left rail (`md:w-[240px]`), white content canvas with `md:p-8 p-4 pt-18 md:pt-8`
- Mobile: 56px top bar with hamburger `Sheet` overlay (existing pattern)
- Header badge: `Platform`
- Sections: single flat list (no group captions). 6 items only.

### Tenant portal `/app` (NEW)
- New component `TenantNav` ‚Äî identical shell to `PlatformNav`
- Desktop: 240px left rail, same surface color (`bg-secondary`)
- Sections: 4 group captions ("Overview", "Cameras", "Organization", "Developer") rendered with `Separator` + caption above each group (existing "Monitoring" caption pattern)
- Header badge: active org name
- Feature-gated items: rendered only if `useFeatures(orgId).{feature} === true`. Gated items disappear entirely (no greyed state, no upsell badge) per D-13.
- Role-gated items: rendered only if role matrix (D-11) permits. Same disappear behavior.

### Loading / redirect UX
- Both layouts show existing `Skeleton` placeholder during `authClient.getSession()` resolution (reuse pattern from `admin/layout.tsx`)
- Wrong-portal redirect: show centered banner copy (see copywriting) for ~300ms before `router.push`, NOT a blank flash

### User management pages (both portals)

- Shell: page `H1` (20px semibold) + primary CTA button top-right + `Table` component
- Table columns (platform Users): Email, Name, Role (Badge), Orgs (count, tooltip lists), Last sign-in, Actions (DropdownMenu)
- Table columns (tenant Team): Email, Name, Role (Badge), Added, Actions (DropdownMenu)
- Create dialog: `Dialog` component, max-width 440px, `space-y-4` stacked fields, footer right-aligned (Cancel ghost + Create primary)
- Role badges use `Badge` variant mapping:
  - `admin` ‚Üí default (primary tint) ‚Äî **only place accent color appears in badges**
  - `operator` ‚Üí `variant="secondary"`
  - `developer` ‚Üí `variant="secondary"`
  - `viewer` ‚Üí `variant="outline"`

### Empty states (page-level)
- Centered block, `mt-12`, icon (48px, `text-muted-foreground`), heading (20px semibold), body (14px `text-muted-foreground`), next-step button if applicable
- Use `UsersRound` icon for empty Team, `Lock` icon for role-gated, `Sparkles` icon for feature-disabled

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Sidebar link active state | `pathname.startsWith(item.href)` ‚Äî preserve existing rule. Exception: `/app/dashboard` uses exact match to avoid lighting up for sub-routes. |
| Sidebar hover (inactive row) | `hover:bg-muted hover:text-foreground` ‚Äî existing |
| Sign-in redirect | `User.role === 'admin'` ‚Üí `/admin`; else ‚Üí `/app/dashboard`. Replace current `/admin` vs `/` branching. |
| Wrong-portal hit | Layout redirects via `router.replace` (not push ‚Äî no history entry). Banner visible during redirect. |
| Missing active org (user portal) | Auto-`setActive(firstOrgId)` mirror of admin pattern. If user has zero orgs ‚Üí redirect `/sign-in` with error toast *"Your account has no organization. Contact your administrator."* |
| Role-gated direct URL | Page component renders role-check empty state (not 404, not redirect) ‚Äî gives user context that page exists but isn't theirs |
| Feature-disabled direct URL | Page component renders feature-disabled empty state (matches Phase 01 UAT-8 intent) |
| Create user success | Close dialog, toast *"User created. They can sign in now."* (green/default toast), refresh table |
| Mobile sidebar | Existing `Sheet` pattern ‚Äî 240px width, slides from left |
