## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`) | Gap **between status pills** in the Status column (D-12); gap between pill icon and pill text; gap between row-menu item icon and label |
| sm | 8px (`gap-2`) | Gap between toolbar elements (counter chip ↔ first action, action ↔ action); gap between ID chip and copy button in sheet header |
| md | 12px (`gap-3`) / 16px (`p-4`) | Bulk toolbar vertical padding (`py-2` = 8, `px-3` = 12 horizontal); sheet header padding (`p-4`) |
| lg | 24px (`mb-6`) | Gap between bulk toolbar and DataTable first row (use `mb-3` = 12px sticky-gap if bar is sticky, see animation spec) |
| xl | 32px | Not used in this phase |

**Bulk toolbar inner layout (D-04):**
- Container: `h-10` (40px tall), `px-3`, flex row, `gap-2`
- Counter chip → actions: `gap-2` (8px)
- Actions → clear×: `ml-auto` plus implicit `gap-2`

**Status column cell layout (D-12):**
- Flex row, `gap-1` (4px between pills)
- Vertical alignment: `items-center`
- Column width: 120px minimum (was 72px — expanding to accommodate up to 3 pills comfortably; LIVE + REC + MAINT is not a valid combination per D-14, but LIVE+REC is common and needs ~96px of content)

**Exceptions:**
- Status pill internal padding: `px-1.5 py-0.5` (6px × 2px) — matches the existing `camera-popup.tsx:201` pulsing LIVE pill precedent. 2px vertical is a deliberate exception to the 4-point scale because pill height is height-constrained by pairing with icon-only row cells; the 6px horizontal keeps `px` a multiple of 2 and visually balanced with 11–12px pill text.
- Checkbox column width: 40px (matches `recordings-columns.tsx:63`).

## Typography

| Role | Size | Weight | Line Height | Font-family |
|------|------|--------|-------------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.5 (20px) | sans (Inter) |
| Label / UI chrome | 12px (`text-xs`) | 500 (`font-medium`) | 1.33 (16px) | sans |
| Pill text (LIVE/REC/MAINT/OFFLINE) | 10px (`text-[10px]`) | 700 (`font-bold`) | 1 | sans, `uppercase tracking-wide` |
| Monospace chip (Camera ID) | 12px (`text-xs`) | 500 | 1.33 | `font-mono` |
| Sheet title (existing) | 18px (`text-lg`) | 600 (`font-semibold`) | 1.33 | sans |

**Weights used:** exactly two — 400 (body) and 500 (labels) for normal UI; pill-bold is a *display* exception, not a general weight. Font-semibold 600 is reused from existing sheet header and is out of this phase's net-new surface.

**Pill typography rationale:** 10px uppercase bold matches the reference LIVE pill in `camera-popup.tsx:201` (`text-[10px] font-bold uppercase tracking-wide`). Keeping parity across surfaces (table + map popup) is a design-system win; do not redesign pill type for the table.

**ID chip truncation (D-18):** display form `1dfaadd7…402a8103` (8 prefix + `…` ellipsis character + 8 suffix). Use the unicode ellipsis `U+2026`, not three dots. `font-mono text-xs` on a `bg-muted` pill.

## Interaction Contracts

### 1. Selection model (D-05)

- First column is `id: "select"` checkbox (copy the exact pattern from `recordings-columns.tsx:42–64`)
- Header checkbox: tri-state (checked / indeterminate / unchecked) reflecting current-page selection
- Row checkbox click wrapped in `<div onClick={(e) => e.stopPropagation()}>` to prevent `onRowClick` from firing when selecting
- `aria-label` on header: `Select all cameras on this page`
- `aria-label` on row: `Select camera {camera.name}`
- `table.getSelectedRowModel().rows.map(r => r.original)` passed through `onRowSelectionChange` prop already exposed by the DataTable primitive — no primitive changes needed
- Selection scope: current page only. Changing page clears selection (per-CONTEXT `<deferred>` — cross-page selection not in scope)

### 2. Bulk toolbar positioning (D-04)

- Rendered by `tenant-cameras-page.tsx` ABOVE the `<CamerasDataTable>` wrapper, not inside the DataTable
- Parent tracks `selectedRows` state via `onRowSelectionChange` prop (same pattern as `recordings-data-table.tsx:62,503`)
- Container: `<div class="sticky top-0 z-20 bg-background/95 backdrop-blur supports-[backdrop-filter]:bg-background/60 border-b">` wrapping the flex row
- Slide-in: `data-[state=open]:animate-in data-[state=open]:slide-in-from-top-2 data-[state=open]:fade-in-0` at 200ms ease-out; slide-out at 150ms ease-in
- When selection drops to 0, bar unmounts immediately (don't animate out — avoids Linear/Gmail jank of a half-faded empty bar)
- Sticky z-index: `z-20` — must be above table header (`z-10` default) but below any Sheet/Dialog portal (z-50)

### 3. Bulk button visibility rules (D-03, D-06b)

Given selected set S:

- `Start Stream`: always visible when `S.length > 0`. No state check — bulk acts as "start all". API rejects already-online cameras; the fan-out reports the per-row reject via D-06a.
- `Start Recording`: always visible. Same semantics.
- `Maintenance`: visible when `S.some(c => !c.maintenanceMode)`. Opens `MaintenanceReasonDialog` scoped to the NOT-in-maintenance subset. If the subset is empty, the button is not shown (only Exit is).
- `Exit Maintenance`: visible when `S.some(c => c.maintenanceMode)`. Runs immediately on the in-maintenance subset (no dialog).
- `Delete`: always visible. Opens AlertDialog confirm.
- When a mixed-state selection shows BOTH `Maintenance` and `Exit Maintenance`, they sit side-by-side. The copy disambiguates (Enter vs Exit); no visual separator between them.

### 4. Bulk fan-out semantics (D-02, D-06a)

- Implementation: `p-limit`-style concurrency **5** (planner may adjust within 3–10; suggested start is 5 per CONTEXT)
- Each request hits the existing per-camera endpoint (`POST /api/cameras/:id/start-stream`, `POST /api/cameras/:id/start-recording`, `POST /api/cameras/:id/maintenance`, `DELETE /api/cameras/:id`)
- Use `Promise.allSettled`, NOT `Promise.all` — one camera's failure must not abort the batch
- During fan-out:
  - Toolbar action buttons all become `disabled`
  - Counter chip swaps to `Processing… ({N})` with spinner (see §Empty/loading/error above)
  - Per-row visual: selected rows get `opacity-60 pointer-events-none` until their promise settles
  - Per-row "in-flight" overlay: small `Loader2 size-3 animate-spin` in the Status column (replaces pills temporarily, not additive) while that row's promise is pending; pills restore from new row state once settled
- On completion:
  - Summary toast fires (see Copywriting §Toasts)
  - Failed rows retain red `AlertTriangle size-3.5` error badge at end of Status pill row; hover tooltip reveals the API error reason
  - `rowSelection` is REDUCED to only the failed row IDs (instant retry loop)
  - If all succeeded, `rowSelection = {}` and toolbar unmounts

### 5. Delete confirm (D-06b)

Single click on the destructive confirm button — NO type-to-confirm. Matches `recordings-data-table.tsx:563–577`. Rationale: recordings-delete uses the same one-click pattern for 10s-of-files destructive actions; camera delete is equally destructive but is NOT more frequent, and pattern parity > novel friction.

### 6. Row menu — asymmetric Maintenance (D-07)

- Label toggles: `camera.maintenanceMode === false` → "Maintenance" (opens reason dialog); `true` → "Exit Maintenance" (runs directly)
- Icon stays `Wrench` in both states. Do NOT add a slash/check variant — lucide doesn't have `WrenchOff`, and adding an overlay glyph introduces a custom-icon maintenance burden. The label change is the affordance.
- Row menu item for Exit Maintenance does NOT show a confirm dialog. Success → toast `Exited maintenance mode`. Failure → existing error toast string preserved.

### 7. Copy actions (D-09, D-10, D-11)

- Both use `navigator.clipboard.writeText()` wrapped in try/catch (exact pattern from `push-url-section.tsx:49–56`)
- On success: Sonner toast from the Copy Table above
- On failure: `toast.error("Couldn't copy to clipboard")` — this matches existing `push-url-section.tsx:55` pattern (`"Couldn't copy — select the URL and copy manually."` is a Push-URL-specific variant; for simple ID/cURL we stick to the generic string since there's no visible text to manually select)

### 8. ID chip (D-17, D-18)

- Implemented as a `<button type="button">` (not a `<span>`) for native keyboard + a11y
- Styled with `variant="ghost"` overridden to look like a chip: `font-mono text-xs h-6 px-2 bg-muted hover:bg-muted/80 rounded-md`
- On click or Enter/Space: copy full UUID; fire toast
- Tooltip: full UUID (all 36 chars), default delay 500ms
- Placement: directly after the breadcrumb line in the sheet header block, separated by a 4px top margin
- The dedicated copy icon-button beside the chip is redundant-but-discoverable — keep it (per CONTEXT `<specifics>` — "pragmatic copy affordance")

### 9. Start Stream / Start Record pill-buttons (D-19, D-20, D-21)

Container (within the View Stream sheet tabs-row right-hand cluster):
- `<div class="flex items-center gap-2 min-w-[340px] justify-end">` — reserves enough horizontal space for both buttons at maximum active width so toggling doesn't reflow the tab row (340px = ~160px × 2 + 20px gap)
- Each button width behaves:
  - Idle: `w-9` (36px, matches current `size="icon-sm"` equivalent → bumping from 28px `icon-sm` to 36px `icon` because active state needs room)
  - Active: `w-[160px]` fixed
- CSS: explicit `width` transition `transition-[width,background-color] duration-150 ease-out`; background transition duration matches (150ms)
- Active-state label appears with `opacity-0 w-0` → `opacity-100 w-auto` staged transition so the text doesn't stretch during the width animation: phase 1 (0–100ms) width grows, phase 2 (100–200ms) label fades in

Pulse animations on active states:
- Uses `motion-safe:animate-pulse` (Tailwind's default `animate-pulse` = 2s cubic-bezier(0.4, 0, 0.6, 1) opacity 1 ↔ 0.5)
- `motion-reduce:animate-none` explicitly set; respects `prefers-reduced-motion` per D-15
- For REC button: the pulsing element is the `size-2 rounded-full bg-red-500` dot, NOT the whole button. Whole-button pulse is visual noise.
- For Stop Stream (red active): the `Radio` icon pulses, not the button background

### 10. Status pill pulse (D-15)

- LIVE + REC pills: `motion-safe:animate-pulse motion-reduce:animate-none`
- MAINT + OFFLINE pills: never animate
- Reconnecting variant (amber outline LIVE): stronger pulse — use custom keyframe OR increase contrast range. Use `motion-safe:animate-pulse` but add a class `[animation-duration:1s]` to double the frequency vs default 2s. Single-purpose CSS class defined inline via Tailwind arbitrary property — no new CSS file needed.

Suggested location: `apps/web/src/app/admin/cameras/components/maintenance-reason-dialog.tsx`. The existing `AlertDialog` block in `cameras-data-table.tsx:280–320` should be replaced by (or refactored into) this new component so that single-row and bulk flows share one dialog with one set of strings.

2. **`Terminal` vs other icon for "Copy cURL example".** CONTEXT didn't specify. Choosing `Terminal` from lucide-react — it communicates "CLI/developer" better than `Code` (which is already taken by Embed Code).

3. **REC button "near-black" shade resolution.** Claude's-Discretion in CONTEXT called this out. Choosing `zinc-900` (`hsl(240 4% 10%)` approx, or `#18181b`) over pure black or `neutral-950`. Rationale: zinc-900 matches Tailwind's `zinc` family which the codebase already uses implicitly via neutral base color; pure black is too harsh next to the green-themed UI; `neutral-950` is the same hue as black and flattens. In dark mode, downshift to `zinc-800` so the REC pill stays readable against the already-dark table background.

4. **Error badge clearing rule.** Claude's-Discretion in CONTEXT offered two options. Chosen: errors clear on next bulk action against the same row; the Clear × button ONLY resets selection (NOT errors). Rationale: user explicitly retries failed rows via the auto-reduced selection, and the pattern is self-healing. An unused Clear × that ALSO clears errors makes the button more destructive than it reads.

5. **Tooltip delay.** Not specified in CONTEXT. Defaulting to shadcn's `TooltipProvider delayDuration={500}` — this is the existing pattern across `cameras-columns.tsx` (no explicit delay) so it inherits base-ui's default (500ms). Apply the same via the shared `TooltipProvider` wrapping the status column.

6. **Bulk toolbar mobile behaviour.** CONTEXT `<deferred>` explicitly pushes mobile polish out. On `< md` breakpoint (`640px`), the toolbar is allowed to wrap via `flex-wrap`. No overflow menu in this phase. This is a DESKTOP-FIRST contract.

7. **Selected-row background.** Inherited from existing DataTable primitive via `data-state=selected` CSS — no override in this phase.

8. **Sheet header ID chip line spacing.** Line 3 sits `mt-1` (4px) below line 2. Sheet header total padding stays `p-4` (existing).

9. **Pill border-radius.** Claude's-Discretion in CONTEXT. Chosen: `rounded-md` (`--radius-md` = 8px equivalent, `calc(0.625 * 0.8) = 0.5rem`). Matches existing `Badge` component's `rounded-4xl` is too round for a 10px-text pill; `rounded-md` reads as a tag/chip. Overriding the default `Badge` radius is acceptable here because the pills are a distinct semantic surface (status signaling, not a tag cluster).

10. **Pill height.** `h-5` (20px) — matches `Badge` primitive default height. Keeps the Status column row-height stable vs. current 3-dot layout (which is ~16px tall). Net vertical drift per row: +4px.
