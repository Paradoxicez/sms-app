## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Auth API | Login form submits credentials to better-auth backend |
| Session cookie | remember me controls cookie max-age (30 days vs browser session) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Spoofing | Session fixation via remember me | accept | better-auth generates new session token on each signIn.email call (built-in). No additional mitigation needed. Verified in better-auth source. |
| T-09-05 | Information Disclosure | Error message leakage | mitigate | Use generic "Invalid email or password. Please try again." message. Do NOT expose whether email exists. This is already the pattern in the current sign-in page and is preserved. |
| T-09-06 | Tampering | Session duration config | accept | 30-day session is an intentional design decision (D-12). updateAge of 24h ensures session token refreshes daily. Acceptable risk for SaaS dashboard. |
</threat_model>
