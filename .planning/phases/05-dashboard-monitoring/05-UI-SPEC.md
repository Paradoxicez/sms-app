## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, stat card internal padding |
| md | 16px | Default element spacing, card body padding |
| lg | 24px | Section padding, gap between stat cards row and charts row |
| xl | 32px | Layout gaps between major dashboard sections |
| 2xl | 48px | Page top/bottom padding |
| 3xl | 64px | Not used this phase |

Exceptions:
- Map container height: 400px (desktop), 300px (mobile) -- fixed height, not spacing token
- Notification dropdown max-height: 384px (scrollable)
- Stat card minimum width: 200px (grid auto-fill)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Table cells, log lines, audit entries, descriptions |
| Label | 12px | 400 (regular) | 1.4 | Stat card labels, chart axis labels, badge text, muted captions |
| Heading | 20px | 600 (semibold) | 1.2 | Page titles ("Dashboard", "Map View", "Audit Log") |
| Display | 28px | 600 (semibold) | 1.1 | Stat card values (camera count, viewer count, bandwidth number) |

Weights limited to 2: 400 (regular) for Body and Label roles, 600 (semibold) for Heading and Display roles. Hierarchy is conveyed through the 4 distinct sizes (12/14/20/28) rather than additional weight variation.

Note: Existing codebase uses `text-sm` (14px), `text-xs` (12px), `text-xl` (20px) Tailwind classes. Map these directly.

## Color

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` | Page background, main content area |
| Secondary (30%) | `--card` / `--secondary` | `hsl(0 0% 100%)` / `hsl(142 30% 97%)` | Stat cards, chart containers, sidebar, notification dropdown |
| Accent (10%) | `--primary` | `hsl(142 71% 45%)` | See reserved-for list below |
| Destructive | `--destructive` | `hsl(0 84% 60%)` | Offline badges, error alerts, destructive actions |

**Accent (--primary green) reserved for:**
- Active sidebar nav items (existing pattern)
- "Online" status badges and map markers
- Primary CTA buttons (e.g., "Export Log", time range toggle active state)
- Chart-1 line/area color (bandwidth chart primary line)
- Notification bell badge (unread count dot)
- Ring/focus states on interactive elements

**Chart colors (already defined in globals.css):**

## Component Inventory

### A. Dashboard Page (`/admin/dashboard`)

**Stat Cards Row** (top of page, responsive grid)
- Layout: CSS grid `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` with `gap-4` (16px)
- Each card: shadcn `<Card>` with `<CardHeader>` (label) + `<CardContent>` (display number + trend indicator)
- Cards for org admin/operator:
  1. Cameras Online -- display count / total, green badge
  2. Cameras Offline -- display count, red badge if > 0
  3. Total Viewers -- live viewer count
  4. Bandwidth -- current aggregate bandwidth (formatted: MB/s or GB/s)
- Additional cards for super admin only:
  5. CPU Usage -- percentage from SRS summaries
  6. Memory Usage -- percentage from SRS summaries
  7. Storage Used -- percentage with progress bar
  8. SRS Uptime -- formatted duration

**Charts Section** (middle, 2-column on desktop)
- Layout: `grid-cols-1 lg:grid-cols-2` with `gap-4`
- Chart 1: Bandwidth Over Time -- Area chart, shadcn chart component
  - X-axis: time labels, Y-axis: bandwidth (auto-scale MB/GB)
  - Time range toggle: `24h | 7d | 30d` using shadcn `<Tabs>` inline
- Chart 2: API Usage Over Time -- Area chart
  - X-axis: time labels, Y-axis: request count
  - Same time range toggle pattern

**Camera Status List** (bottom)
- shadcn `<Table>` with columns: Name, Status (Badge), Viewers, Bandwidth, Uptime, Actions
- Status column uses colored `<Badge>` per status color mapping above
- Real-time updates via Socket.IO (existing `use-camera-status` pattern)
- Clicking camera name navigates to camera detail page
- Sort by status (offline first) by default

### B. Map View Page (`/admin/map`)

- Full-width Leaflet map below page heading
- Map container: `rounded-lg border` with fixed height (400px desktop, 300px mobile)
- Tile provider: OpenStreetMap default (`https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`)
- Camera markers: Colored circle markers per status color mapping
- Cluster markers: Numbered circle with count, neutral color (`--muted`), expand on click/zoom
- Click marker popup: shadcn-styled card-like popup containing:
  - Camera name (heading, 14px semibold)
  - Status badge
  - Viewer count
  - Mini HLS preview (160x90px, 16:9 aspect ratio) using hls.js
  - "View Details" link to camera detail page
- Map auto-fits to bounds of all cameras on load
- Gated by `FeatureKey.MAP` -- show "Map feature not available" empty state when disabled

### C. Audit Log Page (`/admin/audit-log`)

- Page heading "Audit Log" with filter bar below
- Filter bar: Row with `<Select>` for actor, `<Select>` for action type, date range picker (two `<Input type="date">`), "Apply" button
- shadcn `<Table>` with columns: Timestamp, Actor, Action, Resource, IP Address, Details
- Timestamp: formatted as `YYYY-MM-DD HH:mm:ss` in local timezone
- Actor: user name with email in muted text below
- Action: colored badge (create=green, update=blue, delete=red)
- Details: truncated JSON, click to expand in `<Dialog>` or `<Sheet>`
- Pagination: "Load more" button at bottom (cursor-based) or standard page navigation
- Gated by `FeatureKey.AUDIT_LOG` -- show empty state when disabled

### D. Notification Bell (Header Component)

- Bell icon (`<Bell>`) in top-right header area (next to user avatar area)
- Unread count badge: small circle (18px diameter) with count, positioned top-right of bell icon, `bg-primary text-primary-foreground`, hide when count = 0
- Click opens `<Popover>` dropdown (320px wide, max-height 384px with `<ScrollArea>`)
- Each notification item: icon + title + description + timestamp + read/unread indicator
  - Unread: `bg-primary/5` background tint
  - Read: transparent background
- "Mark all as read" link at top of dropdown
- "View all" link at bottom navigating to full notifications page (optional, can be deferred)
- Notification types with icons:
  - Camera online: `<Camera>` icon, green
  - Camera offline: `<Camera>` icon, red
  - Camera degraded: `<Camera>` icon, amber
  - System alert: `<Server>` icon, red

### E. SRS Live Logs (`/admin/stream-engine` -- new tab)

- Added as new `<Tabs>` tab "Live Logs" on existing Stream Engine settings page
- Super admin only (hide tab for other roles)
- Log viewer container: `bg-muted rounded-lg p-4 font-mono text-xs` with fixed height (500px) and `<ScrollArea>`
- Auto-scroll to bottom on new entries, pause auto-scroll when user scrolls up
- Level filter: inline `<Tabs>` with `All | Info | Warn | Error`
- Log line format: `[timestamp] [LEVEL] message` with level-based coloring:
  - Info: `text-foreground`
  - Warn: `text-chart-4`
  - Error: `text-chart-5`
- Connection status indicator: small dot + text ("Connected" / "Disconnected") at top-right of log container

### F. Camera Detail Page Redesign (`/admin/cameras/[id]`)

- Reorganize into tabbed layout using shadcn `<Tabs>`:
  - **Overview tab**: Status card (large status badge + uptime + viewers), live HLS preview player (16:9 aspect, max-width 640px), stream info (codec, resolution, FPS, bitrate)
  - **Settings tab**: Existing camera edit form
  - **Activity tab**: Recent audit log entries for this camera (reuse audit log table, filtered)
- Quick action buttons row: Start/Stop stream, Copy HLS URL, View Embed Code
- Breadcrumb: Projects > [Project] > [Site] > [Camera Name] (using existing `<Breadcrumb>` component)

## Interaction Patterns

### Real-Time Updates
- **Camera status**: Socket.IO via existing `StatusGateway`, events `camera:status` and `camera:viewers` in org-scoped rooms `org:{orgId}` -- stat cards and table update instantly
- **Charts/aggregates**: HTTP polling every 30 seconds via `setInterval` + `useSWR` or `useQuery` with `refetchInterval: 30000`
- **Notifications**: Socket.IO new event `notification:new` in user-scoped room `user:{userId}` -- increment badge count, prepend to dropdown list
- **SRS logs**: Socket.IO namespace or event `srs:log` in admin room -- append to log viewer, respect level filter

### Loading States
- Stat cards: `<Skeleton>` placeholder matching card dimensions (4 skeleton cards in grid)
- Charts: `<Skeleton>` rectangle matching chart container dimensions
- Table: `<Skeleton>` rows (5 rows of skeleton lines)
- Map: Skeleton rectangle with muted background, "Loading map..." text centered
- Notification dropdown: `<Skeleton>` list items (3 items)

### Responsive Breakpoints
- **Mobile (<768px)**: Single column stat cards, single column charts, table horizontally scrollable, map 300px height, notification popover full-width
- **Tablet (768-1024px)**: 2-column stat cards, single column charts, table visible
- **Desktop (>1024px)**: 4-column stat cards, 2-column charts, full table, map 400px height

## Accessibility Notes

- All chart data must have `aria-label` describing the data trend
- Map markers must be keyboard-navigable (Leaflet default supports this)
- Notification bell must announce unread count via `aria-label="N unread notifications"`
- Status badges use both color AND text label (never color-only indication)
- Log viewer must support keyboard scroll and have `role="log"` with `aria-live="polite"`
- Time range toggles must be keyboard-accessible tab groups
- Audit log table rows expandable via Enter key
