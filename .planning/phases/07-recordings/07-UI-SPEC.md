## Color

| Role | CSS Variable | Light Value | Usage |
|------|-------------|-------------|-------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` | Page background, main content area |
| Secondary (30%) | `--card` / `--secondary` | `hsl(0 0% 100%)` / `hsl(142 30% 97%)` | Recording cards, settings panels, timeline background |
| Accent (10%) | `--primary` | `hsl(142 71% 45%)` | See reserved-for list below |

**Accent (--primary green) reserved for:**
- Active "Recording" status indicator (pulsing red dot uses `--chart-5`, NOT accent)
- "Start Recording" CTA button
- Timeline segments where recordings exist (filled bars)
- Storage usage progress bar fill (green when under 80%)
- Active sidebar nav items (existing pattern)
- Ring/focus states on interactive elements

**Recording-specific color mapping:**
| State | Color | CSS Variable / Class | Usage |
|-------|-------|---------------------|-------|
| Recording active | Red | `--chart-5` / `bg-chart-5` | Pulsing red dot next to camera name while recording |
| Has recordings | Green | `--chart-1` / `bg-chart-1` | Filled timeline segments |
| No recordings | Muted | `--muted` / `bg-muted` | Empty timeline segments |
| Storage OK (<80%) | Green | `--chart-1` | Progress bar fill |
| Storage warning (80-90%) | Amber | `--chart-4` | Progress bar fill |
| Storage critical (>90%) | Red | `--chart-5` | Progress bar fill |

## Component Inventory

### A. Recordings Tab on Camera Detail (`/admin/cameras/[id]` -- new tab)

Added as 6th tab "Recordings" alongside existing tabs: Preview, Details, Stream Profile, Activity, Policy.

**Recording Controls Row** (top of tab content)
- Layout: `flex items-center justify-between` row
- Left side: Recording status indicator
  - When not recording: `<Button variant="default">` with `<Circle>` icon -- "Start Recording"
  - When recording: `<Button variant="destructive">` with `<Square>` icon -- "Stop Recording" + pulsing red dot `<Badge>` "REC" with `animate-pulse`
  - Schedule button: `<Button variant="outline">` with `<Clock>` icon -- "Set Schedule" opens schedule dialog
- Right side: Storage usage inline indicator
  - Text: "42.3 GB / 100 GB used" in Label size (12px)
  - `<Progress>` bar (120px wide, 8px height) with color per storage threshold

**Calendar + Timeline Section** (main content area, per D-07)
- Layout: `flex gap-4` -- calendar on left, timeline on right (desktop). Stack vertically on mobile.
- Left: shadcn `<Calendar>` component (single month view)
  - Days with recordings get a green dot indicator below the date number
  - Selected date highlighted with `--primary` background
  - Default to today's date on tab open
- Right: Horizontal 24-hour timeline bar
  - Container: `rounded-lg border bg-card p-4`, full remaining width
  - Timeline bar: 48px height, `rounded-md bg-muted` background
  - Filled segments: `bg-chart-1` (green) overlaid at correct time positions
  - Hour tick marks below the bar: `text-xs text-muted-foreground` at 0, 3, 6, 9, 12, 15, 18, 21, 24
  - Click on timeline: seek to that time position in the player
  - Drag to select range: selection highlight with `bg-primary/20` overlay, drag handles at start/end
  - Selected range shown as text below timeline: "14:30 - 16:45 (2h 15m)"

**Playback Player** (below timeline)
- Reuse existing `<HlsPlayer>` component from camera detail
- Container: `rounded-lg border overflow-hidden`, max-width 800px, 16:9 aspect ratio
- Standard hls.js controls (play, pause, seek, volume, fullscreen)
- Loading state: `<Skeleton>` rectangle matching player dimensions with "Loading recording..." centered text
- No recording for selected range: empty state within player container

**Recordings List** (below player)
- shadcn `<Table>` listing recordings for the selected date
- Columns: Time Range, Duration, Size, Status, Actions
- Time Range: "14:30 - 16:45" formatted
- Duration: "2h 15m" formatted
- Size: "1.2 GB" formatted
- Status: `<Badge>` -- "Complete" (green), "Recording" (red pulse), "Processing" (amber)
- Actions: `<Button variant="ghost" size="icon" aria-label="Download recording">` for Download, `<Button variant="ghost" size="icon" aria-label="Delete recording">` for Delete (trash icon)

### B. Recording Schedule Dialog

- shadcn `<Dialog>` triggered from "Schedule" button
- Title: "Recording Schedule"
- Content:
  - Schedule type: `<Select>` with options "Daily", "Weekly", "Custom"
  - For Daily: Start time `<Input type="time">` + End time `<Input type="time">`
  - For Weekly: Day checkboxes (Mon-Sun) + time range
  - For Custom: List of specific time windows with add/remove
  - Toggle: `<Switch>` "Enable schedule" with label
- Footer: "Save Schedule" primary button + "Discard" secondary button
- Active schedules listed below form as removable chips/badges

### C. Retention Settings (within camera Settings tab or as sub-section)

- Section heading "Retention Policy" within camera detail Settings tab
- `<Select>` for retention period: "7 days", "14 days", "30 days", "60 days", "90 days", "Custom"
- Custom: `<Input type="number">` with "days" suffix label
- Muted helper text: "Leave empty to use organization default (currently 30 days)"
- "Save Retention Policy" button, form state tracked for unsaved changes

### D. Storage Quota Card (Organization Settings or Dashboard integration)

- shadcn `<Card>` showing org-level storage overview
- Layout: stat card pattern matching Phase 5 dashboard cards
- Content:
  - Display size (28px semibold): "42.3 GB / 100 GB"
  - `<Progress>` bar full width, color-coded by threshold
  - Label below: "42% used" in muted text
  - If over 80%: amber warning text "Storage usage is high. Consider adjusting retention policies."
  - If over 90%: red alert text "Storage nearly full. New recordings may be blocked."

### E. Recordings Admin Page (`/admin/recordings` -- optional list view)

- Sidebar nav entry under "Monitoring" group: `Film` icon, "Recordings", `/admin/recordings`
- Page heading: "Recordings"
- Filter bar: `<Select>` for camera, date range picker (two date inputs), `<Select>` for status
- shadcn `<Table>` with columns: Camera, Date, Time Range, Duration, Size, Status
- Camera column: camera name as link to camera detail Recordings tab
- Bulk actions: checkbox column + "Delete Selected Recordings" destructive button in toolbar

## Interaction Patterns

### Recording Controls
- **Start Recording**: POST to backend, button transitions to loading state (spinner), then to "Stop Recording" with red pulsing "REC" badge on success. Toast confirmation: "Recording started."
- **Stop Recording**: `<AlertDialog>` confirmation (non-destructive, footage is saved), then POST to backend, transitions back to "Start Recording" button. Toast: "Recording stopped."
- **Schedule toggle**: `<Dialog>` with form, save via POST, schedules shown as `<Badge>` chips below controls

### Timeline Interaction
- **Date selection**: Click calendar date, timeline and recordings list update for that date via API fetch
- **Timeline click**: Seek player to clicked time position within the selected recording
- **Timeline drag**: Select time range, update player source to play that range
- **Loading**: Timeline shows `<Skeleton>` bar while fetching segment data for the selected date

### Real-Time Updates
- **Recording status**: Socket.IO event `recording:status` in org-scoped room -- updates REC badge and recording status in real-time
- **Storage usage**: HTTP polling every 60 seconds for storage stats, or update on recording stop
- **Recordings list**: Refresh on recording stop event

### Loading States
- Calendar: `<Skeleton>` grid matching calendar dimensions
- Timeline bar: `<Skeleton>` rectangle (full width, 48px height)
- Player: `<Skeleton>` rectangle (16:9 aspect ratio) with "Loading recording..." centered
- Recordings table: `<Skeleton>` rows (3 rows)
- Storage card: `<Skeleton>` matching card dimensions

### Responsive Breakpoints
- **Mobile (<768px)**: Calendar full-width above timeline (stacked), player full-width, table horizontally scrollable
- **Tablet (768-1024px)**: Calendar and timeline side-by-side, player full-width
- **Desktop (>1024px)**: Calendar (280px) + timeline (remaining width) side-by-side, player max-width 800px

## Accessibility Notes

- Recording status badge uses both color AND text ("REC" label, not just red dot)
- Timeline segments have `aria-label` describing time range and duration
- Timeline is keyboard-navigable: arrow keys move selection, Enter confirms
- Calendar follows shadcn accessibility defaults (keyboard navigation, ARIA labels)
- Storage progress bar has `aria-label` with percentage and absolute values
- Delete confirmation dialogs are focus-trapped with keyboard-accessible actions
- Color-coded storage thresholds accompanied by text warnings (never color-only)
- Player controls follow hls.js default accessibility patterns
