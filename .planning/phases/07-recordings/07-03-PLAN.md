**1. use-recordings.ts:**
Custom hooks for recording API interactions. Follow the existing polling hook pattern from Phase 5 (useRef for interval cleanup):

```typescript
// useRecordingStatus(cameraId) -- fetches recording state
// useRecordingTimeline(cameraId, date) -- fetches 24-hour timeline data
// useRecordingCalendar(cameraId, year, month) -- fetches days with recordings
// useRecordingsList(cameraId, date) -- fetches recordings list for date
// useStorageQuota() -- fetches org storage usage
// Recording API mutation helpers:
//   startRecording(cameraId), stopRecording(cameraId)
//   deleteRecording(id)
```

Each hook uses `apiFetch` for API calls. Polling hooks (status, storage) use 60s interval per UI-SPEC. Timeline/list/calendar hooks refetch on date change, not polling.
