<objective>
Add the test layer that locks Plan 01 + Plan 02's behavior. This plan implements Mitigation 4 (D-14) — the hybrid (mock + real Redis) test strategy that Phase 21 was missing. Phase 21's tests all mocked the queue, which is precisely why the active-job collision was discovered only at manual UAT (`21-VALIDATION.md` § "Manual UAT — 2026-04-25"). This plan ensures regression is impossible: the BKR06 11-PATCH scenario becomes a runnable test.

Three unit tests + one integration test:

1. **`profile-restart-active-job-pubsub.test.ts` (mock-based, fast)** — exercises Plan 01's publisher branch: when `getJob` returns a job whose `isActive()` resolves to true, the service publishes to `camera:{id}:restart` with the correct JSON payload AND does NOT call `queue.add` for that camera. When `isActive()` resolves to false (or the job is null), it falls through to remove-then-add (regression check on the existing path).

2. **`stream-processor-restart-signal.test.ts` (mock-based)** — exercises Plan 02's signal-handler path with `ioredis-mock`: feed a published message into the subscriber, assert `gracefulRestart` was called once. Then send TWO rapid messages, assert `gracefulRestart` was still called only once because Mitigation 3 dedup'd the second.

3. **`stream-processor-fingerprint-safety-net.test.ts` (mock-based)** — pins Plan 02 Mitigation 2: stub `systemPrisma.camera.findUnique` to return a profile that fingerprints differently from `job.data.profile`, assert `gracefulRestart` fires from the safety net path.

4. **`profile-restart-active-job.integration.test.ts` (real Redis)** — Mitigation 4. Connects to the real Redis at `localhost:6379` (the one already used by docker-compose `sms-app-redis-1`), spawns a stub long-running FFmpeg replacement (Node `setTimeout` resolved promise that mimics a 60s-running process holding a `runningProcesses` entry in FfmpegService), publishes via the production `StreamsService.enqueueProfileRestart` path, and asserts the stub is killed within 5 seconds. Reproduces the BKR06 11-PATCH scenario: 11 sequential PATCH-equivalent calls, each must produce a fresh kill.

Closes Mitigation 4 of: GAP-21-DEFECT-ACTIVE-JOB-COLLISION.

**Why a separate plan:** test scope is non-trivial (4 files, ~260 lines of test code, real-Redis dependency adds an `ioredis-mock` install + a Redis-availability skip-guard), and splitting tests from implementation lets Plans 01+02 run in parallel (Wave 1) while this plan picks up both their outputs in Wave 2.
</objective>

**Step 3 — Write `apps/api/tests/streams/stream-processor-restart-signal.test.ts`:**

Use `ioredis-mock` for in-memory pub/sub. Construct StreamProcessor with the mock Redis, push messages, assert handler behavior. Critical: `ioredis-mock` instances of the same constructor share a backing store, so `redis.duplicate()` produces a connected sibling that receives `redis.publish`.

```typescript
import { describe, it, expect, vi, beforeEach, afterEach } from 'vitest';
import RedisMock from 'ioredis-mock';
import { StreamProcessor } from '../../src/streams/processors/stream.processor';

describe('Phase 21.1 — StreamProcessor restart-signal subscriber + Mitigation 3 dedup', () => {
  let ffmpegService: any;
  let statusService: any;
  let systemPrisma: any;
  let redis: any;
  let processor: StreamProcessor;

  beforeEach(() => {
    ffmpegService = {
      startStream: vi.fn(() => new Promise(() => {})), // never resolves — simulates running FFmpeg
      gracefulRestart: vi.fn().mockResolvedValue(undefined),
      isRunning: vi.fn().mockReturnValue(true),
      stopStream: vi.fn(),
    };
    statusService = { transition: vi.fn().mockResolvedValue(undefined) };
    systemPrisma = {
      camera: {
        findUnique: vi.fn().mockResolvedValue({
          id: 'cam-1',
          streamProfile: {
            codec: 'libx264', preset: 'veryfast', resolution: '1920x1080',
            fps: 30, videoBitrate: '2000k', audioCodec: 'aac', audioBitrate: '128k',
          },
        }),
      },
    };
    redis = new RedisMock();
    processor = new StreamProcessor(ffmpegService, statusService, redis, systemPrisma);
  });

  afterEach(async () => {
    await redis.quit().catch(() => {});
  });

  it('signal on camera:{id}:restart triggers gracefulRestart(cameraId, 5000) once', async () => {
    // Run process() in the background — it will hang on ffmpegService.startStream
    const processPromise = processor.process({
      id: 'j1', name: 'start', attemptsMade: 0,
      data: {
        cameraId: 'cam-1', orgId: 'org-1',
        inputUrl: 'rtsp://a', profile: {
          codec: 'libx264', preset: 'veryfast', resolution: '1920x1080',
          fps: 30, videoBitrate: '2000k', audioCodec: 'aac', audioBitrate: '128k',
        }, needsTranscode: false,
      },
    } as any);

    // Wait briefly for subscribe to register
    await new Promise((r) => setTimeout(r, 50));

    await redis.publish('camera:cam-1:restart', JSON.stringify({
      profile: { videoBitrate: '2500k' },
      inputUrl: 'rtsp://a', needsTranscode: false,
      fingerprint: 'sha256:new',
    }));

    // Wait for handler to run
    await new Promise((r) => setTimeout(r, 50));

    expect(ffmpegService.gracefulRestart).toHaveBeenCalledWith('cam-1', 5_000);
    expect(ffmpegService.gracefulRestart).toHaveBeenCalledTimes(1);

    // Cleanup — make startStream resolve so process() returns
    // (Test framework tear-down handles dangling promises.)
  });

  it('Mitigation 3: two rapid signals while gracefulRestart is mid-flight → only ONE call', async () => {
    let resolveGraceful: (() => void) | null = null;
    ffmpegService.gracefulRestart = vi.fn(() => new Promise<void>((r) => { resolveGraceful = r; }));

    const processPromise = processor.process({
      id: 'j1', name: 'start', attemptsMade: 0,
      data: { cameraId: 'cam-1', orgId: 'org-1', inputUrl: 'rtsp://a',
        profile: { codec: 'libx264' } as any, needsTranscode: false },
    } as any);

    await new Promise((r) => setTimeout(r, 50));

    await redis.publish('camera:cam-1:restart', JSON.stringify({ profile: {} }));
    await redis.publish('camera:cam-1:restart', JSON.stringify({ profile: {} }));

    await new Promise((r) => setTimeout(r, 50));

    expect(ffmpegService.gracefulRestart).toHaveBeenCalledTimes(1);

    // Now resolve the in-flight one and confirm a third signal re-arms
    resolveGraceful!();
    await new Promise((r) => setTimeout(r, 20));

    await redis.publish('camera:cam-1:restart', JSON.stringify({ profile: {} }));
    await new Promise((r) => setTimeout(r, 50));

    expect(ffmpegService.gracefulRestart).toHaveBeenCalledTimes(2);
  });

  it('Mitigation 1: subscriber.quit and unsubscribe fire when process() resolves', async () => {
    // Make startStream resolve immediately so process() returns
    ffmpegService.startStream = vi.fn().mockResolvedValue(undefined);
    // Spy on duplicate's unsubscribe/quit by intercepting RedisMock.duplicate
    const origDuplicate = redis.duplicate.bind(redis);
    let capturedSub: any = null;
```
