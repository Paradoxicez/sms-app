## Spacing Scale

Inherited from Phase 1/2 (8-point scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, status indicator margins |
| sm | 8px | Compact element spacing, table cell padding, badge gaps |
| md | 16px | Default element spacing, card internal padding, form field gaps |
| lg | 24px | Section padding, form group gaps, dialog section spacing |
| xl | 32px | Layout gaps between cards, policy card grid gap |
| 2xl | 48px | Major section breaks, page top/bottom padding |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Embed page video player: 100vw x 100vh fullscreen, no spacing tokens applied
- Code snippet blocks in embed dialog: 12px internal padding (monospace content)
- Touch targets for copy-to-clipboard buttons: minimum 44x44px

## Copywriting Contract

### Primary CTAs

| Context | Label | Icon |
|---------|-------|------|
| Policy list page | "Create Policy" | Plus |
| Policy form save | "Save Policy" | none |
| Camera detail embed trigger | `</>` (icon-only button with tooltip "Embed Code") | Code2 |
| Embed dialog copy button | "Copy" / "Copied!" | Copy / Check |
| Session list (if admin view exists) | No CTA -- read-only list | none |

### Empty States

| Context | Heading | Body |
|---------|---------|------|
| Policy list (no policies) | No playback policies | The system default policy is active. Create a policy to customize TTL, viewer limits, or domain restrictions for specific projects, sites, or cameras. |
| Camera policy tab (no override) | Using inherited policy | This camera uses the {level}-level policy "{policy_name}". Create a camera-specific policy to override. |
| Camera sessions (no sessions) | No active sessions | Playback sessions will appear here when developers create sessions via the API. |
| Embed dialog (stream not active) | Stream not active | Start the camera stream before generating embed code. The embed URL requires an active stream. |

Empty state layout: Centered vertically. Lucide icon (48px, muted color) above heading. Heading in 20px/600. Body in 14px/400 muted text. Primary CTA button below body with 16px gap.

### Error States

| Context | Copy |
|---------|------|
| Session creation rejected (viewer limit) | Viewer limit reached ({current}/{max}). No more viewers can connect to this camera. Wait for existing sessions to expire or increase the limit in the playback policy. |
| Session creation rejected (domain) | Playback not allowed from this domain. Add "{domain}" to the domain allowlist in the playback policy. |
| Session expired | This playback session has expired. Request a new session from the API to continue viewing. |
| Policy save failed | Could not save policy. Please check the form values and try again. |
| Embed page - session not found | Session not found. The playback session may have expired or been revoked. |
| Embed page - stream offline | Stream is currently offline. The camera may be disconnected or the stream has not been started. |
| Embed page - loading failed | Unable to load video stream. Please check your connection and try again. |
| HLS key fetch rejected | Access denied. Your playback session is no longer valid. |

Error text: 12px, destructive color, displayed below the relevant element with 4px gap. Embed page errors use 16px white text centered on black background.

### Destructive Confirmations

| Action | Dialog Title | Dialog Body | Confirm Button |
|--------|-------------|-------------|----------------|
| Delete Policy | Delete Policy | This will remove the policy "{policy_name}". Entities using this policy will fall back to the next level in the inheritance chain. | "Delete Policy" (destructive) |

All destructive dialogs use AlertDialog. Confirm button uses destructive variant. Cancel on the left, destructive confirm on the right.

## Interaction Patterns

### Policy CRUD Flow
1. Navigate to Policies page via sidebar
2. Click "Create Policy" to open policy form page
3. Select policy Level via RadioGroup (System/Project/Site/Camera)
4. If Project/Site/Camera selected: entity selector dropdown appears filtered by org
5. Fill in policy fields -- empty fields show "(inherited)" placeholder text in muted color
6. Resolved Policy Preview card updates live as fields change, showing:
   - Each field value + source level badge (e.g., "TTL: 7200s" with "System" badge if inherited)
   - Fields set at current level highlighted with accent left-border
7. Click "Save Policy" -- toast notification "Policy saved"
8. Edit: same form, pre-populated with existing values
9. Delete: AlertDialog confirmation, then redirect to policy list

### Domain Allowlist Editor
1. Text input with "Add Domain" button (or Enter key to add)
2. Input placeholder: "example.com or *.example.com"
3. Added domains rendered as removable tags below input (Badge variant with X button)
4. Wildcard patterns (starting with `*.`) shown with a subtle asterisk icon
5. Empty list shows hint text: "Empty = allow all domains"
6. Validation: reject obviously invalid patterns (no spaces, must have at least one dot)

### Embed Code Dialog (D-19)
1. Triggered by `</>` icon button on camera detail page header
2. Dialog opens with Tabs: "iframe" | "hls.js" | "React"
3. Each tab shows a CodeBlock with pre-filled snippet:
   - **iframe tab:** `<iframe src="https://{domain}/embed/{session}" ...>`
   - **hls.js tab:** JavaScript snippet with hls.js initialization + URL
   - **React tab:** React component snippet with hls.js hook
4. All snippets use the camera's actual stream URL pattern (placeholder for session token)
5. Note below code: "Replace {token} with a valid playback token from the API" in Label size (12px), muted color
6. "Copy" button (top-right of code block) changes to "Copied!" with Check icon for 2 seconds
7. Code block: dark background (hsl(0 0% 9%)), monospace font, 12px size, horizontal scroll for long lines

### Embed Page (/embed/{session})
1. Page loads with black background, centered Loader2 spinner (white, 32px)
2. Fetch session info from `GET /api/playback/sessions/{id}` to get hlsUrl
3. If session not found or expired: show error text centered on black background
4. If session valid: initialize hls.js with hlsUrl, render fullscreen video
5. Video player: native controls, autoplay muted, no custom UI chrome
6. On HLS error (network/media): show error overlay text on video
7. No branding, no navigation, no header/footer -- pure video embed

### Policy Tab on Camera Detail
1. New tab "Policy" added to camera detail Tabs component (after Logs tab)
2. Tab content shows:
   - **Resolved Policy Card:** Read-only display of all resolved policy fields
   - Each field shows value + colored badge indicating source level
   - Example: "TTL: 7200s" [System badge] | "Max Viewers: 25" [Site badge] | "Domains: *.example.com" [Camera badge]
3. Below resolved policy: "Edit Camera Policy" link/button (if camera-level policy exists) or "Create Camera Policy" button (if not)
4. Below policy: SessionsTable showing active sessions for this camera
   - Columns: Session ID (truncated), Created, Expires, Viewer Count, Status
   - Expired sessions shown in muted text
   - No action buttons (sessions are API-managed, not dashboard-managed)

### Sessions Table
1. Read-only table showing recent playback sessions per camera
2. Columns: Session ID (first 8 chars + "..."), Created At, Expires At, Status (active/expired badge)
3. Expired sessions: entire row in muted text color
4. Active sessions: normal text, green "Active" badge
5. Expiry column shows relative time ("in 45 min" or "23 min ago") with tooltip for exact datetime
6. Auto-refresh via polling every 30 seconds (no WebSocket needed for sessions)
7. Pagination: show last 20 sessions with "Load More" button
