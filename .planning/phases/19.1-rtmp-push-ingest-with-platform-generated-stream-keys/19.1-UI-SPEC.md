## Spacing Scale

Declared values (must be multiples of 4). The codebase uses Tailwind's default `0.25rem` scale, which already satisfies the 8-point system.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon row gaps in status stack; tooltip trigger padding |
| sm | 8px (`gap-2`, `p-2`, `space-y-2`) | Form label → input gap; stacked action icon spacing |
| md | 16px (`gap-4`, `p-4`, `space-y-4`) | Default form-row spacing; card content padding; sheet header padding |
| lg | 24px (`gap-6`, `p-6`, `space-y-6`) | Section padding inside push-url-section; banner inner padding |
| xl | 32px (`gap-8`, `p-8`) | Layout gaps (rarely used in this phase — only empty-state centering) |
| 2xl | 48px (`gap-12`) | Major section breaks (not used in dialog surfaces) |
| 3xl | 64px (`gap-16`) | Page-level spacing (not used — all surfaces are dialog/sheet) |

**Exceptions for this phase:**
- Push URL input height stays at `h-9` (36px) — matches existing Input default; not a 4-multiple but consistent with shadcn default. No override.
- Inline action icon buttons use `p-0.5` (2px) padding inside `rounded-md` — consistent with `CodecStatusCell` FailedCell retry button (tight click target; the button itself is ≥24px via icon + padding).
- Segmented control (toggle-group) height matches existing `select` trigger at 36px — inherits from shadcn toggle-group defaults.

## Typography

All sizes resolve against Tailwind's default scale. Four sizes total, two weights max. Aligned with existing camera-form-dialog and view-stream-sheet patterns.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / form input | 14px (`text-sm`) | 400 (regular) | 1.5 (`leading-normal`) |
| Mono (URL, stream key, codec) | 12px (`text-xs font-mono`) | 400 (regular) | 1.5 (`leading-normal`) |
| Label / section title | 14px (`text-sm`) | 600 (semibold, via `font-medium` which resolves 500 in Inter — existing convention) | 1.5 |
| Dialog title | 18px (`text-lg`) | 600 (`font-semibold`) | 1.3 (`leading-tight`) |

**Weights declared:** 400 (regular) for body/mono, 500–600 (medium/semibold) for labels and titles. The codebase uses Tailwind's `font-medium` (500) as the "semibold-ish" weight — we follow the existing convention rather than introducing a 3rd weight.

**Line-height rationale:** body at 1.5 per GSD default; heading at 1.3 because dialog titles are short single-line strings where 1.2 would be too tight with descender clipping in Inter.

## Color

The project's green theme is locked in `apps/web/src/app/globals.css:50-84` (light) and `:86-118` (dark). All values below are HSL tokens from that file; do not hardcode hex.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` = `hsl(0 0% 100%)` (light) / `hsl(0 0% 9%)` (dark) | Dialog body, sheet body, form backgrounds |
| Secondary (30%) | `--card` = same as background; `--secondary` = `hsl(142 30% 97%)` (light, subtle green tint) | Card surfaces in view-stream-sheet (Camera Info, Push URL); sidebar |
| Accent (10%) | `--primary` = `hsl(142 71% 45%)` | Primary CTAs only (see reserved list) |
| Destructive | `--destructive` = `hsl(0 84% 60%)` | Rotate key confirmation CTA; delete row; validation error text |
| Amber warning | `text-amber-600 dark:text-amber-500` | Probe failed indicator, maintenance wrench icon, duplicate icon, **codec-mismatch banner** (D-16). Reuses existing palette from `CodecStatusCell:110` and `cameras-columns:104`. |

**Accent reserved for (explicit list — do not expand):**
1. "Save Camera" / "Done" / "Confirm Import" / "Rotate key" primary buttons (ONE per surface).
2. "Created" state checkmark icon color (optional; muted-green alternative OK).
3. Focus rings (`--ring` token, same green) on form inputs and interactive elements — inherited from shadcn defaults, do not override.
4. Active state of the `[Pull | Push]` segmented control (toggle-group `data-state="on"` — uses `--primary` via existing toggle-group styles).
5. Success checkmark in bulk-import result panel (existing pattern — lucide `Check` with `text-primary`).

**Not accent** (use muted, secondary, or neutral):
- Secondary buttons ("Cancel", "Back", "Dismiss", "Done" in Created state if the primary is elsewhere on the surface — e.g. Copy is primary, Done is outline).
- Stream-URL mono text (use `text-muted-foreground`).
- Inline hint / helper text (`text-muted-foreground`).
- "Setup guide →" link (`text-muted-foreground` + underline on hover; NOT primary — docs links are intentionally quieter to keep the Copy CTA as the visual anchor).

**Codec-mismatch banner color contract:**
- Surface: `bg-amber-50 dark:bg-amber-950/30` + `border-amber-200 dark:border-amber-900`.
- Icon + heading text: `text-amber-700 dark:text-amber-400`.
- Body copy: `text-foreground` (readable, not amber).
- Accept button: primary (default green). Dismiss button: outline (muted).
- Matches the existing amber-on-surface pattern used in `codec-status-cell.tsx` but scaled up to a full banner (8–12px vertical padding, 16px horizontal, rounded-md).

## Layout & Interaction Patterns

This section is load-bearing for the planner and executor — it codifies the visual patterns that the six net-new composites must follow.

### Add Camera dialog — Pull vs Push mode switch (D-08)

- Segmented control rendered at the **top** of the form, **above the Name field** (before the Name input).
- `<IngestModeToggle>` uses shadcn `toggle-group` with `type="single"` and `value`/`onValueChange` props. Defaults to `pull`.
- Selection persists in local component state only (no URL query param, no localStorage).
- Switching from Pull → Push: immediately hides the `streamUrl` input, replaces with the push-mode hint block. `streamUrl` state is cleared.
- Switching from Push → Pull: restores empty `streamUrl` input.
- Switching modes clears any existing validation errors.
- The segmented control is NOT rendered in **edit mode** — editing a camera cannot change its ingestMode (backend constraint per CONTEXT ambiguity note; planner confirms).

### Post-save reveal pattern (D-09) — shared `<CreatedUrlReveal>` component

**Surface swap contract** (same dialog, content swap — no route change, no new dialog):

1. Form `onSubmit` succeeds → backend returns `{ id, streamUrl, ingestMode: "push" }`.
2. Component state flips: `phase: "form" → "reveal"`.
3. Dialog title changes, form content unmounts, `<CreatedUrlReveal>` mounts in its place.
4. URL shown in a disabled Input with `font-mono text-xs`, full-width, with trailing `Copy` button (same Input + Button pattern used in `code-block.tsx` and `embed-code-dialog.tsx` — planner references those).
5. `Done` button (outline variant) closes the dialog and triggers `onSuccess()` parent callback to refresh the cameras table.
6. Closing via Escape or backdrop click is **allowed** — the URL is persistently available in the camera detail sheet, so there is no data-loss risk.

**Shared between create-push (D-09) and rotate-key (D-20).** The component accepts props `{ url: string; title: string; helperText?: string; onClose: () => void }`. Title and helperText differ but the reveal body is identical.

### Push URL section in ViewStreamSheet (D-07, D-19)

- Renders as a full-width `Card` in the **Preview tab**, positioned **above** the Camera Info card (higher priority — user needs access to rotate/copy more often than they inspect codec).
- Only renders when `camera.ingestMode === "push"`. Pull cameras do not see this section at all.
- Card layout:
  - Header: `<CardTitle>Push URL</CardTitle>` + `<CardDescription>Paste this into your camera or encoder.</CardDescription>`
  - Content: URL Input (read-only, full width, `font-mono text-xs`) with trailing icon-button cluster `[Copy] [Rotate] [Setup guide →]` (Copy and Rotate are icon-sm buttons with tooltips; Setup guide is a text link).
- **Key is shown full** in this surface because the viewer is always the owning org's admin (RLS enforced). If a future read-only viewer role appears, the mask-in-non-owner-contexts rule applies — mask before render.

### Codec-mismatch banner placement (D-16)

- Renders at the **top of the Preview tab**, above the HLS player.
- Only visible when `normalizeCodecInfo(camera.codecInfo).status === "mismatch"`.
- Dismissing the banner hides it **for the current view session only** (component-local state, not persisted). Next mismatch-triggering publish will re-show it.
- Accepting calls `POST /api/cameras/:id/enable-auto-transcode` (planner defines exact endpoint) and optimistically updates `camera.codecInfo.status` to `pending` (the probe will refresh it).

### Rotate key flow (D-19, D-20)

- Entry point is an inline icon button in the Push URL section (RotateCw icon). NOT exposed in the DataTable row dropdown — per D-19 this is intentional friction.
- Click opens `<RotateKeyDialog>` (an `alert-dialog`, not a plain dialog, because the action is destructive).
- Confirm → fires `POST /api/cameras/:id/rotate-key` → on success, dialog swaps to `<CreatedUrlReveal>` state showing the new URL (same pattern as D-09).
- The parent `ViewStreamSheet` refreshes the camera row after the reveal dialog closes (via `onSuccess` callback).

### Bulk-import result panel extension (D-14)

- Existing result-panel layout stays unchanged for pull-only imports.
- When `result.pushImported > 0`, a new row appears **below** the existing counter row: `<PushUrlsDownloadButton>` with label `Download push URLs (CSV)`.
- Button click triggers client-side CSV generation from the server response (response must include `{ id, name, streamUrl, ingestMode }` for each imported row). Server does NOT return a file — client synthesizes.
- CSV format: `name,streamUrl` (two columns, push rows only, header row included).
- Button becomes disabled and label changes to `Downloaded` for 3s after click.
- Button is **not a one-time** affordance in the UI state, but the user should be warned (via the tooltip) that individual URLs can still be retrieved from each camera's detail sheet.

### Status column — no new indicator (D-25)

- The Camera table status column does NOT add a new push-vs-pull indicator.
- The existing three-icon stack (online/offline dot + recording dot + maintenance wrench per 15-CONTEXT D-16) is unchanged.
- Codec column (Phase 19 D-05) absorbs the new `mismatch` state visually — reuse the existing amber AlertTriangle icon with a different tooltip (`Codec mismatch — open camera to resolve`).
- **Do not** add a column for `ingestMode`. Per D-17, the Stream Profile column already tells the user the path; ingestMode is visible in the detail sheet's Push URL section presence.

### First-publish empty state (D-26)

- When push camera has `firstPublishAt === null`, the Push URL section renders normally, but a `<WaitingForFirstPublish>` block renders **below** the URL card (same Preview tab).
- Block uses muted-surface card styling (`bg-muted/30 border-dashed`), centered content, `Loader2` spinner icon (not animated — would draw too much attention).
- Heading `Waiting for first publish…` at `text-base font-medium`.
- Troubleshooting checklist as `<ul>` with 3 items (see copy above), `text-sm text-muted-foreground`, bullet-point style.
- Block auto-hides when `camera.status` transitions to `online` (WebSocket-driven — uses existing `useCameraStatus` hook).

## Accessibility

- All icon-only buttons (Copy, Rotate, Setup-guide in compact layouts) require `aria-label` props matching the tooltip text.
- Segmented control (toggle-group) uses Radix's built-in keyboard navigation — no custom handlers.
- The one-time reveal Input is read-only and focus-trapped inside the dialog; user can select + Ctrl+C as a fallback to the Copy button.
- Codec-mismatch banner uses `role="alert"` (per WCAG live-region guidance) — screen readers announce it on first render.
- Destructive confirmation uses shadcn `alert-dialog` which ships with proper `role="alertdialog"` semantics.
- Color-only signals (amber warning, green success) are always paired with an icon or text label — never color alone.
