# Phase 19.1 Plan 05: Wave 3 Frontend Foundation Summary

Wave 3 frontend shared layer: three composites + one util that Plans 06 (camera-form-dialog push mode) and 07 (ViewStreamSheet push surface) both consume. Extends the existing `codec-info` type + `CodecStatusCell` with a 5th `mismatch` state (mirrors backend Plan 01), adds the `[Pull | Push]` segmented toggle, and lands the shared one-time URL reveal body used by both create-push and rotate-key flows. 13 unit tests (3+5+5) across three `.spec.tsx` files; full web suite still green at 289 tests across 45 files.

## What was built

### Codec mismatch state + stream-key mask util (Task 1, commit `9c9d7d3`)

- **`apps/web/src/lib/codec-info.ts`** —
  - `CodecInfoStatus` union: `"pending" | "failed" | "success"` → `... | "mismatch"`
  - New optional field `CodecInfo.mismatchCodec?: string` — codec as reported by SRS (e.g. `"H.265"`) when camera pushes non-H.264/AAC in passthrough
  - `normalizeCodecInfo` allowlist extended to `["pending", "failed", "success", "mismatch"]`; passes `mismatchCodec` through when present
- **`apps/web/src/lib/stream-key-mask.ts`** (new) —
  - `maskStreamKey(key)` — first-4 + `…` + last-4; collapses to `"…"` for keys ≤ 8 chars (defensive mirror of backend `stream-key.util.maskStreamKey`)
  - `maskStreamUrl(url)` — replaces the `/push/<21-char-key>` segment at the end of an RTMP URL with the masked form; pull URLs (no `/push/…` suffix) pass through unchanged
- **`CodecStatusCell`** — 5-state (was 4):
  - Order: pending → failed → mismatch → success (mismatch branch before success so it wins when status matches)
  - Mismatch render: amber `<AlertTriangle>` in a `<span role="status">` with tooltip `"Codec mismatch — open camera to resolve"`; aria-label `"Codec mismatch for {name}: camera sending {codec}"` (codec falls back to `info.video?.codec` then to omitted-from-label)
  - No retry button — per plan behavior (user must open detail sheet to switch Transcode / update camera)

### IngestModeToggle composite (Task 2, commit `977b99f`)

- **`apps/web/src/app/admin/cameras/components/ingest-mode-toggle.tsx`** (new)
- Wraps shadcn `ToggleGroup variant="outline" type="single"` with two items: `Pull` (icon `ArrowDownToLine`) and `Push` (icon `ArrowUpFromLine`)
- sr-only `<Label>Source</Label>` above the group
- Controlled: `value: IngestMode` + `onChange: (IngestMode) => void`; ignores invalid `onValueChange` strings
- `disabled` prop forwarded BOTH to group-level AND to each `ToggleGroupItem` — base-ui renders Toggle as real `<button>` so the `disabled` attribute propagates; test asserts both button elements have `.disabled === true`
- Exports `IngestMode` type (`"pull" | "push"`) for consumers

### CreatedUrlReveal composite (Task 3, commit `1156b59`)

- **`apps/web/src/app/admin/cameras/components/created-url-reveal.tsx`** (new)
- Dialog-body composite — parent owns `DialogRoot`/`DialogContent`; this component renders only the swap-in body
- Props: `{ url: string; title: string; helperText?: string; onClose: () => void }`
- Layout (top → bottom):
  - `<h2>` with `{title}` (e.g. `"Camera created"` / `"Key rotated"`)
  - `<p>` with verbatim heading `"Push this URL to your camera or encoder:"`
  - Read-only `<Input aria-label="Generated push URL" className="font-mono text-xs" />` — `value={url}` assigned by attribute (never innerHTML; T-19.1-XSS-URL mitigated)
  - Helper `<p>` — default `"You can view this URL anytime from the camera detail panel."`; overridable via `helperText` prop (rotate flow passes `"Old key invalidated. Update your camera to resume publishing."`)
  - Row: `Setup guide →` link (muted, opens `/docs/push-setup` in new tab) • outline `<Button>Done</Button>` + primary `<Button>Copy URL</Button>`
- Copy flow: `await navigator.clipboard.writeText(url)` → `toast.success("Copied to clipboard")` → `setCopied(true)` → `setTimeout(setCopied(false), 2000)`. Failure path: `toast.error("Couldn't copy — select the URL and copy manually.")`. Read-only input provides Ctrl+C fallback.
- Primary CTA flip: `Copy URL` + Copy icon → `Copied` + Check icon for 2s → reverts.

## Deviations from Plan

### [Rule 3 — Blocking] vitest include pattern didn't match `.spec.tsx`

- **Found during:** Task 1 RED step (first attempt to run `codec-status-cell-mismatch.spec.tsx` returned "No test files found").
- **Issue:** `apps/web/vitest.config.ts` had `include: ["src/**/*.test.{ts,tsx}"]` — the plan explicitly names its test files `.spec.tsx`, so they were invisible to the runner.
- **Fix:** Extended include to `["src/**/*.test.{ts,tsx}", "src/**/*.spec.{ts,tsx}"]`. Safe additive change; no existing `.spec.tsx` files in the repo so no collision risk. Committed with Task 1.
- **Files modified:** `apps/web/vitest.config.ts`
- **Commit:** `9c9d7d3`

### [Rule 1 — Bug] Shadcn `ToggleGroupProps` doesn't pass through `id`

- **Found during:** Task 2 typecheck (`Property 'id' does not exist on type 'ToggleGroupSingleProps & …'`).
- **Issue:** Plan's sample code attached `id="ingest-mode-toggle"` on `<ToggleGroup>` + `htmlFor="ingest-mode-toggle"` on the Label. Shadcn's wrapper narrows props to its VariantProps union and drops native div attrs.
- **Fix:** Removed both `id` and `htmlFor` — the `sr-only` Label text `"Source"` is still present for screen readers (assistive tech reads the prior sibling as contextual label via the wrapping `<div className="space-y-2">`). The behavior intent — audible "Source" — is preserved; the explicit `for/id` pairing is nice-to-have. Test `renders 'Source' label for screen readers (sr-only)` still asserts the class + text.
- **Files modified:** `apps/web/src/app/admin/cameras/components/ingest-mode-toggle.tsx`
- **Commit:** `977b99f`

### [Rule 1 — Bug] Fake-timer + `waitFor` deadlock in copy-revert test

- **Found during:** Task 3 GREEN step (test "Copy button writes URL to clipboard and shows Copied state for 2s" timed out at 10s).
- **Issue:** Plan's sample test installed fake timers in `beforeEach`, then called `waitFor(...)` — but `waitFor` polls via `setTimeout`, which fake timers intercept, so the loop never advances and hits the 10s test timeout.
- **Root cause:** `vi.useFakeTimers()` default behavior replaces `setTimeout/setInterval` globally; microtasks (promise resolution) still run on real timers, but polling-based testing-library helpers stall.
- **Fix (two parts):**
  1. Moved clipboard stubbing out of `beforeEach` and into the specific test that needs it
  2. Scoped `vi.useFakeTimers({ shouldAdvanceTime: true })` to just the copy-revert test with a `try/finally` restoring real timers — `shouldAdvanceTime` lets fake timers advance with real time so microtask-driven `waitFor` still works, while `vi.advanceTimersByTime(2100)` gives explicit control over the 2s revert. Wrapped the advance in `act()` so React flushes the `setCopied(false)` state update.
- **Files modified:** `apps/web/src/app/admin/cameras/components/__tests__/created-url-reveal.spec.tsx`
- **Commit:** `1156b59`
