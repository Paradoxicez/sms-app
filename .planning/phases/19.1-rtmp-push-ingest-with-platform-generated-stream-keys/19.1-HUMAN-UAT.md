## Tests

### 1. End-to-end push ingest — single camera happy path
expected: Register a push camera via the Add Camera dialog. Copy the generated URL. Paste into an RTMP encoder (OBS / FFmpeg). Within ~2s the camera status flips to online in the DataTable, codec column populates with H.264/AAC, the HLS player in ViewStreamSheet plays the stream from `live/{orgId}/{cameraId}/index.m3u8`.
result: [pending]

### 2. Codec-mismatch kick flow — H.265 publisher on Passthrough profile
expected: Configure an RTMP encoder to publish H.265 + AAC to a push camera with a Passthrough stream profile. The publish connects, then within ~1s SRS probe detects the mismatch, disconnects the publisher (encoder sees error), `codecInfo.status` becomes `mismatch` with `mismatchCodec='H.265'`, CodecMismatchBanner appears in ViewStreamSheet above the HLS player. Clicking "Enable auto-transcode" PATCHes `needsTranscode:true`; user retries publish; stream now works via FFmpeg transcode.
result: [pending]

### 3. Stream-key rotation — destructive UX
expected: While a push camera is actively publishing, open its detail sheet, click Rotate. Confirm. The active publisher is force-disconnected, a new URL is revealed, dialog swaps to the "Key rotated" reveal. Old URL no longer works; pasting the new URL into the encoder resumes publishing.
result: [pending]

### 4. Delete-while-publishing — kick before hard-delete (D-22)
expected: While a push camera is publishing, delete it from the DataTable. The publisher is force-disconnected and the camera row is removed. Cascade delete applies to recordings/playback sessions.
result: [pending]

### 5. Maintenance mode + push (D-23)
expected: Enable maintenance mode on a push camera. Publish via encoder. Status transitions to online (no 403), HLS is served, but notifications/webhooks are suppressed (Phase 15 gate). Exit maintenance mode resumes notifications.
result: [pending]

### 6. Double-publish on same key (D-24)
expected: Two RTMP encoders attempt to publish to the same `push/<key>` URL concurrently. SRS accepts the first, rejects the second with its native double-publisher disconnect. Platform does NOT need custom logic (D-24 relies on SRS default).
result: [pending]

### 7. Bulk-import CSV download (D-14)

result: [pending]

### 8. Created URL reveal dialog — dialog stays open (D-09)
expected: Create a push camera with name only. Submit. Dialog body swaps to "Camera created" with the generated `rtmp://` URL, Copy button, Done button. Copy flashes "Copied" for 2s. Clicking Done closes dialog and refreshes the camera list (new camera visible).
