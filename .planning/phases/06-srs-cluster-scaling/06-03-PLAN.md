Backend API endpoints (from Plan 01):
```
GET    /api/cluster/nodes          -> SrsNode[]
GET    /api/cluster/nodes/:id      -> SrsNode
POST   /api/cluster/nodes          -> SrsNode (body: { name, apiUrl, hlsUrl, hlsPort?, isLocal? })
PATCH  /api/cluster/nodes/:id      -> SrsNode
DELETE /api/cluster/nodes/:id      -> 204
POST   /api/cluster/nodes/:id/test -> { success, version?, error? }
GET    /api/cluster/nodes/:id/config -> { config, configVersion }
POST   /api/cluster/nodes/:id/reload -> 200
```

SrsNode type:
```typescript
interface SrsNode {
  id: string;
  name: string;
  role: 'ORIGIN' | 'EDGE';
  status: 'ONLINE' | 'OFFLINE' | 'DEGRADED' | 'CONNECTING';
  apiUrl: string;
  hlsUrl: string;
  hlsPort: number;
  cpu: number | null;
  memory: number | null;
  bandwidth: string; // BigInt serialized
  viewers: number;
  srsVersion: string | null;
  uptime: number | null;
  missedChecks: number;
  lastHealthAt: string | null;
  configVersion: number;
  isLocal: boolean;
  metadata: any;
  createdAt: string;
  updatedAt: string;
}
```

Socket.IO events (namespace /cluster-status):
```
node:health -> { nodeId, status, cpu, memory, bandwidth, viewers }
node:status -> { nodeId, status }
```

1. **useClusterNodes hook** (`apps/web/src/hooks/use-cluster-nodes.ts`):
- Fetch nodes from `GET /api/cluster/nodes` using apiFetch pattern
- Connect to Socket.IO namespace `/cluster-status`
- Listen for `node:health` events: update node metrics in local state
- Listen for `node:status` events: update node status in local state
- Return: `{ nodes, loading, error, refetch }`
- Derive summary stats: totalNodes, onlineNodes, totalViewers (sum of all viewers), totalBandwidth (sum of all bandwidth)
- Use useRef for Socket.IO cleanup (existing pattern from Phase 5)

2. **ClusterStats component** (`apps/web/src/app/admin/cluster/components/cluster-stats.tsx`):
- CSS grid: `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-4`
- 4 shadcn Cards per UI-SPEC:
  a. "Total Nodes" -- count of all nodes
  b. "Online Nodes" -- count of status=ONLINE nodes, green Badge
  c. "Total Edge Viewers" -- aggregate viewers across all edge nodes
  d. "Cluster Bandwidth" -- aggregate bandwidth formatted as MB/s or GB/s
- Each card: CardHeader (label, 12px text-muted-foreground) + CardContent (display number, 28px font-semibold)
- Loading state: 4 Skeleton cards matching dimensions

3. **NodeTable component** (`apps/web/src/app/admin/cluster/components/node-table.tsx`):
- shadcn Table with columns: Name, Role, URL, Status, CPU, Memory, Viewers, Bandwidth, Actions
- Role column: `<Badge variant="outline">Origin</Badge>` or `<Badge variant="secondary">Edge</Badge>`
- Status column: colored Badge per UI-SPEC color mapping:
  - ONLINE: green (`bg-chart-1 text-white`)
  - OFFLINE: red (`bg-chart-5 text-white`) using destructive variant
  - DEGRADED: amber (`bg-chart-4 text-white`)
  - CONNECTING: blue (`bg-blue-500 text-white`)
- CPU/Memory: inline Progress bar (64px wide) + percentage text, colored per threshold (0-70% green, 70-90% amber, 90-100% red)
- Viewers: integer
- Bandwidth: formatted string
- Actions: DropdownMenu with "View Details", "Reload Config", "Remove Node" (only for EDGE)
- Sort by status offline-first (matching Phase 5 camera table pattern)
- Origin row: no "Remove Node" option
- Loading state: 3 Skeleton rows

4. **AddNodeDialog component** (`apps/web/src/app/admin/cluster/components/add-node-dialog.tsx`):
- shadcn Dialog with title "Add Edge Node"
- Form fields:
  - Name: Input, placeholder "e.g., edge-node-01", required
  - API URL: Input, placeholder "e.g., http://192.168.1.100:8080", required (this is the nginx HLS URL)
  - HLS Port: Input type="number", default "8080"
- "Test Connection" button (secondary variant): POST /api/cluster/nodes/:id/test style -- but since node doesn't exist yet, call a general test endpoint or just validate URL format

  - Success: green check icon + "Connection successful" text
  - Failure: red X icon + error message
- "Add Node" button (primary): disabled until test passes
- On submit: POST /api/cluster/nodes with { name, apiUrl: same as hlsUrl for nginx edges, hlsUrl, hlsPort }
- After success: close dialog, toast "Edge node '{name}' added successfully", call refetch
- For nginx edges: apiUrl and hlsUrl are the same base URL (nginx serves both status and HLS on same port)
