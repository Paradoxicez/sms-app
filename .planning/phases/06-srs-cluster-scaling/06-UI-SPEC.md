**Accent (--primary green) reserved for:**
- Active sidebar nav items (existing pattern)
- "Online" status badges for nodes
- "Add Node" primary CTA button
- Progress bars for CPU/memory usage (below threshold)
- Ring/focus states on interactive elements

**Node status color mapping (consistent with camera status pattern from Phase 5):**

| Status | Color | Tailwind Class | Badge Variant |
|--------|-------|---------------|---------------|

| Offline | Red | `text-chart-5` / `bg-chart-5` | `destructive` |
| Degraded | Amber | `text-chart-4` / `bg-chart-4` | amber custom |

| 0-70% | `--chart-1` (green) | Healthy |
| 70-90% | `--chart-4` (amber) | Warning |
| 90-100% | `--chart-5` (red) | Critical |

## Component Inventory

### A. Cluster Nodes Page (`/admin/cluster`)

**Summary Stats Row** (top of page)
- Layout: CSS grid `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` with `gap-4` (16px)
- Each card: shadcn `<Card>` with `<CardHeader>` (label, 12px) + `<CardContent>` (display number, 28px)
- Cards:
  1. Total Nodes -- count of all registered nodes (origin + edge)
  2. Online Nodes -- count of nodes with status=online, green badge
  3. Total Edge Viewers -- aggregate viewer count across all edge nodes
  4. Cluster Bandwidth -- aggregate bandwidth across all nodes (formatted MB/s or GB/s)

**Node List Table** (main content area)
- shadcn `<Table>` with columns: Name, Role (Badge: origin/edge), URL, Status (Badge), CPU, Memory, Viewers, Bandwidth, Actions
- Role column: `<Badge variant="outline">Origin</Badge>` or `<Badge variant="secondary">Edge</Badge>`
- Status column: colored `<Badge>` per node status color mapping above
- CPU/Memory columns: inline `<Progress>` bar (64px wide) with percentage text, colored per health threshold
- Viewers: integer count
- Bandwidth: formatted string (e.g., "24.5 MB/s")
- Actions column: `<DropdownMenu>` with options:
  - "View Details" -- opens detail dialog
  - "Reload Config" -- triggers config reload on that node
  - "Remove Node" -- opens destructive confirmation dialog (only for edge nodes, origin cannot be removed)
- Origin row visually distinct: no remove option, "Origin" role badge uses outline variant
- Real-time updates via Socket.IO for status, CPU, memory, viewers, bandwidth
- Sort by status (offline first) by default, matching Phase 5 camera table pattern

**Add Node Button** (top-right, next to page heading)
- `<Button>` with `<Plus>` icon: "Add Edge Node"
- Opens `<Dialog>` with form

### B. Add Edge Node Dialog

- shadcn `<Dialog>` with `<DialogHeader>` title "Add Edge Node"
- Form fields:
  - Name: `<Input>` placeholder "e.g., edge-node-01" (required)
  - URL: `<Input>` placeholder "e.g., http://192.168.1.100:1985" (required, SRS API URL)
  - HLS Port: `<Input type="number">` default "8080" (the port where HLS is served)
- "Test Connection" button: secondary variant, triggers backend validation
  - Success: green check icon + "Connection successful" text + SRS version info
  - Failure: red X icon + "Connection failed: {error message}"
- Footer: "Cancel" (ghost) + "Add Node" (primary, disabled until test passes)
- After successful add: close dialog, toast "Edge node added successfully", refresh table

### C. Node Detail Dialog

- shadcn `<Dialog>` (wide: `max-w-2xl`) with `<DialogHeader>` title "{Node Name}"
- Content sections:
  - **Info**: Name, Role, URL, Status badge, SRS version, uptime
  - **Health Metrics**: CPU, Memory, Bandwidth as larger `<Progress>` bars with percentage labels, colored per threshold
  - **Active Streams**: count of streams being served by this node
  - **Active Viewers**: count of connected clients
  - **Config Status**: "Config in sync" or "Config outdated -- reload required"
- Footer: "Close" button + "Reload Config" button (secondary)

### D. Remove Node Confirmation

- shadcn `<AlertDialog>` for destructive action
- Title: "Remove Edge Node"
- Description: "This will remove {node name} from the cluster. Active viewers on this node will be disrupted and need to reconnect. This action cannot be undone."
- Actions: "Cancel" (outline) + "Remove Node" (destructive variant)
- After removal: close dialog, toast "Edge node removed", refresh table

### E. Stream Engine Page Update (`/admin/stream-engine`)

- Add new `<Tabs>` tab "Cluster" alongside existing Settings and Live Logs tabs
- Cluster tab content: link/redirect to `/admin/cluster` page, or embed summary view with "Manage Cluster" CTA button
- Super admin only visibility for Cluster tab

### F. Navigation Update

Add "Cluster" item to sidebar under existing "Stream Engine" section or as a sibling in the admin nav:

```
Stream Engine    /admin/stream-engine    Server icon
Cluster Nodes    /admin/cluster          Network icon
```

Use `lucide-react` icon `Network` for cluster nodes page.

## Interaction Patterns

### Real-Time Updates
- **Node health**: Socket.IO events in admin-scoped room (super admin only), event `node:health` with payload `{ nodeId, status, cpu, memory, bandwidth, viewers }` -- updates table rows and stat cards instantly
- **Node status change**: Socket.IO event `node:status` with payload `{ nodeId, status }` -- updates badge color in table
- **Summary stats**: Derived from node health events (sum viewers, sum bandwidth, count online)

### Loading States
- Stat cards: `<Skeleton>` placeholder matching card dimensions (4 skeleton cards in grid)
- Table: `<Skeleton>` rows (3 rows of skeleton lines)
- Add node dialog test connection: `<Button>` shows spinner icon while testing
- Node detail dialog: `<Skeleton>` blocks for metrics while loading

### Responsive Breakpoints
- **Mobile (<768px)**: Single column stat cards, table horizontally scrollable, dialog full-width
- **Tablet (768-1024px)**: 2-column stat cards, table visible
- **Desktop (>1024px)**: 4-column stat cards, full table, wide detail dialog

### Error Handling
- Network failure on node list fetch: inline error message with "Retry" button replacing table
- Node test connection timeout: 10-second timeout, show error message in dialog
- Failed config reload: toast with error, node remains in table unchanged

## Accessibility Notes

- Node status badges use both color AND text label (never color-only indication)
- Progress bars for CPU/memory include percentage text alongside visual indicator
- Table rows navigable via keyboard, actions accessible via dropdown menu keyboard navigation
- Remove node confirmation requires explicit button click (no keyboard shortcut for destructive action)
- "Test Connection" result announced via `aria-live="polite"` region
- Toast notifications use sonner which has built-in aria-live support
