| Property | Value |

## Color

| Role | Value | Usage in this phase |
|------|-------|---------------------|
| Dominant (60%) | `hsl(0 0% 100%)` / `var(--background)` | Page background, table background |
| Secondary (30%) | `hsl(220 14% 96%)` / `var(--muted)` | Table row hover (`bg-muted/50`), selected row (`bg-muted`), toolbar area |
| Accent (10%) | `hsl(142 71% 45%)` / `var(--primary)` | Primary CTA buttons (e.g., "Add User"), checkbox checked state, active sort arrow, focus ring, active pagination page |
| Destructive | `hsl(0 84% 60%)` / `var(--destructive)` | Delete/Revoke/Deactivate menu item text, destructive confirmation dialog buttons |

Accent reserved for:
- Primary CTA buttons in toolbar (e.g., "Add User", "Add API Key", "Add Webhook")
- Checkbox checked fill color
- Active column sort direction arrow
- Focus ring on interactive elements
- Active pagination page number

NOT used for: table borders, filter chips at rest, row action "..." button, badge backgrounds, pagination arrows.

## Table Specifications

### 1. Users Table

| Column | Width | Content | Sortable |
|--------|-------|---------|----------|
| Email | flex-1 (min 200px) | Plain text | Yes |
| Name | flex-1 (min 160px) | Plain text | Yes |
| Role | 120px | Badge (color per role) | Yes |
| Orgs | 80px | Count number (e.g., "3") | Yes |

## Interaction Contracts

All interactions inherit from Phase 8 DataTable contract. Phase 10 adds table-specific behaviors:

### Destructive Actions

| Action | Table | Confirmation Approach |
|--------|-------|-----------------------|
| Deactivate user | Users | AlertDialog: "Are you sure you want to deactivate this user? They will lose access immediately." |
| Revoke API key | API Keys | AlertDialog: "Revoke API key '{name}'? Applications using this key will stop working immediately." |
| Delete API key | API Keys | AlertDialog: "Permanently delete API key '{name}'? This action cannot be undone." |
| Delete webhook | Webhooks | AlertDialog: "Delete webhook '{name}'? You will stop receiving event notifications at this URL." |
| Delete stream profile | Stream Profiles | AlertDialog: "Delete profile '{name}'? Cameras using this profile will fall back to the default profile." |

All destructive AlertDialogs use:
- Title: Action name (e.g., "Deactivate User")
- Description: Consequence-focused copy (see table above)
- Cancel button: `variant="outline"` with text "Cancel"
- Confirm button: `variant="destructive"` with text matching action verb (e.g., "Deactivate", "Revoke", "Delete")

### Non-Destructive Actions

| Action | Table | Behavior |
|--------|-------|----------|
| View details | Users | Navigates to user detail page or opens detail dialog |
| Edit role | Users | Opens role edit dialog with Select dropdown |
| Copy key | API Keys | Copies full API key to clipboard, shows toast "API key copied to clipboard" |
| View Details | Audit Log | Opens Dialog showing full audit log entry as formatted JSON |
| Edit | Webhooks | Opens edit form (existing edit flow) |
| Toggle active/inactive | Webhooks | Immediate toggle with toast confirmation "Webhook enabled" / "Webhook disabled" |
| Test webhook | Webhooks | Sends ping to webhook URL, shows toast "Test event sent" on success or "Test failed: {error}" on failure |
| Edit | Stream Profiles | Opens edit form (existing edit flow) |
| Duplicate | Stream Profiles | Creates copy with name "{original} (copy)", opens edit form pre-filled, shows toast "Profile duplicated" |

### Audit Log Detail Dialog

| Element | Specification |
|---------|---------------|
| Trigger | "View Details" from row action menu |
| Dialog width | `max-w-[600px]` |
| Content | Scrollable JSON view of full audit entry |
| JSON display | `<pre>` block with `bg-muted rounded-md p-4 text-xs font-mono overflow-auto max-h-[400px]` |
| Close | "Close" button (`variant="outline"`) or X button in header |

## Layout Specifications

### Page Layout (consistent across all 5 pages)

```
[Page Heading]                              [Primary CTA Button]
                    16px gap
[DataTable]
  [Toolbar: Search | Faceted Filters | (DateRangePicker for Audit)]
                    16px gap
  [Table with sortable headers]
                    16px gap
  [Pagination bar]
```

- Page heading: `text-xl font-semibold` left-aligned
- Primary CTA: `Button variant="default" size="sm"` right-aligned, same row as heading
- Heading row: `flex items-center justify-between mb-4`
- DataTable occupies full content width within the page container

DateRangePicker renders as part of the DataTableToolbar, positioned after the faceted filter chips:

```
[Search (240px)]  [Action Filter]  [DateRangePicker]  ----stretches----
```

## Accessibility Contract

Inherited from Phase 8 plus:

| Requirement | Implementation |
|-------------|----------------|
| Destructive action confirmation | AlertDialog traps focus, Escape cancels, Enter on focused confirm button executes |
| Copy to clipboard | Toast announces success for screen readers via `role="status"` (Sonner default) |
| Audit detail dialog | Dialog traps focus, JSON content inside scrollable region with `tabindex="0"` for keyboard scroll |
| Toggle webhook | Toast announces new state for screen readers |
| Masked API key | Full key not exposed in DOM; copy action fetches full key from API |
