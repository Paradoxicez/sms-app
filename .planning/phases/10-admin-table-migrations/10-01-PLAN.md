2. **audit.service.ts** — Replace `findAll` method to use offset pagination with count:
   - Change return type from `{ items, nextCursor }` to `{ items: any[]; totalCount: number }`
   - Compute `skip = (query.page - 1) * query.pageSize`
   - Use `Promise.all([prisma.auditLog.findMany({ where, orderBy: { createdAt: 'desc' }, take: query.pageSize, skip }), prisma.auditLog.count({ where })])` to get items + totalCount in parallel
   - Remove all cursor-related logic (cursor, skip: 1, take + 1, nextCursor extraction)
   - Add search filter: if `query.search` is provided, add to where clause — but since audit log doesn't have a direct user relation in Prisma, search will filter on existing fields. For the tenant service, search on `resource` and `ip` fields using `contains` mode (user info is joined separately by admin service)
   - Keep the existing `where` filter building for userId, action, resource, dateFrom, dateTo

3. **admin-audit-log.service.ts** — Same offset pagination change:
   - Change return type from `{ items, nextCursor }` to `{ items: any[]; totalCount: number }`
   - Compute `skip = (query.page - 1) * query.pageSize`
   - Use `Promise.all` for parallel findMany + count
   - Remove all cursor logic
   - Keep the existing user/org name joining logic intact
   - Add search: when `query.search` is provided, after fetching userIds that match the search, add them to the where clause. Simpler approach: search in `resource` field with `contains`, and also pre-query users by name/email containing the search string, then include matching userIds in the where. Actually, the cleanest approach for admin: add an OR condition — `{ OR: [{ resource: { contains: query.search, mode: 'insensitive' } }] }` and handle user search by pre-querying users matching the search, then adding their IDs.

   Concrete implementation for admin-audit-log.service.ts search:
   ```typescript
   if (query.search) {
     const matchingUsers = await this.rawPrisma.user.findMany({
       where: {
         OR: [
           { name: { contains: query.search, mode: 'insensitive' } },
           { email: { contains: query.search, mode: 'insensitive' } },
         ],
       },
       select: { id: true },
     });
     const matchingUserIds = matchingUsers.map((u: any) => u.id);
     if (matchingUserIds.length > 0) {
       where.OR = [
         { userId: { in: matchingUserIds } },
         { resource: { contains: query.search, mode: 'insensitive' } },
       ];
     } else {
       where.resource = { contains: query.search, mode: 'insensitive' };
     }
   }
   ```

   For tenant audit.service.ts, same pattern but using the tenant prisma client.
