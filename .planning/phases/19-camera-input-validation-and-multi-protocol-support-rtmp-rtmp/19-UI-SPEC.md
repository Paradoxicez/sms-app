# Phase 19 — UI Design Contract

> Visual and interaction contract for camera input validation (Add Camera dialog + Bulk Import dialog) and the 3-state probe status display in the Camera table. Covers `/admin/cameras` table column, Add Camera form dialog, Bulk Import dialog, and post-import result toast. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Four UI touchpoints inside the cameras module:

1. **Camera table codec column** (`cameras-columns.tsx:148-172`) — replace single `—` fallback with 4-state cell: pending (spinner), failed (amber warning + retry icon), no data (`—`), success (existing `H.264` text). CONTEXT D-05..D-07.
2. **Camera table retry affordance** — inline retry icon inside failed-probe cell. Re-enqueues BullMQ `probe:{cameraId}` job. CONTEXT D-06.
3. **Bulk Import dialog** (`bulk-import-dialog.tsx`) — add third "Duplicate" row state alongside existing ✓/✗ icons; extend client-side `validateRow` to cover new allowed prefixes (`rtsp|srt|rtmp|rtmps`), non-empty host, and within-file dedup; footer counter becomes `N valid, M duplicate` when duplicates exist; success toast reports imported vs skipped. CONTEXT D-08, D-10, D-15, D-16.
4. **Add Camera dialog** (`camera-form-dialog.tsx:141-188`) — live prefix validation under Stream URL input mirroring backend zod refine (`rtsp|srt|rtmp|rtmps`); inline error message; submit button disabled while invalid. CONTEXT D-15.

No new components, no new primitives, no new dependencies. All icons already in `lucide-react`. Sonner for toasts already wired.

|-------|-------|---------------------|
| xs | 4px | Gap between prefix error icon and message text, row status icon inset inside 16px table cell, inline retry icon margin-left |

**Exceptions:**
- Codec cell status icons: fixed `size-3.5` (14px) — matches existing `cameras-columns.tsx:101` wrench icon sizing for visual parity inside table cells. Row height unchanged.
- Retry icon (`RotateCw`): `size-3.5` (14px) with `ml-1.5` (6px) from the failed label text — 6px is a 1.5× multiple of xs (4px). This matches Lucide's visual affordance for icon-as-clickable at tiny sizes without bumping row height.
- Duplicate icon (`Copy` or `Files`): `size-4` (16px) — matches the existing `Check` / `X` status icons in `bulk-import-dialog.tsx:538,545` (`h-4 w-4`) for row-level visual parity.

### Bulk Import Row Status Color Map (CONTEXT D-08)

| Row State | Icon | Color | Animation | Tooltip |
|-----------|------|-------|-----------|---------|

### Error Reason Copy Dictionary (CONTEXT "Claude's Discretion" — error tooltips)

Normalize raw ffprobe stderr into these short English phrases. Fallback to raw trimmed stderr if none match (truncate at 80 chars). Keep English-only per user preference (MEMORY.md).

| Raw stderr pattern | Normalized tooltip |
|--------------------|--------------------|
| `Connection refused` / `ECONNREFUSED` | `Connection refused` |
| `Network is unreachable` / `ENETUNREACH` | `Network unreachable` |
| `401 Unauthorized` / `authorization required` / `Authentication failed` | `Auth failed — check credentials` |
| `404 Not Found` / `Stream not found` | `Stream path not found` |
| `timed out` / `ETIMEDOUT` / `Timeout` | `Timeout — camera not responding` |
| `Invalid data found when processing input` | `Invalid stream format` |
| `Unsupported codec` / `No decoder for codec` | `Unsupported codec` |
| `SSL handshake` / `TLS error` | `TLS handshake failed` |
| `unable to resolve host` / `ENOTFOUND` / `getaddrinfo` | `Hostname not resolvable` |
| (no match) | first 80 chars of trimmed stderr |

Executor implements this mapping inside `StreamProbeProcessor` before it writes `codecInfo.error`. The UI then renders whatever string is stored — no client-side normalization.

## Interaction Patterns

### Codec Cell (cameras-columns.tsx → codec-status-cell.tsx)

- **Pending state render:** `Loader2` with `animate-spin`. Respects `prefers-reduced-motion` via `motion-safe:animate-spin motion-reduce:opacity-60` (falls back to dim-but-static on reduced-motion).
- **Failed state render:** Side-by-side `AlertTriangle` (non-interactive, icon only) + `RotateCw` (interactive `<button>` with `hover:bg-accent` and focus ring).
- **Failed — retry click:** fires `onRetry(cameraId)`. During in-flight request, swap `RotateCw` for `Loader2` spinner. Success: emit toast `Probe retry queued.` (default variant). Failure: emit toast `Couldn't retry probe. Try again in a moment.` (error variant).
- **Retry idempotency:** The backend (`probe:{cameraId}` jobId, D-04) already merges duplicate enqueues — clicking retry multiple times in rapid succession is safe; the UI does not debounce.
- **Keyboard:** retry icon is a focusable `<button>` with `aria-label="Retry probe for {camera name}"`. Tab order: natural (after whatever focusable lives before the codec column in the row). `Enter` / `Space` triggers retry. `Escape` does nothing.
- **Live update:** when the probe job completes, the camera list refetch (existing polling or Socket.IO-via-`useCameraStatus`) flips `codecInfo.status` from `pending` → `success`/`failed` and the cell re-renders. No explicit polling added for the probe state.

### Add Camera Dialog (camera-form-dialog.tsx)

- **Live prefix validation:** On every `onChange`, compute `validatePrefix(streamUrl)`. Error appears as soon as user types a non-matching prefix.
- **Debounce:** None. Regex is O(1); keystroke-level re-render has negligible cost and matches the existing `onChange` pattern for other fields (name, lat, lng).
- **Submit disabled:** existing disabled logic at `camera-form-dialog.tsx:331` extended with `|| !!streamUrlError`. User cannot submit a URL that fails the prefix check. Backend still enforces (defense in depth).
- **Duplicate detection on submit:** When server returns `P2002` translated to `DuplicateStreamUrlError` (D-11), the dialog-level error slot (`camera-form-dialog.tsx:323-325`) displays the normalized message. Input field keeps focus; user can edit and retry.
- **Pasting URL with trailing whitespace:** `streamUrl.trim()` happens on submit (`camera-form-dialog.tsx:152`). Live validation should also `.trim()` before testing prefix so pasted URLs with stray whitespace don't show false-negative error. Add `value.trim()` inside `validatePrefix` before regex.

### Bulk Import Dialog (bulk-import-dialog.tsx)

- **Row-level live validation:** `validateRow` runs on every edit (existing `handleCellEdit` at `bulk-import-dialog.tsx:294-303`). Extended to also set the `duplicate` flag based on matching URL in earlier rows of the current preview.
- **Duplicate reason:**
  - `within-file`: another row (lower index) has identical `streamUrl` after both are trimmed. Exact match, no normalization (D-09).
  - `against-db`: (future — when planner wires optional pre-check API call) server confirmed the URL already exists for this org. If server pre-check is deferred, the row still shows as valid client-side; server's P2002 skips it during import and the post-import toast reports it.
- **Tooltip copy differs by reason:**
  - within-file: `Duplicate of existing camera` (also acceptable: `Duplicate of row {N}` — planner's call; within-file tooltip prefix is "Duplicate of row {firstIndex+1}" when pointing at the duplicate index ≥ 1).
  - against-db: `Already imported in this organization`.
- **Row edits clear dup:** If user edits the `streamUrl` cell of a flagged-duplicate row to be unique, `validateRow` recomputes and the ⧉ icon flips to ✓.
- **Import button enablement:** `canImport = validCount + duplicateCount > 0 && errorCount === 0 && !!selectedSiteId`. Button tooltip (future polish, not required) could explain why it's disabled — baseline ships without tooltip since disabled state is self-evident from visible error counters.
- **Post-import toast cascade:**
  - If server returned `imported > 0 && skipped === 0`: `Imported {N} cameras successfully.`
  - If `imported > 0 && skipped > 0`: `Imported {N} cameras, skipped {M} duplicates.`
  - If `imported === 0 && skipped > 0`: `No cameras imported — all {M} rows were duplicates.` (sonner warning variant)
  - If `imported === 0 && skipped === 0`: existing error toast `Import failed. Check camera limits and try again.`
- **Row deletion (×):** unchanged. User can manually remove a duplicate-flagged row if they don't want it in the import payload (the client-side skip makes this optional, but the affordance stays for UX consistency).

## Accessibility

- **Codec cell spinner (pending):** `aria-live="polite"` on the cell so screen readers announce state transitions. `aria-label="Probing codec for camera {name}"`. Icon has `aria-hidden="true"` (label covers it).
- **Codec cell failed:** Parent `<span>` has `role="status"` + `aria-label="Probe failed for {camera name}: {reason}"`. Retry `<button>` has explicit `aria-label="Retry probe for {camera name}"`. Focus ring inherits `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` (shadcn default tied to `--ring` = green primary).
- **Codec cell success / no-data:** No new a11y requirements; inherits existing cell text semantics.
- **Add Camera inline error:** `<Input aria-invalid={!!streamUrlError} aria-describedby="cam-url-error">`. Error message element has `id="cam-url-error"` + `role="alert"`. Screen readers announce the error on the next focus.
- **Helper text when no error:** `<p id="cam-url-help" className="text-xs text-muted-foreground">`. Input's `aria-describedby` points at whichever element is currently rendered (helper vs error).
- **Bulk import status icons:** Each status cell `<td>` has `aria-label` matching the tooltip copy. Icons have `aria-hidden="true"`. Screen readers read the aria-label without duplicating icon semantics.
- **Footer counters:** visible text (no aria needed). Color-blind users are not disadvantaged because each counter has a leading icon (✓, ⧉, ⚠) — color is redundant with iconography.
- **Color contrast:**
  - Amber `AlertTriangle` on white background: `#D97706` (Tailwind `amber-600`) = 4.52:1 — passes AA for non-text UI (≥ 3:1).
  - Amber `Copy` icon in bulk-import row (same `amber-600`): passes same threshold.
  - `AlertTriangle` inside a hover-highlighted row (`bg-destructive/10` per `bulk-import-dialog.tsx:465`): the amber isn't on a destructive-tinted row (duplicates are not errors, so the row background stays default). Contrast verified at 4.52:1 on both default and hover fills.
- **Tooltip appearance:** shadcn `TooltipContent` uses `role="tooltip"` + `aria-describedby` on trigger. Escape closes. Works with both mouse and keyboard focus. 300ms open delay (shadcn default).
- **Animations:** `animate-spin` on `Loader2` respects `prefers-reduced-motion` by wrapping with `motion-safe:animate-spin`. On reduced-motion, spinner appears as a static opaque icon (keeps the state visible — never renders as blank).
