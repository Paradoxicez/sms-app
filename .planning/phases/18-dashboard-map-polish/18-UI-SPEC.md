# Phase 18 — UI Design Contract

> Visual and interaction contract for Dashboard & Map Polish. Covers tenant dashboard (`/app/dashboard`), super admin dashboard (`/admin/dashboard`), and map marker/popup (`/app/map`, `/admin/map`). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

Three distinct surfaces, one phase:

1. **Tenant dashboard** — remove `<SystemMetrics />`, add 2 stat cards (Recording, In Maintenance), replace `<CameraStatusTable />` with a new `<IssuesPanel />` (CONTEXT D-01..D-04).
2. **Super admin dashboard** — add 2 stat cards (Active Streams, Recordings Active), insert 4 new sections (Cluster Nodes, Platform Issues, Storage Forecast, Recent Audit), migrate Org Summary to DataTable (CONTEXT D-05..D-12).
3. **Map marker + popup** — teardrop SVG pin with camera icon + multi-status badges, marker clustering via `react-leaflet-cluster`, popup redesign with 16:9 preview + status overlay + action restructure (CONTEXT D-13..D-22).

No new registry installs. All primitives already present (shadcn base-nova, Recharts, react-leaflet-cluster v4.1.3, leaflet.markercluster v1.5.3 — verified in `apps/web/package.json`).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| xs | 4px | Gap between stat-card icon + value, marker badge inset, popup overlay padding |
| sm | 8px | Stat card grid gap on dense layouts, Issues panel row vertical padding, popup button gap, badge gap inside cells |
| md | 16px | Default card padding, `gap-4` between stat cards and between dashboard sections (existing `space-y-6 gap-4` pattern), popup outer padding, cluster bubble padding |
| lg | 24px | Section padding (`space-y-6` container vertical rhythm — matches `tenant-dashboard-page.tsx:77`), storage-forecast card content padding |
| xl | 32px | Reserved for page-top padding in `(app)/layout.tsx` wrapper (unchanged) |
| 2xl | 48px | Reserved — not used in this phase |
| 3xl | 64px | Reserved — not used in this phase |

**Exceptions:**
- Map teardrop pin: `28px × 36px` SVG viewBox — pin is 28 wide (multiple of 4), 36 tall (multiple of 4). Anchor at `[14, 36]` (bottom-center point).
- Status badges on marker: `8px × 8px` red dot (recording), `10px × 10px` wrench badge (maintenance) — sized by accessibility contrast need, not grid, per CONTEXT D-14.
- Popup preview: `240px × 135px` (16:9) — matches CONTEXT D-17 exactly.
- Popup width: Leaflet `maxWidth={320}` `minWidth={280}` — per CONTEXT D-22.

## Color

60/30/10 split inherited from existing CSS tokens (`globals.css:51-84`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` light / `hsl(0 0% 9%)` dark (`--background`) | Page background, card surfaces via `--card`, popup content background, DataTable rows |
| Secondary (30%) | `hsl(142 30% 97%)` light / `hsl(142 20% 15%)` dark (`--secondary`, `--sidebar`, `--accent`) | Sidebar nav (unchanged), subtle row-hover fill `bg-accent/40`, cluster bubble fill background, stat-card header subtle variant |
| Accent (10%) | `hsl(142 71% 45%)` (`--primary`, `--chart-1`) | Storage-forecast chart line, Active Streams stat icon accent, Recording status dot on marker (override — see exception below), primary CTA button background, active-state sidebar nav, focus rings |
| Destructive | `hsl(0 84% 60%)` (`--destructive`, `--chart-5`) | Offline marker pin color, cluster bubble worst-status fill (any offline child), "Investigate" row in Issues panel when severity = `offline`, Storage-forecast "days until full" warning badge when ≤ 14 days |
| Amber / degraded | `hsl(38 92% 50%)` (`--chart-4`) | Degraded / reconnecting marker pin fill, cluster bubble when worst-status = degraded, cluster / maintenance icons on marker, Issues panel row for degraded/reconnecting |
| Blue / connecting | `#3b82f6` (Tailwind `blue-500`) | Connecting marker pin fill only (matches existing `camera-marker.tsx:24`) |

**Accent reserved for:** (1) Primary CTA buttons inside popup (`View Stream`), (2) Storage-forecast Recharts line stroke via `stroke=var(--chart-1)`, (3) Focus rings on interactive elements (DataTable rows, popup buttons, Issues panel rows), (4) Active-state sidebar nav (unchanged), (5) Recording status dot on marker + popup "REC" text color, (6) Recharts default series for single-line charts. **Never** used for: generic row hover (use `--accent/40`), cluster bubble default fill (use neutral `--card` until worst-status dictates), table body backgrounds, or stat-card headers.

| Worst Child Status | Bubble Fill | Count Text |

Outer ring: `hsl(0 0% 100% / 0.7)` semi-transparent white, 3px thick (matches react-leaflet-cluster default expectation). Click cluster → zoom-in (library default).

## Interaction Patterns

### Tenant Dashboard Interactions

- **Stat cards**: pure display, no click behavior (matches existing).
- **Issues panel real-time**: Socket.IO `useCameraStatus(orgId, onStatusChange, onViewersChange)` updates the underlying camera list → panel re-derives. Polling fallback: `useDashboardIssues` refetches every 30s (Claude discretion default per D-23).
- **Issues row click**:
  - Entire row is `<button>` or `tabIndex={0}` with `role="button"`.
  - Primary action (`Investigate` / `View` / `Restart`) rendered as explicit Button on the right; cell click inside the button stops propagation.
  - Full row click → same as primary action (navigation to camera detail for offline/degraded/maintenance; restart call for recording-failed).
  - Keyboard: `Enter`/`Space` triggers primary action. `aria-label="Investigate offline camera {name}"`.
- **Empty state**: reward signal — centered `CheckCircle2` icon (32px, `text-primary`) + heading + body. Matches the "dashboard = good news" UX spirit from CONTEXT specifics.

### Super Admin Dashboard Interactions

- **Stat cards**: pure display.
- **Platform Issues**: same interaction pattern as tenant Issues panel. Action buttons navigate to relevant admin pages (`/admin/cluster`, `/admin/organizations/{id}`).
- **Cluster Nodes**: static table; rows not clickable in Phase 18 (CONTEXT deferred: per-node control is future ops feature). Node status dot + status text inline per row.
- **Storage Forecast**:
  - ToggleGroup `7d` / `30d` (single selection, default `7d`). Changes re-fetch chart data and recompute `daysUntilFull`.
  - Chart hover shows Recharts default tooltip: `{date} · {storage used GB}`.
  - No click behavior on chart itself.
- **Org Health DataTable**:
  - Default sort: usage % descending (orgs closest to plan limit first) — per D-12.
  - Row click: navigates to `/admin/organizations/{id}` (same as View action).
  - Actions dropdown (`⋮`): View / Manage — navigates to respective routes.
  - Faceted filters: Plan (select), Status (select: Healthy / Has Issues).
  - Pagination: 25 per page (matches Phase 14 DataTable default).
- **Recent Activity**: static list, not clickable per-row. Footer link `View full audit log →` navigates to `/admin/audit`.

### Map Marker Interactions

- **Click marker** → opens Leaflet Popup (existing behavior preserved).
- **Drag marker** (only when `onDragEnd` prop supplied, e.g. in `/app/map` edit mode) → calls `onDragEnd(id, name, lat, lng)` (existing).
- **Pulse animation** on `reconnecting` status preserved (existing CSS animation).
- **Recording dot** `animate-pulse` (Tailwind, 2s loop) only when `isRecording=true` AND `status=online` (no pulse on offline even if `isRecording` persists in DB).
- **Cluster click**: default react-leaflet-cluster behavior — zoom into cluster bounds.
- **Marker keyboard accessibility**: Leaflet default — marker receives focus on tab, `Enter` opens popup, `Escape` closes.

### Map Popup Interactions

- **Preview**:
  - Auto-plays muted HLS when `status === 'online'` (existing behavior preserved via `PreviewVideo` memoized component — **do not change**).
  - Offline fallback: text "Stream offline" (existing).
  - Status overlay: absolutely positioned top-left; non-interactive (no click handler).
- **Primary actions**:
  - `View Stream` → closes popup, calls `onViewStream(id)` → opens `ViewStreamSheet` (existing behavior, preserved).
  - `View Recordings` → closes popup, navigates to `/app/recordings?camera={id}` via `router.push` (tenant) or `/admin/recordings?camera={id}` (super admin).
- **Dropdown menu (`⋮`)**:
  - `Set Location` → closes popup, calls `onSetLocation(id, name)` (existing).
  - `Toggle Maintenance`:
    - Reads current `maintenanceMode` state.
    - Opens confirmation dialog (reused from Phase 15-04).
    - On confirm, calls `onToggleMaintenance(id, !currentState)` → API `POST /api/cameras/:id/maintenance` (enter) or `DELETE /api/cameras/:id/maintenance` (exit).
    - Success toast: `กล้อง {name} เข้าโหมดซ่อมบำรุงแล้ว` / `Camera {name} is now in maintenance mode`.
    - Failure toast: `ไม่สามารถเปลี่ยนโหมดได้ · {error}`.
  - `Open Camera Detail` → closes popup, navigates to `/app/cameras/{id}` or `/admin/cameras/{id}` based on portal.
- **Keyboard**: dropdown inherits shadcn `DropdownMenu` keyboard nav (Arrow keys, Enter, Escape).

## Accessibility

- **Stat cards** use semantic `<dl>`? No — keep existing `<p>` (label) + `<span>` (value) pattern from `stat-card.tsx` (unchanged for Phase 18 consistency; future a11y pass can convert).
- **Issues panel rows**: `role="button"` + `tabIndex={0}` + descriptive `aria-label` combining camera name and severity (e.g., `aria-label="Offline camera Lobby Cam — click to investigate"`).
- **Marker pin**: `<svg role="img" aria-label="Camera {name} — status {status}">` inside `L.divIcon`. Badge elements decorative (`aria-hidden="true"`) since status is already in the label.
- **Cluster bubble**: `aria-label="{N} cameras in this area, worst status {status}"`.
- **Popup**: `role="dialog"` inherited from Leaflet; camera name becomes `aria-labelledby` target. Dropdown trigger has explicit `aria-label="More actions for {camera name}"`.
- **Storage forecast chart**: Recharts renders with `role="img"`; companion caption with `daysUntilFull` ensures chart meaning is text-readable.
- **Org Health DataTable**: Phase 14 pattern provides `aria-sort` on headers and row focus states. No new a11y work required.
- **Focus rings**: all interactive elements inherit `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` (shadcn default, tied to `--ring` = green primary).
- **Color contrast**:
  - Pin white camera icon on `#22c55e` green: ratio 2.28:1 (fails AA text, but this is decorative iconography, not text — Lucide stroke 2px meets non-text 3:1 UI contrast per WCAG 2.1 SC 1.4.11).
  - Pin white camera icon on `#ef4444` red: ratio 3.76:1 — passes 3:1 for UI components.
  - Pin white camera icon on `#f59e0b` amber: ratio 2.05:1 — **adds 1px dark stroke (`#000` at 40% opacity) around camera icon path** to guarantee 3:1 on amber/yellow-ish backgrounds. This is the only Claude's Discretion accessibility tuning per CONTEXT.
  - Red recording dot on green/amber pin at 8×8px meets 3:1 against all background fills tested.
- **Animations**: `animate-pulse` recording dot + `reconnecting` pulse respect `prefers-reduced-motion` via Tailwind `motion-safe:animate-pulse` wrapper.
- **Map popup preview video**: `<video muted autoPlay playsInline>` — muted by default meets WCAG 1.4.2 (no auto-audio). Alternative text via popup camera name already present above video.
