3. **Step [1/4] — wss:// upgrade probe** (Phase 27 SC#2):

   Use the WebSocket-handshake-via-curl pattern. Socket.IO needs `EIO=4&transport=websocket` and the standard 4 WS upgrade headers:

   ```bash
   log "[1/4] wss:// upgrade through Caddy (Phase 27 SC#2)"
   # Generate a fresh 16-byte base64 Sec-WebSocket-Key per RFC 6455
   local ws_key
   ws_key=$(openssl rand -base64 16)

   local ws_response
   ws_response=$(curl -i -sS --max-time 10 \
     -H "Connection: Upgrade" \
     -H "Upgrade: websocket" \
     -H "Sec-WebSocket-Key: ${ws_key}" \
     -H "Sec-WebSocket-Version: 13" \
     "https://${DOMAIN}/socket.io/?EIO=4&transport=websocket" 2>&1 || echo "CURL_FAIL")

   if echo "${ws_response}" | grep -qE '^HTTP/1\.1 101 Switching Protocols'; then
     pass_check "wss://${DOMAIN}/socket.io/ → HTTP/1.1 101 Switching Protocols"
     PASS=$((PASS+1))
   else
     fail_check "wss upgrade did not return 101 — Caddy /socket.io handle broken?"
     log "  Response head:"
     echo "${ws_response}" | head -10 | sed 's/^/    /'
     FAIL=$((FAIL+1))
   fi
   echo
   ```
   Note: curl does not parse the WS upgrade further (it is a stream, not HTTP after 101) — we only need the 101 line. Caddy reverse-proxies the upgrade to api:3003; the api side (NestJS Socket.IO) completes the handshake and would accept the connection.

4. **Step [2/4] — HLS reachability via Caddy → SRS** (Phase 30 SC#2 + Phase 29 SC#5 automated):

   The deployed surface for HLS is NOT proxied through Caddy in the v1.3 topology — SRS exposes :8080 directly per docker-compose.yml. The PUBLIC playback contract is via PlaybackService which mints a signed URL pointing at `https://${DOMAIN}:8080/<m3u8 path>` OR via the `/srs-hls/` mount on api. We assert the simpler: SRS HTTP server on :8080 returns a non-empty m3u8 for at least one active stream.

   ```bash
   log "[2/4] HLS reachability via SRS :8080"
   # SRS lists active streams via its HTTP API on 1985 — but 1985 binds 127.0.0.1
   # only (Phase 26 D-07). Query it from inside the srs container.
   local active_streams_json
   active_streams_json=$(${DC} exec -T srs curl -s --max-time 5 'http://127.0.0.1:1985/api/v1/streams' || echo '{}')
   local stream_count
   stream_count=$(echo "${active_streams_json}" | grep -oE '"streams":\[' | wc -l | tr -d ' ')

   if [[ "${stream_count}" -ge 1 ]]; then
     pass_check "SRS reports active streams (operator registered camera per D-14 #2)"
     PASS=$((PASS+1))
   else
     fail_check "SRS reports 0 active streams — operator did not register a camera before running this verifier?"

   fi

   # Probe HLS m3u8 directly from the host (port 8080 is exposed externally per compose)
   ```
