- The wrapper MUST verify all 4 verify-*.sh exist + are executable BEFORE starting any step (pre-flight) — partial run with a missing verifier wastes operator time.

- The IP-detection `curl ipify.org` is BEST-EFFORT — operator may not have outbound HTTP. Falls back to `<vm-public-ip>` placeholder which the operator replaces.
- `record_rc` and `MAX_RC` use `local`-free declarations because they cross subshell boundaries inside the if/else blocks. Test by running `bash -n` after authoring.
- Do NOT use `set +e` to allow individual verifier failures — instead, capture each `bash ...` call inside `if; then ... else local rc=$?; ... fi` so the script keeps `set -e` semantics globally. Note: `local` inside an `if` body that is NOT inside a function is a syntax error — extract failure-handling into a function or use plain assignment.
