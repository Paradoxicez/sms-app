<objective>
Add two bulk verbs — Stop Stream and Stop Recording — to the camera list bulk actions toolbar, mirroring the existing Start variants exactly.

Purpose: Operators currently have row-by-row Stop actions (in the row dropdown) but no way to stop streaming or recording across many selected cameras at once. The Start counterparts have shipped since Phase 20 Plan 03; this is a symmetric gap closure.

Output: A user can multi-select cameras in the cameras data table and click Stop Stream / Stop Recording in the sticky bulk toolbar. The library, hook, toolbar, and container are all extended in lockstep so the new verbs share the existing pre-filter + concurrency + toast + error-tooltip pipeline.

Backend is already in place — `POST /api/cameras/:id/stream/stop` (streams.controller.ts:38) and `POST /api/recordings/stop` (recordings.controller.ts:144). Pure frontend feature.
</objective>
