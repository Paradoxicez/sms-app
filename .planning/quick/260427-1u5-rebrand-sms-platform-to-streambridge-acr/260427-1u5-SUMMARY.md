# Quick Task 260427-1u5: Rebrand SMS Platform to StreamBridge Summary

Replaced all user-facing "SMS Platform" / "SMS" references with "StreamBridge" across the web UI and API boot strings, redesigned the sign-in left panel (dropped the three vanity stat boxes for four protocol pills RTSP/RTMP/SRT/WebRTC plus an "Ingest protocols supported" caption), introduced a reusable `StreamBridgeLogo` SVG component with icon and lockup variants in light/dark themes, and preserved every internal identifier (package names, docker services, env vars, DB schema, SRS templates).

## What Was Built

### Task 1 — StreamBridgeLogo component (commit `ee03850`)

Created `apps/web/src/components/brand/streambridge-logo.tsx`:

- Named-export `StreamBridgeLogo` React component, server-compatible (no `"use client"`)
- Two variants: `"icon"` (48×48 viewBox) and `"lockup"` (220×48 with text)
- Two themes: `"light"` (dot `#22C55E`, text `#0F172A`) and `"dark"` (dot `#06B6D4`, text `#FFFFFF`)

- Brand gradient hardcoded as hex literals `#22C55E → #06B6D4` per design tokens
- `useId()` used for linearGradient id to prevent collisions when multiple logos render on the same page

- **Files modified:** none (environmental)
- **Commit:** none (no source change)

**2. [Rule 3 — Blocking] Orphaned next-server returning 500 on smoke test**

- **Found during:** Task 3 smoke test
- **Issue:** A `next-server (v15.5.15)` was already running on port 3000 in production mode (PID 74780, despite the user's note that the dev server was Turbopack). Removing `apps/web/.next/` to recover from the disk-full above invalidated its in-flight handles, and all three smoke routes returned 500 Internal Server Error.
- **Fix:** Killed PID 74780 and restarted via `pnpm --filter @sms-platform/web dev` (Turbopack). After ~12s warm-up, all three smoke routes returned 200 and HTML contained "StreamBridge", "Ingest protocols supported", "RTSP" with no "SMS Platform" or "99.9%" residue.
- **Files modified:** none (environmental)
- **Commit:** none

Both deviations are environmental, not code-related; the rebrand commits themselves stand on green builds and passing smoke tests.
