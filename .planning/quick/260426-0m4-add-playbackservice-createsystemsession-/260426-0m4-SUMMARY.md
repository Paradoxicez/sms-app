# Quick Task 260426-0m4: PlaybackService.createSystemSession Summary

System-context twin of `PlaybackService.createSession` for background callers (no HTTP request → no CLS ORG_ID), wired into `SnapshotService.refreshOne` to close the cascading "Camera not found" regression introduced by 260426-06n.

## What Changed

### PlaybackService — `createSystemSession(cameraId, orgId)`

New method inserted directly after `createSession`. Mirrors `createSession` line-for-line with two differences:

1. Every Prisma op uses `this.systemPrisma` (RLS bypass) instead of `this.tenantPrisma` (RLS-scoped).
2. The user-viewer-limit check is intentionally skipped — background snapshot/recording tasks are not user viewers.

Defense-in-depth org/camera mismatch check is preserved (`camera.orgId !== orgId → NotFoundException`). JWT claim shape and hlsUrl formula are byte-identical to `createSession`, so the URL FFmpeg requests is the URL `on_play` will validate against — preserves the 260426-06n auth contract.

### SnapshotService — `refreshOne` call swap

Single-line change inside the existing try block: `createSession(...)` → `createSystemSession(...)`. Surrounding comment block updated to reference 260426-0m4 and explain the RLS-bypass rationale. No change to inFlight dedup, MinIO upload, thumbnail update, or the fire-and-forget swallow contract.

### Unit tests

New sibling `describe('createSystemSession (system-context for background callers)', ...)` block in `tests/cluster/load-balancer.test.ts` with two cases:

- **Test A:** `createSystemSession` routes to least-loaded edge using systemPrisma and **does not consult `getViewerCount`** (asserted via `expect(mockStatusService.getViewerCount).not.toHaveBeenCalled()`).
- **Test B:** `createSystemSession` throws `NotFoundException` when `camera.orgId !== orgId` and **does not write a session row** (asserted via `expect(mockPrisma.playbackSession.create).not.toHaveBeenCalled()`).

The same `mockPrisma` is passed to both `tenantPrisma` and `systemPrisma` constructor slots — production code reaches `systemPrisma`; tests assert the contract (return shape, viewer-limit skip, org-mismatch rejection) which is the load-bearing behavior.

## Decisions Made

- **New method vs flag on existing method:** Chose to add `createSystemSession` rather than branch `createSession` on a `system: true` flag. Reasoning: keeps HTTP-path security posture byte-unchanged, makes background-caller intent explicit at every call site, and matches the existing twin-method pattern used in `verifyToken` (uses systemPrisma directly without a tenant-scoped sibling).
- **Viewer-limit skip:** Unconditional for system callers, not configurable. Documented in JSDoc. A future caller that DOES want viewer-limit enforcement should call `createSession` (which requires CLS context and is the correct path for user-facing flows).
- **Test fixture:** Reused the shared mock pattern from the existing `Playback routing to edges` describe block — same `mockPrisma` for both constructor slots. Tests assert the contract (return shape + viewer-limit-skip + org-mismatch reject), not which concrete prisma instance the production class touched. The production code path is verified by the build (TypeScript would reject `this.systemPrisma.camera.findUnique` if the property didn't exist) and by code review.
