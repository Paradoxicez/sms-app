<objective>
Move the snapshot trigger from the SRS `on_publish` callback to the SRS `on_hls` callback (first segment only). Fixes the FFmpeg 404 race where `on_publish` fires before any `.ts` segment exists, so `ffmpeg -i .../live/{org}/{cam}.m3u8` gets `Server returned 404 Not Found` and every snapshot attempt fails.

Purpose: Snapshot capture needs an existing HLS playlist to read from. SRS only writes the playlist after the first segment is produced — which is exactly the moment `on_hls` fires with `seq_no === 0`. Switching the trigger eliminates the race entirely; no retry/backoff needed inside SnapshotService.

Output:
- Two deletions in `srs-callback.controller.ts` (push branch + live branch of `on_publish`).
- One addition inside the existing `onHls` handler: `seq_no === 0` guard + `refreshOneFireAndForget` call.
- Test updates in `apps/api/tests/srs/callbacks.test.ts` covering the new behavior contract.

NO SRS conf change required — `on_hls` is already wired to `/api/srs/callbacks/on-hls` (config/srs.conf:46) and the `onHls` handler already exists for recording archival.
NO SnapshotService change.
NO frontend / DB / module-wiring changes.
